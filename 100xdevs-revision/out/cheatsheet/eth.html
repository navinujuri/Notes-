ETH intro, EVM — Cheat Sheet

Cohort 3.0 · 10 lessons · quick reference

Ethereum & the Blockchain

World computer
Decentralized state machine; every full node re-executes every tx to converge on the same deterministic global state.
Ether units
1 ether = 1e9 gwei = 1e18 wei
Smallest unit = wei; gas prices are quoted in gwei.
Account-based model
Ethereum tracks balances per account (mapping address -> account), unlike Bitcoin's UTXO model.
Block linkage
Ordered blocks; each header holds parentHash + stateRoot + txRoot, keccak256-chained so tampering is detectable.

Account Types

EOA (Externally Owned Account)
controlled by a private key; nonce = tx count sent
Only EOAs can INITIATE a transaction; no code; codeHash = keccak256("").
Contract account
Has bytecode + storage, no private key; only acts when called; nonce = number of contracts it created.
Account state fields (both types)
{ nonce, balance, storageRoot, codeHash }
Address derivation
EOA:     keccak256(pubkey)[12:]
CREATE:  keccak256(rlp([sender, nonce]))[12:]
CREATE2: keccak256(0xff ++ sender ++ salt ++ keccak256(initcode))[12:]
Address = last 20 bytes; CREATE2 lets you compute the address before deploy.

EVM: Virtual Machine Model

Physical vs virtual
EVM is a sandboxed VM (like the JVM) abstracting host hardware/OS; the same bytecode runs identically on any node.
Stack machine
stack: max 1024 items, each 256-bit (32 bytes)
Operands are popped/pushed; there are no CPU registers.
Single-threaded execution
One tx runs at a time, sequentially, to completion — no concurrency, no parallelism. State transitions are ordered.
Deterministic & isolated
No syscalls, no real randomness, no network I/O — same input => same output on all nodes. Don't use block.timestamp for randomness.

Data Locations

storage
contract C { uint256 x; }  // x -> slot 0
Persistent per-contract key-value; 2^256 slots of 32 bytes; survives across txs; most expensive.
memory
bytes memory b = new bytes(64);
Volatile, byte-addressable, zero-initialized, wiped after the call; expansion cost grows quadratically.
calldata
function f(uint[] calldata a) external
Read-only immutable tx input; cheapest for external args — always prefer for external array/bytes params.
stack
Scratch space for the EVM; 1024 deep; "stack too deep" when >16 local slots are simultaneously accessible.
transient storage (EIP-1153)
TSTORE 0x5d / TLOAD 0x5c
Cancun+; storage-like but automatically cleared at end of the transaction. Great for reentrancy locks.

Gas & Gas Limit

Fee formula (EIP-1559)
fee = gasUsed * (baseFee + priorityFee)
baseFee is burned; priorityFee (tip) goes to the validator.
Intrinsic gas (before execution)
21000 base
+ 4 gas / zero calldata byte
+ 16 gas / non-zero calldata byte
+ 32000 for contract creation
Charged up-front regardless of what the code does.
gas limit (per tx)
Max gas you authorize; unused gas is refunded. Set too low => 'out of gas' revert.
Out of gas
All state changes revert, but ALL gas up to the limit is consumed (not refunded).
Block gas limit
~30,000,000 gas/block  (target ~15M under EIP-1559)
Caps total computation per block; baseFee rises when blocks run above target.

Opcodes (Common)

Arithmetic
ADD 0x01  MUL 0x02  SUB 0x03  DIV 0x04  MOD 0x06
Compare / bitwise
LT 0x10  GT 0x11  EQ 0x14  AND 0x16  OR 0x17
Stack ops
PUSH0 0x5f  PUSH1..32 0x60-0x7f  POP 0x50  DUP1 0x80  SWAP1 0x90
Memory / storage
MLOAD 0x51  MSTORE 0x52  SLOAD 0x54  SSTORE 0x55
Context (msg / tx)
CALLER 0x33 (msg.sender)  CALLVALUE 0x34 (msg.value)  ADDRESS 0x30
Control flow
JUMP 0x56  JUMPI 0x57  JUMPDEST 0x5b  RETURN 0xf3  REVERT 0xfd  STOP 0x00
Calls / create
CALL 0xf1  DELEGATECALL 0xf4  STATICCALL 0xfa  CREATE 0xf0  CREATE2 0xf5

Opcodes with Gas

Cheap ops
ADD/SUB/PUSH/DUP/MLOAD/MSTORE = 3   MUL/DIV/MOD = 5   JUMPI = 10
Base context ops (ADDRESS, CALLER) = 2.
SLOAD
cold 2100 / warm 100
EIP-2929: first access to a slot in a tx is 'cold'; subsequent are 'warm'.
SSTORE
0 -> non-zero: 20000
update non-zero: 2900 (+2100 if cold)
Refund up to 4800 gas when clearing a slot back to 0 (EIP-3529).
CALL
warm 100 / cold 2600  (+9000 if value>0, +25000 if new account)
A 2300-gas stipend is forwarded to the callee on a value transfer.
KECCAK256 (0x20)
30 + 6 * ceil(len/32)
CREATE / LOG
CREATE = 32000 + initcode + 200/deployed byte
LOGn = 375 + 375*n + 8/byte

Quick tips & gotchas

  • Only EOAs can initiate a transaction — every contract execution ultimately originates from an EOA-signed tx.
  • storage is ~100-20000x pricier than memory: cache storage vars into a local (memory/stack) inside loops.
  • Reverts refund unused gas but keep the gas already consumed; a true 'out of gas' burns the entire gas limit.
  • Cold vs warm (EIP-2929): the first touch of an address or slot in a tx costs far more — batch accesses to reuse warmth.
  • EVM words are natively 256-bit; uint8 saves NO gas unless multiple small vars pack into one 32-byte storage slot.
  • Cost order calldata < memory < storage — mark external array/bytes params `calldata` to avoid a memory copy.
  • EVM is single-threaded and deterministic: no threads, no true RNG; block.timestamp is validator-influenceable.
  • PUSH0 (0x5f) and TSTORE/TLOAD exist only on Shanghai/Cancun+ — verify the target chain's fork before relying on them.
