Setup & Commands

Install Jest + supertest
npm i -D jest supertest
# ESM: add cross-env & flag, or use babel-jest / ts-jest
Install Vitest + supertest
npm i -D vitest supertest
npm i -D @vitest/coverage-v8
package.json scripts
"scripts": {
  "test": "vitest",           // watch mode
  "test:run": "vitest run",   // one-shot (CI)
  "coverage": "vitest run --coverage"
}
Jest equivalent: "test": "jest --watch", CI uses plain `jest`.
vitest.config.ts
import { defineConfig } from 'vitest/config'
export default defineConfig({
  test: { globals: true, environment: 'node' }
})
globals:true lets you skip imports of describe/it/expect (Jest-like).
Jest config (jest.config.js)
export default {
  testEnvironment: 'node',
  coveragePathIgnorePatterns: ['/node_modules/']
}
Jest ESM needs `node --experimental-vm-modules` or babel-jest; Vitest handles ESM natively.
Run subsets / filters
vitest run auth           # files matching 'auth'
vitest -t "creates user"  # by test name
jest -t "creates user"

Structure, Hooks & Assertions

Suite / test
describe('userService', () => {
  it('returns a user', () => { expect(1 + 1).toBe(2) })
  test.skip('todo', () => {})
  it.only('focus this', () => {})
})
Lifecycle hooks
beforeAll(async () => { /* once */ })
afterAll(async () => { /* once */ })
beforeEach(() => {})
afterEach(() => {})
Value matchers
expect(x).toBe(5)            // Object.is / referential
expect(obj).toEqual({a:1})   // deep, ignores undefined
expect(obj).toStrictEqual({a:1}) // deep + types + undefined
expect(arr).toContain('x')
expect(s).toMatch(/regex/)
Async assertions
await expect(fetchUser()).resolves.toEqual({id:1})
await expect(boom()).rejects.toThrow('fail')
Error assertion
expect(() => parse('')).toThrow(TypeError)
Wrap the call in a fn — passing parse('') throws before expect runs.
Mock-call matchers
expect(fn).toHaveBeenCalledTimes(1)
expect(fn).toHaveBeenCalledWith('a', 2)
expect(fn).toHaveReturnedWith(42)

Testing Express Endpoints (supertest)

Separate app from listen
// app.js  -> export the app, no listen here
export const app = express()
// server.js
import { app } from './app.js'
app.listen(3000)
Critical: import `app` in tests so supertest can bind an ephemeral port itself.
GET request
import request from 'supertest'
import { app } from '../app.js'
it('GET /health', async () => {
  const res = await request(app).get('/health')
  expect(res.status).toBe(200)
  expect(res.body).toEqual({ ok: true })
})
POST with JSON body
const res = await request(app)
  .post('/users')
  .send({ name: 'ada' })
  .set('Authorization', `Bearer ${token}`)
expect(res.status).toBe(201)
expect(res.body).toHaveProperty('id')
Chained expectations
await request(app)
  .get('/users/1')
  .expect('Content-Type', /json/)
  .expect(200)
.expect() throws on mismatch — no separate assertion needed.
Validation / error path
const res = await request(app).post('/users').send({})
expect(res.status).toBe(400)
expect(res.body.error).toMatch(/name required/)

Mocking, Spies & Return Values

Create a mock fn
const fn = vi.fn()          // jest.fn()
fn.mockReturnValue(42)
expect(fn()).toBe(42)
Mock a whole module
vi.mock('../db.js')  // auto-mock, factory optional
vi.mock('../mailer.js', () => ({
  sendMail: vi.fn().mockResolvedValue(true)
}))
vi.mock/jest.mock is HOISTED to top of file — factory can't close over outer vars; use vi.hoisted() if needed.
Return-value control
fn.mockReturnValue(1)          // every call
fn.mockReturnValueOnce(1)      // next call only, queued
fn.mockResolvedValue(user)     // Promise.resolve
fn.mockRejectedValue(new Error('x'))
fn.mockImplementation((a) => a * 2)
Spy on existing method
const spy = vi.spyOn(mathUtils, 'add')
mathUtils.add(2, 3)
expect(spy).toHaveBeenCalledWith(2, 3) // REAL impl still runs
spy.mockReturnValue(99)                // now overridden
Spy vs Mock
Spy: wraps a real fn, records calls, real code runs unless you override. Mock: fully synthetic replacement, no real impl. Use spy to observe, mock to isolate/replace.
Deep mocking nested objects
import { mockDeep } from 'vitest-mock-extended' // jest-mock-extended
const prisma = mockDeep<PrismaClient>()
prisma.user.findUnique.mockResolvedValue({ id: 1 })
mockDeep auto-mocks every nested method/property — ideal for ORMs/clients.
Reset between tests
afterEach(() => {
  vi.clearAllMocks()   // clears calls, keeps impl
  // vi.resetAllMocks() // clears calls + impl
  // vi.restoreAllMocks() // restore spies to original
})
Or config `clearMocks: true`. clear≠reset≠restore — mixing them up leaks state across tests.

Adding a Database

In-memory Mongo (integration)
npm i -D mongodb-memory-server
Spins a real MongoDB in RAM — tests real queries without a shared server.
Wire up memory server
import { MongoMemoryServer } from 'mongodb-memory-server'
import mongoose from 'mongoose'
let mongo
beforeAll(async () => {
  mongo = await MongoMemoryServer.create()
  await mongoose.connect(mongo.getUri())
})
afterAll(async () => {
  await mongoose.disconnect(); await mongo.stop()
})
Clean state per test
afterEach(async () => {
  const { collections } = mongoose.connection
  for (const key in collections) await collections[key].deleteMany()
})
Prevents order-dependent flakiness from leftover docs.
Mock the DB (unit test)
vi.mock('../models/User.js')
User.findById.mockResolvedValue({ _id: '1', name: 'ada' })
Mock the model when unit-testing a service in isolation; use memory-server when the query itself is under test.

Jest → Vitest Migration

API is near drop-in
jest.fn()      -> vi.fn()
jest.mock()    -> vi.mock()
jest.spyOn()   -> vi.spyOn()
jest.useFakeTimers() -> vi.useFakeTimers()
jest.clearAllMocks() -> vi.clearAllMocks()
Keep Jest globals
Set test.globals:true in vitest config so describe/it/expect need no import — otherwise `import { describe, it, expect } from 'vitest'`.
Why switch
Vitest: native ESM/TS, no babel, faster HMR watch, shares Vite config. Jest ESM needs experimental flags + transforms.
Timers & mocks import
import { vi } from 'vitest'  // replaces global `jest`
`jest` global doesn't exist in Vitest — rename or alias.
Gotchas
Snapshot format differs slightly; `jest.requireActual` -> `vi.importActual` (async). Mock hoisting semantics match.

CI/CD & Test Pyramid

GitHub Actions workflow
.github/workflows/test.yml:
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: npm }
      - run: npm ci
      - run: npm run test:run -- --coverage
Use `vitest run` / `jest` (non-watch) in CI — watch mode hangs the runner.
Fail fast / gate merges
- run: npm run lint
- run: npm run test:run
Make the job a required status check on the branch protection rule so red tests block merge.
Unit tests
One module in isolation, deps mocked. Fast, many. E.g. a pure service function with mocked repo.
Integration tests
Multiple real units together (route + service + in-memory DB). Supertest + mongodb-memory-server lives here.
E2E tests
Full stack via HTTP/browser (Playwright/Cypress), real-ish env. Slowest, fewest — run in separate CI job/stage.
Pyramid ratio
Many unit > some integration > few E2E. Push logic down to fast unit tests; reserve E2E for critical user flows.

Quick tips & gotchas

  • Export `app` separately from `app.listen()` — supertest binds its own ephemeral port, so a listening server in the module causes port conflicts and open handles.
  • toBe = Object.is (references/primitives); toEqual = deep, ignores undefined; toStrictEqual = deep + checks undefined props and types. Wrong choice = false pass/fail.
  • vi.mock / jest.mock are hoisted above imports — the factory runs before any top-level const, so it can't reference outer variables. Use vi.hoisted() to share values.
  • clearAllMocks (wipe calls), resetAllMocks (calls + implementations), restoreAllMocks (undo spies). Set clearMocks:true in config or call in afterEach to stop state leaking between tests.
  • Use mockResolvedValue/mockRejectedValue for async deps — mockReturnValue(Promise.resolve()) works but is noisier and easy to forget to await.
  • Clean DB collections in afterEach with mongodb-memory-server; leftover docs cause order-dependent flaky tests that pass alone but fail in the suite.
  • In CI always run the non-watch command (`vitest run`, plain `jest`) or the job hangs forever waiting for file changes.
  • Prefer spyOn when you need the real implementation to run while asserting it was called; reach for a full mock only when you must replace behavior or cut a slow/external dependency.
