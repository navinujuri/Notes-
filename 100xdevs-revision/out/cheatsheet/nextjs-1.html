NextJS (Client side) — Cheat Sheet

Cohort 2.0 · 14 lessons · quick reference

Bootstrap & structure

Create an app
npx create-next-app@latest my-app
cd my-app && npm run dev
App Router layout
app/
  layout.tsx      # shared shell (persists)
  page.tsx        # UI for '/'
  dashboard/
    page.tsx      # UI for '/dashboard'
A folder = a route segment; page.tsx renders it.

Routing

Static route
app/about/page.tsx  ->  /about
Dynamic route
app/blog/[slug]/page.tsx

export default function Page({ params }: { params: { slug: string } }) {
  return <h1>{params.slug}</h1>
}
Route group (no URL segment)
app/(marketing)/about/page.tsx  ->  /about
Parentheses group files/layouts without adding to the URL.
Navigate
import Link from 'next/link'
<Link href="/dashboard">Go</Link>

// programmatic (client component):
const router = useRouter(); router.push('/dashboard')

Server vs Client components

Server (default)
// no directive needed
export default async function Page() {
  const data = await db.query(...)   // runs on server
  return <div>{data.title}</div>
}
Ships zero JS, can touch DB/secrets, can be async.
Client
'use client'
import { useState } from 'react'
export default function Counter() {
  const [n, setN] = useState(0)
  return <button onClick={() => setN(n+1)}>{n}</button>
}
Needed for state, effects, event handlers, browser APIs.
Rule
Server can import Client; Client cannot import Server (pass server-rendered JSX as children instead). Keep 'use client' at the leaves.

Layouts

Root layout
// app/layout.tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return <html><body><Nav />{children}</body></html>
}
Persists across navigation — Nav doesn't remount/refetch.
Nested layout
app/dashboard/layout.tsx  // wraps only /dashboard/*

SEO & metadata

Static metadata
export const metadata = {
  title: 'Dashboard',
  description: '...'
}
Dynamic metadata
export async function generateMetadata({ params }) {
  const post = await getPost(params.slug)
  return { title: post.title }
}

Quick tips & gotchas

  • 'use client' does NOT mean client-only — the component still server-renders once, then hydrates.
  • Default to Server Components; only reach for Client Components at interactive leaves to keep the JS bundle small.
  • Fetching in a Client Component reintroduces waterfalls + exposes endpoints — prefer fetching in a Server Component.
  • Route groups (folder) organise layouts without changing the URL.
  • A hydration mismatch = server HTML ≠ first client render (often Date.now(), random, or window access during render).
