Setup & Monorepo

Three backend services + one frontend, own tsconfig each

deploy/    # worker: queue -> build -> S3
request/   # serve dist/<id>/* by subdomain
frontend/  # React (Vite) dashboard

Shared S3 / R2 client (aws.ts)

S3 client pointed at Cloudflare R2
import { S3 } from "aws-sdk";
export const s3 = new S3({
  accessKeyId: process.env.S3_ACCESS_KEY,
  secretAccessKey: process.env.S3_SECRET_KEY,
  endpoint: "https://<acct>.r2.cloudflarestorage.com",
});
R2 is S3-compatible: same SDK, just set endpoint. Never hardcode keys.
Upload one file
import fs from "fs";
export const uploadFile = async (key: string, localPath: string) => {
  await s3.upload({
    Bucket: "vercel",
    Key: key,
    Body: fs.readFileSync(localPath),
  }).promise();
};
aws-sdk v2 calls need .promise() to await.
Recursively list local files (source walk)
export const getAllFiles = (dir: string): string[] =>
  fs.readdirSync(dir).flatMap(f => {
    const p = path.join(dir, f);
    return fs.statSync(p).isDirectory() ? getAllFiles(p) : [p];
  });

Upload Service (producer)

Non-enumerable deployment id (not auto-increment)
const chars = "abcdefghijklmnopqrstuvwxyz1234567890";
export const generate = () =>
  Array.from({length:5}, () => chars[Math.floor(Math.random()*chars.length)]).join("");
Random id doubles as subdomain + S3 key prefix; sequential ids let others enumerate deployments.
POST /deploy — clone, upload source, enqueue
app.post("/deploy", async (req, res) => {
  const { repoUrl } = req.body;
  const id = generate();
  await simpleGit().clone(repoUrl, path.join(__dirname, `output/${id}`));
  const files = getAllFiles(path.join(__dirname, `output/${id}`));
  await Promise.all(files.map(f =>
    uploadFile(f.slice(__dirname.length + 1), f)));
  publisher.lPush("build-queue", id);
  publisher.hSet("status", id, "uploaded");
  res.json({ id });
});
Return id immediately; the slow build happens async in the worker.
GET /status?id= — for frontend polling
app.get("/status", async (req, res) => {
  const status = await subscriber.hGet("status", req.query.id as string);
  res.json({ status });
});
Redis clients
import { createClient } from "redis";
const publisher = createClient(); await publisher.connect();
const subscriber = createClient(); await subscriber.connect();

Deploy Service (worker / consumer)

Blocking pull loop
while (true) {
  const res = await subscriber.brPop("build-queue", 0); // 0 = block forever
  const id = res!.element;
  await downloadS3Folder(`output/${id}`);
  await buildProject(id);
  await copyFinalDist(id);
  await publisher.hSet("status", id, "deployed");
}
brPop blocks with no busy-wait. LPUSH+BRPOP = FIFO at-least-once queue.
Build with child_process, capture logs + exit code
import { exec } from "child_process";
export const buildProject = (id: string) => new Promise(resolve => {
  const child = exec(`cd ${path.join(__dirname,`output/${id}`)} && npm install && npm run build`);
  child.stdout?.on("data", d => console.log(d));
  child.stderr?.on("data", d => console.error(d));
  child.on("close", code => resolve(code));
});
npm install runs untrusted postinstall scripts — sandbox in a container in prod.
Download S3 prefix (stream each object)
const list = await s3.listObjectsV2({Bucket:"vercel", Prefix:prefix}).promise();
await Promise.all((list.Contents ?? []).map(({Key}) => new Promise(r => {
  const out = path.join(__dirname, Key!);
  fs.mkdirSync(path.dirname(out), {recursive:true});
  s3.getObject({Bucket:"vercel", Key:Key!}).createReadStream()
    .pipe(fs.createWriteStream(out)).on("finish", r);
})));
Upload built dist under its own prefix
export const copyFinalDist = (id: string) => {
  const base = path.join(__dirname, `output/${id}/dist`);
  getAllFiles(base).forEach(f =>
    uploadFile(`dist/${id}/` + f.slice(base.length + 1), f));
};
Keep source (output/<id>) and build (dist/<id>) separate; artifact keyed by id => idempotent, rollback-able.

Request Handler (serving)

Resolve subdomain -> id, path -> S3 key
app.get("/*", async (req, res) => {
  const id = req.hostname.split(".")[0];   // a1b2c3.domain.com
  let filePath = req.path === "/" ? "/index.html" : req.path;
  const obj = await s3.getObject({
    Bucket: "vercel", Key: `dist/${id}${filePath}`,
  }).promise();
  res.set("Content-Type", contentType(filePath));
  res.send(obj.Body);
});
Default to index.html; for SPA client routes fall back to index.html on NoSuchKey, else return 404.
MIME by extension (assets 404/blank if wrong)
const contentType = (p: string) =>
  p.endsWith(".html") ? "text/html"
: p.endsWith(".css")  ? "text/css"
: p.endsWith(".js")   ? "application/javascript"
: p.endsWith(".svg")  ? "image/svg+xml"
: "application/octet-stream";

Frontend (React + Vite)

Upload then poll status until deployed
const { data } = await axios.post(`${UPLOAD_URL}/deploy`, { repoUrl });
setId(data.id);
const t = setInterval(async () => {
  const r = await axios.get(`${UPLOAD_URL}/status?id=${data.id}`);
  if (r.data.status === "deployed") { clearInterval(t); setDone(true); }
}, 3000);
Polling is stateless + reconnect-proof; fine for short builds. Use SSE/WebSocket for live log streaming.
Surface the live URL

Config, DNS & Production

Wildcard DNS -> request handler
*.yourdomain.com  A  <request-handler-ip>
One record routes every deployment subdomain to the same server; id parsed from Host header.
Status state machine (hash in Redis)
uploaded -> building -> deployed | failed
Atomic transitions; on build exit!=0 set status=failed and store stderr for the UI.
Sandbox untrusted builds
# per-build ephemeral container, no host mounts, network + cpu/mem limits, hard timeout
Malicious postinstall can exfiltrate secrets — never build on the host.
Durability upgrade path
Redis LPUSH/BRPOP -> Redis Streams (consumer groups) / SQS (visibility timeout + DLQ)
Naive RPOP loses in-flight jobs on worker crash; idempotent id-keyed builds make retries safe.
CDN in front of serving
Cache keyed on host+path so most requests never hit origin/S3.

Quick tips & gotchas

  • aws-sdk v2 calls are lazy — forget .promise() and nothing runs (or you await undefined). Migrate to @aws-sdk/client-s3 v3 for real promises.
  • Two Redis roles: the deploy worker BRPOPs the queue; the upload service HSETs status. Don't reuse a single subscribed connection for commands — a client in subscribe mode can't run normal commands.
  • Keep source and build artifacts under different prefixes (output/<id> vs dist/<id>). Serving the raw source instead of dist is the classic 'blank page' bug.
  • Set Content-Type from the file extension. Serving .js/.css as text/plain (or octet-stream) makes the browser refuse them and the site loads unstyled/broken.
  • Random, non-enumerable ids only — sequential ints let anyone guess other users' subdomains, and can collide under concurrent uploads.
  • At-least-once queue means a job can run twice; make builds idempotent and keyed by id so a re-run overwrites the same artifact rather than duplicating.
  • Capture build stdout/stderr + exit code and flip status to failed — a silent build failure looks identical to 'still building' to the user.
  • On localhost, id.localhost resolves automatically in Chrome; on a real domain you need a wildcard *.domain.com DNS record pointing at the request handler.
