Typescript — Cheat Sheet

Cohort 2.0 · 11 lessons · quick reference

Setup & the tsc compiler

Install compiler (global or dev dep)
npm i -D typescript
npx tsc --version
Compile a single file → .js
tsc app.ts        # emits app.js
tsc app.ts --outDir dist
Init a project config
tsc --init        # creates tsconfig.json
Compile whole project (uses tsconfig)
tsc               # compiles per tsconfig, no file args
tsc --watch       # recompile on save
Passing file args to tsc IGNORES tsconfig.json.
Type-check only, no output
tsc --noEmit
Great for CI / pre-commit gates.
Run TS directly (no manual build)
npx tsx app.ts    # or: npx ts-node app.ts
Language of TS
TS is a STRONGLY-typed superset of JS, statically checked at compile time. It compiles (transpiles) down to plain JS; types are fully erased at runtime — no runtime cost, no runtime checks.

tsconfig.json essentials

Minimal sane config
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "NodeNext",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
strict
Umbrella flag: turns on noImplicitAny, strictNullChecks, etc. Always keep true.
target vs module
target = JS syntax level emitted; module = module system (CommonJS | ESNext | NodeNext).
outDir / rootDir
Keep source in rootDir, build output in outDir so they don't mix.
noImplicitAny / strictNullChecks
Flag untyped params; make null|undefined distinct types you must handle.

Basic types

Primitives & annotations
let id: number = 1;
let name: string = "x";
let ok: boolean = true;
let big: bigint = 10n;
let sym: symbol = Symbol();
Type inference — skip redundant annotations
let count = 5;        // inferred number
count = "no";         // Error
Let TS infer locals; annotate function params & public APIs.
Functions: params & return
function add(a: number, b: number): number { return a + b; }
const greet = (n: string): void => console.log(n);
Optional & default params
function log(msg: string, level?: string) {}
function inc(x: number, by = 1) {}
Optional params must come after required ones.
any / unknown / never
let a: any;       // opt out of checking (avoid)
let u: unknown;   // must narrow before use
function fail(): never { throw new Error(); }
Prefer unknown over any — forces a type check before use.
Union & literal types
let x: string | number;
let dir: "left" | "right";
null vs undefined
let v: string | null = null;
With strictNullChecks, null isn't assignable to string.

Interfaces & type aliases

Interface (object shape)
interface User {
  id: number;
  name: string;
  email?: string;      // optional
  readonly createdAt: Date;
}
Type alias (any type, not just objects)
type ID = number | string;
type Point = { x: number; y: number };
type Handler = (e: Event) => void;
Interface extends / merges
interface Admin extends User { role: string }
Interfaces are open — same-name declarations MERGE. Types cannot re-declare.
Type composition (intersection)
type Admin = User & { role: string };
Interface vs Type — pick
Interface for object/class shapes & public APIs (extendable, mergeable). Type for unions, primitives, tuples, mapped/conditional types.
Method / index signatures
interface Dict {
  [key: string]: number;   // index signature
  size(): number;
}

Arrays, tuples & enums

Array declarations (two syntaxes)
let nums: number[] = [1, 2, 3];
let strs: Array<string> = ["a"];
Array of objects
let users: User[] = [{ id: 1, name: "a", createdAt: new Date() }];
readonly array
let ro: readonly number[] = [1, 2];
// ro.push(3) → Error
Tuple (fixed length + types)
let pair: [string, number] = ["age", 30];
let rgb: [number, number, number];
Order & length are type-checked; useful for return-multiple.
Numeric enum
enum Direction { Up, Down, Left, Right }
Direction.Up      // 0
Auto-increments from 0; numeric enums are reverse-mapped.
String enum
enum Status {
  Active = "ACTIVE",
  Done   = "DONE"
}
String enums are safer/readable; no reverse mapping.
const enum (inlined, zero runtime)
const enum Log { Info, Warn }
Emits no JS object — values inlined. Or skip enums, use a union of literals.

Generics

Generic function
function identity<T>(arg: T): T { return arg; }
identity<string>("x");
identity(42);        // T inferred
Generic with array
function first<T>(arr: T[]): T | undefined { return arr[0]; }
Constraints (extends)
function len<T extends { length: number }>(x: T): number {
  return x.length;
}
Constrains T so you can safely access .length.
Generic interface / type
interface ApiResp<T> { data: T; status: number }
type Pair<K, V> = { key: K; value: V };
Generic class
class Box<T> {
  constructor(private value: T) {}
  get(): T { return this.value; }
}
Multiple type params + defaults
function merge<A, B = {}>(a: A, b: B) { return { ...a, ...b }; }
Built-in utility types
Partial<T>  Required<T>  Readonly<T>
Pick<T,K>   Omit<T,K>    Record<K,V>
ReturnType<F>  Awaited<P>
Prefer these over hand-rolling generic transforms.

Modules: export & import

Named exports
// math.ts
export const PI = 3.14;
export function add(a: number, b: number) { return a + b; }
Named imports
import { PI, add } from "./math";
import { add as sum } from "./math";   // rename
Default export / import
// user.ts
export default class User {}
// other.ts
import User from "./user";
One default per module; import name is arbitrary.
Mix default + named
import User, { PI } from "./mod";
Namespace / wildcard import
import * as math from "./math";
math.add(1, 2);
Re-export (barrel index.ts)
export { add } from "./math";
export * from "./user";
Type-only import/export
import type { User } from "./user";
export type { ID } from "./types";
Erased at compile time — avoids circular-import & runtime pitfalls.

Quick tips & gotchas

  • Types are erased at runtime — you cannot `if (x instanceof MyInterface)`. Use type guards, `typeof`, `in`, or discriminated unions to narrow.
  • Prefer `unknown` over `any`: `any` disables all checking and silently spreads; `unknown` forces you to narrow before use.
  • `tsc file.ts` ignores tsconfig.json entirely — run bare `tsc` (or `tsc -p tsconfig.json`) to honor project settings.
  • Turn on `strict` from day one. Retrofitting strictNullChecks onto a large codebase is painful.
  • Interfaces merge across declarations (declaration merging) and can `extends`; type aliases cannot be reopened — use interfaces for extensible public shapes.
  • Avoid enums when a union of string literals works: `type Status = 'active' | 'done'` is lighter, fully erased, and easier to narrow.
  • Use `import type`/`export type` for type-only symbols to prevent accidental runtime imports and circular-dependency issues.
  • Let inference do the work for locals and return types; annotate function parameters and exported API boundaries explicitly.
