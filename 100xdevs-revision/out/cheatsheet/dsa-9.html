Binary search, two sum and strings — Cheat Sheet

Cohort 2.0 · 9 lessons · quick reference

Binary Search — core template

Iterative search (returns index or -1)
function bsearch(a, t) {
  let lo = 0, hi = a.length - 1;
  while (lo <= hi) {
    const mid = lo + ((hi - lo) >> 1); // avoids overflow
    if (a[mid] === t) return mid;
    if (a[mid] < t) lo = mid + 1;
    else hi = mid - 1;
  }
  return -1;
}
Array MUST be sorted. O(log n) time, O(1) space. Use `lo + (hi-lo)/2`, never `(lo+hi)/2`.
Loop invariant choice
`while (lo <= hi)` with `hi = mid-1`/`lo = mid+1` for exact-match. Use `while (lo < hi)` with `hi = mid` for boundary/bound searches (see below).
Recursive form
const bs = (a, t, lo=0, hi=a.length-1) =>
  lo > hi ? -1
  : a[(lo+hi)>>1] === t ? (lo+hi)>>1
  : a[(lo+hi)>>1] < t ? bs(a,t,((lo+hi)>>1)+1,hi)
  : bs(a,t,lo,((lo+hi)>>1)-1);
O(log n) stack depth. Iterative is preferred to avoid call overhead.
Infinite-loop trap
If you write `while(lo<hi)` and set `lo = mid` (not mid+1), you loop forever when hi=lo+1. Always advance one side past mid.

lower / upper bound & first-last occurrence

lower_bound — first index with a[i] >= t
function lowerBound(a, t) {
  let lo = 0, hi = a.length; // hi = length, NOT length-1
  while (lo < hi) {
    const mid = lo + ((hi - lo) >> 1);
    if (a[mid] < t) lo = mid + 1;
    else hi = mid;
  }
  return lo; // insertion point; == a.length if all < t
}
This is the C++ std::lower_bound. Returns count of elements < t.
upper_bound — first index with a[i] > t
// same loop, flip the comparison:
if (a[mid] <= t) lo = mid + 1;
else hi = mid;
count of t in range = upperBound(t) - lowerBound(t).
First & last occurrence of t
const first = lowerBound(a, t);          // a[first] === t if present
const last  = upperBound(a, t) - 1;      // last index equal to t
const exists = first < a.length && a[first] === t;
JS has no built-in lower_bound
There is no std lib binary search in JS. `Array.indexOf` is O(n). Roll your own or reuse the templates above.

Binary Search on the Answer (monotonic predicate)

When to use
Answer space is monotone: if x works, everything >= x works (or <=). Binary search the value, not an index. Classics: min ship capacity, Koko eating bananas, sqrt, allocate books.
Generic 'minimum feasible' template
let lo = MIN_ANS, hi = MAX_ANS;
while (lo < hi) {
  const mid = lo + ((hi - lo) >> 1);
  if (feasible(mid)) hi = mid;   // shrink toward smaller answers
  else lo = mid + 1;
}
return lo; // smallest value that satisfies feasible()
For 'maximum feasible', use mid = lo+((hi-lo+1)>>1) (upper mid) and lo=mid / hi=mid-1.
Integer sqrt
function isqrt(n) {
  let lo = 0, hi = n;
  while (lo < hi) {
    const mid = lo + ((hi - lo + 1) >> 1);
    if (mid * mid <= n) lo = mid; else hi = mid - 1;
  }
  return lo;
}
Upper-mid variant prevents infinite loop when setting lo = mid.
Float answer (precision loop)
let lo = 0, hi = X;
for (let it = 0; it < 100; it++) { // fixed iterations, no while
  const mid = (lo + hi) / 2;
  if (feasible(mid)) hi = mid; else lo = mid;
}
return lo;
For reals loop a fixed count (~100) or until hi-lo < eps.

Two Sum patterns

Unsorted array → hash map, O(n)
function twoSum(nums, target) {
  const seen = new Map(); // value -> index
  for (let i = 0; i < nums.length; i++) {
    const need = target - nums[i];
    if (seen.has(need)) return [seen.get(need), i];
    seen.set(nums[i], i);
  }
  return [];
}
Check `need` BEFORE inserting nums[i] so you don't pair an element with itself.
Sorted array → two pointers, O(n) / O(1)
let l = 0, r = a.length - 1;
while (l < r) {
  const s = a[l] + a[r];
  if (s === target) return [l, r];
  if (s < target) l++; else r--;
}
return [];
Only works on sorted input. If you must sort first it becomes O(n log n) and you lose original indices.
Two-sum via binary search
for (let i = 0; i < n; i++) {
  const j = lowerBound(a, target - a[i]);
  if (j < n && j !== i && a[j] === target - a[i]) return [i, j];
}
O(n log n). Two-pointer is strictly better on sorted data; BS variant matters when array is pre-sorted and you query many targets.
Count pairs / all pairs
Sorted + two pointers naturally extends: on match, move both and skip duplicates for distinct pairs (3Sum builds on this).

String toolkit (JS)

Char <-> code
'a'.charCodeAt(0)        // 97
String.fromCharCode(97)  // 'a'
s[i].charCodeAt(0) - 97  // 0..25 index for lowercase
Strings are immutable in JS — `s[i] = 'x'` silently does nothing. Convert with `s.split('')` to mutate.
Frequency array (lowercase)
const f = new Array(26).fill(0);
for (const ch of s) f[ch.charCodeAt(0) - 97]++;
O(1) space vs a Map. Use a Map for unicode / arbitrary charsets.
Common ops
s.slice(i, j)      // substring [i, j)
s.split('').reverse().join('')  // reverse
s.includes('ab')   // substring test
[...s].sort().join('')          // sort chars
`slice` accepts negatives; `substring` does not and swaps args if start>end.
Anagram check
const key = s => [...s].sort().join('');
key(a) === key(b);   // O(n log n)
// or compare 26-length freq arrays for O(n)

Sliding window — Q: Vasya and String

Problem
String of 'a'/'b', you may flip <= k chars. Find longest substring of a single letter. Answer = max over 'keep a' and 'keep b' windows.
Solution — two-pass sliding window
function maxBeauty(s, k) {
  const run = (keep) => {
    let l = 0, flips = 0, best = 0;
    for (let r = 0; r < s.length; r++) {
      if (s[r] !== keep) flips++;        // must flip this char
      while (flips > k) {                 // shrink from left
        if (s[l] !== keep) flips--;
        l++;
      }
      best = Math.max(best, r - l + 1);
    }
    return best;
  };
  return Math.max(run('a'), run('b'));
}
O(n) time, O(1) space. `flips` = count of the OTHER char inside the window; keep it <= k.
Variable-size window recipe
let l = 0;
for (let r = 0; r < n; r++) {
  add(a[r]);
  while (!valid()) { remove(a[l]); l++; }
  best = Math.max(best, r - l + 1);
}
Grow right, shrink left while invalid, record window size. Generalizes 'longest substring with <= k of X'.
Related: longest substring without repeats
const last = new Map(); let l = 0, best = 0;
for (let r = 0; r < s.length; r++) {
  if (last.has(s[r]) && last.get(s[r]) >= l) l = last.get(s[r]) + 1;
  last.set(s[r], r);
  best = Math.max(best, r - l + 1);
}

Quick tips & gotchas

  • Precondition #1 for binary search: the array is SORTED (or the predicate is monotonic). If it isn't, sort first or you'll get garbage.
  • Use `lo + ((hi - lo) >> 1)` for mid. `(lo + hi) / 2` overflows in fixed-width languages and `/2` on non-integers needs a floor in JS.
  • Pick ONE template style and memorize it: `[lo, hi]` closed with `<=` for exact match; `[lo, hi)` half-open with `<` for lower/upper bound. Mixing them causes off-by-one and infinite loops.
  • 'Binary search on the answer' is the highest-leverage pattern — whenever the answer is monotone (feasible past a threshold), search the value space in O(log(range)).
  • Two Sum: hash map for unsorted (O(n)), two pointers for sorted (O(n), O(1) space). Sorting just to two-pointer costs O(n log n) and destroys original indices.
  • JS strings are immutable and there is no built-in binary search — carry your own lowerBound/upperBound helpers into contests.
  • Sliding window replaces an O(n^2) substring scan with O(n): expand right, contract left while the window is invalid. Vasya-style 'flip <= k' problems track the count of the minority char.
  • Off-by-one sanity check: run your BS on arrays of size 0, 1, 2 and with target smaller than all / larger than all elements before trusting it.
