Actionable Docker — Cheat Sheet

Cohort 2.0 · 6 lessons · quick reference

Install & Verify

Linux (Ubuntu/Debian) — official convenience script
curl -fsSL https://get.docker.com | sh
Fast path; for prod prefer the apt repo install from docs.docker.com.
macOS / Windows
# Install Docker Desktop from docker.com
# includes engine, CLI, compose, and a Linux VM
On Windows use the WSL2 backend for near-native Linux perf.
Run docker without sudo (Linux)
sudo usermod -aG docker $USER
# log out/in for the group to apply
Adding to the docker group = root-equivalent. Only for trusted users.
Confirm it works
docker version      # client + daemon
docker info         # system-wide status
docker run hello-world
If `Cannot connect to the Docker daemon` — the daemon/Desktop isn't running.

What Docker Is For

Run software without polluting your host
docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=pw postgres
Spin up Postgres/Redis/Mongo in seconds; delete the container, host stays clean.
Reproducible environments ("works on my machine" killer)
The image pins OS libs + runtime + deps. Same image runs identically anywhere.
Isolation
Each container = isolated filesystem, network, and processes. Two apps can need different Node/Python versions with no conflict.
Image vs Container
Image = immutable blueprint (a class). Container = a running instance of that image (an object).

Where Packages (Images) Come From

Docker Hub — the default public registry
docker pull redis          # = docker.io/library/redis:latest
docker pull node:20-alpine
Bare names like `redis`/`node` are Official Images. Browse: hub.docker.com.
Image reference anatomy
registry/namespace/repo:tag
ghcr.io/acme/api:1.4.2
No tag => `:latest` (a mutable pointer, not a stable version — pin real tags).
Other registries
ghcr.io/...        # GitHub Container Registry
quay.io/...        # Red Hat Quay
<acct>.dkr.ecr.<region>.amazonaws.com/...  # AWS ECR
Fully-qualified names bypass Docker Hub entirely.
Authenticate to pull/push private images
docker login                 # Docker Hub
docker login ghcr.io -u USER
Token/PAT goes in the password prompt, not your account password.

Common Commands — Images

Pull / list / remove
docker pull nginx:1.27
docker images            # or: docker image ls
docker rmi nginx:1.27
Build from a Dockerfile
docker build -t myapp:1.0 .
docker build -t myapp:1.0 -f api.Dockerfile .
`.` is the build context (files sent to daemon). Add a .dockerignore.
Tag & push
docker tag myapp:1.0 user/myapp:1.0
docker push user/myapp:1.0
Inspect / history
docker inspect nginx:1.27
docker history myapp:1.0
`inspect` dumps JSON — pipe to `jq` or use `--format`.

Common Commands — Containers

Run (foreground) vs detached
docker run -it ubuntu bash        # interactive shell
docker run -d --name web -p 8080:80 nginx
-p HOST:CONTAINER maps ports. -it = interactive TTY; -d = background.
Env vars & volumes
docker run -d \
  -e NODE_ENV=production \
  -v $(pwd)/data:/app/data \
  --name api myapp:1.0
-v HOST:CONTAINER persists data past container life. Use named volumes for DBs.
List / stop / start / remove
docker ps            # running
docker ps -a         # all incl. stopped
docker stop web && docker rm web
docker rm -f web     # force-kill + remove
`docker run --rm ...` auto-deletes the container on exit.
Logs & exec into a running container
docker logs -f web
docker exec -it web sh    # or bash
`exec` runs a NEW process; use it to debug a live container.
Copy files & check stats
docker cp web:/etc/nginx/nginx.conf .
docker stats           # live CPU/mem
Cleanup
docker system prune -a        # dangling images, stopped containers, networks
docker volume prune
-a also removes unused (not just dangling) images. Frees a LOT of disk.

Common Packages (ready-to-run)

PostgreSQL
docker run -d --name pg -p 5432:5432 \
  -e POSTGRES_PASSWORD=pw -v pgdata:/var/lib/postgresql/data postgres:16
Redis
docker run -d --name redis -p 6379:6379 redis:7
MongoDB
docker run -d --name mongo -p 27017:27017 \
  -e MONGO_INITDB_ROOT_PASSWORD=pw mongo:7
MySQL
docker run -d --name mysql -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=pw mysql:8
Nginx (static/reverse proxy)
docker run -d -p 8080:80 \
  -v $(pwd)/site:/usr/share/nginx/html:ro nginx
Node / language base images
docker run -it --rm -v $(pwd):/app -w /app node:20-alpine npm ci
`-alpine` tags are tiny; `-slim` is a middle ground. Great for one-off tooling.

Dockerfile & Compose Essentials

Minimal Node Dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
EXPOSE 3000
CMD ["node", "index.js"]
Copy manifests + install BEFORE copying source so the deps layer caches.
Multi-container with Compose (docker-compose.yml)
services:
  api:
    build: .
    ports: ["3000:3000"]
    depends_on: [db]
  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: pw
YAML file; `depends_on` controls start order, not readiness.
Compose lifecycle
docker compose up -d     # build+start
docker compose logs -f
docker compose down      # add -v to drop volumes
Modern Docker: `docker compose` (space), not the old `docker-compose` binary.

Quick tips & gotchas

  • `:latest` is a mutable tag, not "newest stable" — always pin explicit versions (postgres:16, node:20-alpine) for reproducibility.
  • -p HOST:CONTAINER order trips everyone up: the LEFT number is the port on your machine, the RIGHT is inside the container.
  • Container data vanishes on `docker rm`. Anything you must keep (DB files) needs a volume (-v).
  • `docker exec -it <c> sh` to poke around a live container; `docker run` makes a fresh one — don't confuse the two.
  • Alpine images sometimes lack bash/glibc — use `sh` for exec, and watch for native-module build failures.
  • `docker system prune -a` reclaims tons of disk but deletes all unused images — you'll re-pull them next run.
  • Add a .dockerignore (node_modules, .git) or your build context balloons and builds crawl.
  • On Windows/macOS, `localhost` inside a container is NOT your host — use `host.docker.internal` to reach host services.
