Recap & Routing Model

App Router file conventions
app/
  layout.tsx     // shared shell (required at root)
  page.tsx       // route UI
  loading.tsx    // Suspense fallback
  error.tsx      // 'use client' error boundary
  not-found.tsx  // 404 UI
  route.ts       // API endpoint (GET/POST...)
Folders = URL segments. Only page.tsx / route.ts make a segment routable.
Server vs Client Components
// default = Server Component (RSC)
// opt into client with directive at top of file:
'use client';
RSC: no hooks/state/browser APIs; can be async. 'use client' is contagious to children imports.
Special file segment types (this part)
(group)      // route group — no URL segment
[id]         // dynamic segment
[...slug]    // catch-all
[[...slug]]  // optional catch-all
_folder      // private (excluded from routing)

Route Groups ( )

Organize without touching URL
app/(marketing)/about/page.tsx   -> /about
app/(shop)/cart/page.tsx         -> /cart
Parens are stripped from the path — purely for grouping + scoped layouts.
Scoped layout per group
app/(marketing)/layout.tsx  // wraps only marketing pages
app/(app)/layout.tsx        // separate shell for the app
Multiple root layouts
app/(marketing)/layout.tsx  // each has its own <html><body>
app/(app)/layout.tsx
Delete the top-level app/layout.tsx and give each group a root layout. Navigating between groups = full page reload.

Dynamic Routes [param]

Single dynamic segment
// app/blog/[slug]/page.tsx  -> /blog/hello
export default async function Page(
  { params }: { params: Promise<{ slug: string }> }
) {
  const { slug } = await params;
  return <h1>{slug}</h1>;
}
Next 15+: params & searchParams are Promises — await them (or use React.use() in a client comp).
Multiple segments
// app/shop/[category]/[id]/page.tsx
const { category, id } = await params;
Read params in a Client Component
'use client';
import { useParams } from 'next/navigation';
const { slug } = useParams();
searchParams (query string)
export default async function Page(
  { searchParams }: { searchParams: Promise<{ q?: string }> }
) {
  const { q } = await searchParams;
}
Accessing searchParams opts the page into dynamic rendering.

Catch-all [...] & Optional [[...]]

Catch-all — one or more segments
// app/docs/[...slug]/page.tsx
// matches /docs/a, /docs/a/b/c   (NOT /docs)
const { slug } = await params; // slug: string[]  e.g. ['a','b','c']
Optional catch-all — also matches parent
// app/docs/[[...slug]]/page.tsx
// matches /docs  AND  /docs/a/b
const { slug } = await params; // string[] | undefined
The extra brackets make the whole segment optional — slug is undefined at /docs.
Typical use
Docs sites, file-tree browsers, i18n prefixes, CMS pages with arbitrary nesting.

Static Site Generation (SSG / ISR)

Pre-render dynamic pages at build time
// app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = await fetch('https://api/posts').then(r => r.json());
  return posts.map((p) => ({ slug: p.slug }));
}
Returned params are baked to static HTML at build. Runs on the server at build time.
Control unknown params
export const dynamicParams = true;  // default: render on-demand + cache
// dynamicParams = false -> 404 for any slug not in generateStaticParams
false = fully static allowlist; true = SSG for known + SSR/ISR for the rest.
Incremental Static Regeneration (ISR)
export const revalidate = 3600; // seconds — rebuild page in background
// or per-fetch:
fetch(url, { next: { revalidate: 60 } });
Force static / dynamic
export const dynamic = 'force-static';  // or 'force-dynamic'
fetch(url, { cache: 'no-store' });      // opt a page into dynamic
In RSC, fetch is cached by default; cookies()/headers()/searchParams flip a page to dynamic.

Hydration & RSC

What hydration is
Server sends static HTML; React attaches event listeners on the client to make it interactive. Server output must match the first client render.
Client Component (needs state/effects/events)
'use client';
import { useState } from 'react';
export default function Counter() {
  const [n, setN] = useState(0);
  return <button onClick={() => setN(n + 1)}>{n}</button>;
}
Common hydration-mismatch causes
// BAD in initial render:
<p>{Date.now()}</p>
<p>{Math.random()}</p>
{typeof window !== 'undefined' && <X/>}
new Date().toLocaleString() // locale/timezone differs
Move client-only values into useEffect, or gate with a mounted flag.
Escape hatches
<time suppressHydrationWarning>{Date.now()}</time>
// or dynamic import with no SSR:
const Chart = dynamic(() => import('./Chart'), { ssr: false });

Middleware

File location & signature
// middleware.ts  (project root, sibling of app/)
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(req: NextRequest) {
  return NextResponse.next();
}
One middleware file per project. Runs BEFORE the request completes, on every matched route.
Redirect / rewrite / block
const token = req.cookies.get('token')?.value;
if (!token) {
  return NextResponse.redirect(new URL('/login', req.url));
}
// rewrite (URL unchanged in browser):
return NextResponse.rewrite(new URL('/proxy', req.url));
Matcher config (scope it!)
export const config = {
  matcher: ['/dashboard/:path*', '/api/:path*'],
};
// exclude static assets:
// matcher: ['/((?!_next/static|_next/image|favicon.ico).*)']
Set headers / cookies on response
const res = NextResponse.next();
res.cookies.set('visited', '1');
res.headers.set('x-custom', 'value');
return res;
Runs on the Edge runtime by default — no Node APIs (fs, native modules). Keep it fast.

Quick tips & gotchas

  • Route groups (folder) never appear in the URL — use them only for organization and scoped/multiple layouts. Navigating between different root layouts triggers a full reload.
  • Next 15+: params and searchParams are Promises. await them in Server Components or unwrap with React.use() in Client Components — forgetting this is the #1 upgrade bug.
  • [...slug] does NOT match the base path; [[...slug]] does (slug is undefined there). Pick optional catch-all when the parent route needs to render.
  • generateStaticParams runs at build. dynamicParams=false makes it a strict allowlist (404 otherwise); true (default) renders unknown params on-demand and caches them.
  • In RSC, fetch is cached by default = static. cookies(), headers(), searchParams, or cache:'no-store' flip a route to dynamic — know which one you tripped.
  • Hydration errors come from non-deterministic first renders: Date.now(), Math.random(), typeof window checks, locale/timezone formatting, and browser extensions injecting DOM.
  • Middleware runs on the Edge runtime by default — no fs/Node native modules. Always set a matcher so it doesn't run on _next static assets and images.
  • Use NextResponse.rewrite to change what renders without changing the browser URL; redirect changes the URL. Set cookies/headers on the response you return, not the request.
