Rust Bootcamp — Cheat Sheet

Cohort 2.0 · 23 lessons · quick reference

Setup, Cargo & Project Layout

Install toolchain (rustup)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
# Windows: download rustup-init.exe
Installs rustc, cargo, rustup. `rustup update` to upgrade; `rustup default stable`.
Verify install
rustc --version && cargo --version
New project / new lib
cargo new my_app        # binary (src/main.rs)
cargo new my_lib --lib  # library (src/lib.rs)
Creates Cargo.toml + git repo. `cargo init` inside an existing dir.
Build / run / check
cargo run           # compile + run
cargo build         # debug build -> target/debug
cargo build --release  # optimized -> target/release
cargo check         # typecheck only, no codegen (fast)
`cargo check` is the fastest feedback loop; use it while iterating.
Format, lint, test
cargo fmt           # rustfmt
cargo clippy        # linter, catches idiom issues
cargo test          # run #[test] fns
Clippy is the single biggest idiom teacher — run it often.
VSCode setup
// install extension: rust-analyzer (NOT the deprecated 'Rust')
rust-analyzer gives inline types, go-to-def, inlay hints. Add CodeLLDB for debugging.
Hello world
fn main() {
    println!("Hello, world!");
}
`println!` is a macro (note the `!`), not a function.

Variables, Types & Control Flow

Immutable by default / mut
let x = 5;          // immutable
let mut y = 10;     // mutable
y += 1;
Reassigning an immutable `let` is a compile error. Mutability is opt-in.
Type annotation & inference
let n: i64 = 42;
let f = 3.14;       // inferred f64
let b: bool = true;
Types inferred when unambiguous; integer default i32, float default f64.
Shadowing
let s = "42";
let s: i32 = s.parse().unwrap();  // new var, new type
Shadowing creates a NEW binding — can change type; different from `mut`.
Constants
const MAX: u32 = 100_000;
Must have explicit type, UPPER_SNAKE, evaluated at compile time.
Scalar types
// i8..i128/isize, u8..u128/usize, f32/f64, bool, char
let c: char = 'z';  // 4 bytes, Unicode scalar
`usize`/`isize` match pointer width; use `usize` for indexing.
if / else (expression)
let big = if n > 10 { "yes" } else { "no" };
`if` is an expression; both arms must return the same type. No parens needed around condition.
loop / while / for
for i in 0..5 { println!("{i}"); }   // 0..5 exclusive, 0..=5 inclusive
while n > 0 { n -= 1; }
let v = loop { break 42; };          // loop returns a value
`loop { break val; }` yields a value. Label loops with `'outer:` to break outer.
Iterate a collection
let arr = [1, 2, 3];
for x in arr.iter() { println!("{x}"); }

Functions & Ownership Core

Function signature
fn add(a: i32, b: i32) -> i32 {
    a + b   // no semicolon = return expression
}
Trailing expression (no `;`) is the return value. `return` also works but is non-idiomatic for the last line.
Ownership rule
let s1 = String::from("hi");
let s2 = s1;        // s1 MOVED into s2
// println!("{s1}"); // ERROR: value moved
Each value has ONE owner; when owner drops, value freed. Assignment/passing moves heap data.
Clone (deep copy)
let s2 = s1.clone();  // explicit deep copy, s1 still valid
Copy types (integers, bool, char, f64, fixed arrays of Copy) copy implicitly; heap types move.
Borrowing (references)
fn len(s: &String) -> usize { s.len() }
len(&s1);           // borrow, s1 keeps ownership
`&` = shared/immutable borrow. Can have many `&` at once.
Mutable borrow
fn push(s: &mut String) { s.push_str("!"); }
let mut s = String::from("hi");
push(&mut s);
Only ONE `&mut` at a time, and no `&` while a `&mut` is live (data-race prevention at compile time).
Stack vs heap (mental model)
let x = 5;                 // stack: fixed size, fast
let s = String::from("x"); // ptr/len/cap on stack, data on heap
Stack = LIFO, known size, cheap. Heap = dynamic size (String, Vec, Box), pointer lives on stack.
Slices
let s = String::from("hello world");
let hello = &s[0..5];   // &str slice, borrows s
Slices are borrows into a contiguous sequence; `&str` is a string slice, `&[T]` a array/vec slice.

Structs, Enums & Pattern Matching

Struct definition
struct User {
    name: String,
    age: u32,
    active: bool,
}
let u = User { name: "a".into(), age: 30, active: true };
Tuple structs: `struct Point(i32, i32);` Unit struct: `struct Unit;`
impl block (methods)
impl User {
    fn new(name: String) -> Self {
        Self { name, age: 0, active: true }
    }
    fn greet(&self) -> String {
        format!("Hi {}", self.name)
    }
}
`&self` borrows, `&mut self` mutates, `self` consumes. No `self` param = associated fn (call as `User::new`).
Enums with data
enum Shape {
    Circle(f64),
    Rect { w: f64, h: f64 },
    Unit,
}
Each variant can carry different data. Enums are Rust's tagged unions / sum types.
match (exhaustive)
let area = match shape {
    Shape::Circle(r) => 3.14 * r * r,
    Shape::Rect { w, h } => w * h,
    Shape::Unit => 0.0,
};
`match` MUST be exhaustive; use `_ =>` as catch-all. It's an expression returning a value.
if let / while let
if let Some(x) = maybe {
    println!("{x}");
}
Concise single-pattern match; skip the boilerplate when you only care about one variant.
match guards & binding
match n {
    x if x < 0 => "neg",
    0 => "zero",
    _ => "pos",
}
Derive traits
#[derive(Debug, Clone, PartialEq)]
struct P { x: i32 }
println!("{:?}", p);  // Debug print
`{:?}` needs Debug, `{:#?}` pretty-prints. Common derives: Debug, Clone, Copy, PartialEq, Eq, Hash, Default.

Error Handling: Result & Option

Option enum
enum Option<T> { Some(T), None }
let found: Option<i32> = v.iter().position(|&x| x == 3).map(|i| i as i32);
Replaces null. Compiler forces you to handle `None`.
Result enum
enum Result<T, E> { Ok(T), Err(E) }
fn parse(s: &str) -> Result<i32, std::num::ParseIntError> {
    s.parse::<i32>()
}
? operator (propagate)
fn read() -> Result<String, std::io::Error> {
    let s = std::fs::read_to_string("f.txt")?;  // early-return Err
    Ok(s)
}
`?` unwraps Ok/Some or returns the Err/None early. Only usable in fns returning Result/Option.
unwrap / expect
let n = "5".parse::<i32>().unwrap();       // panics on Err
let n = "5".parse::<i32>().expect("bad"); // panics with msg
Fine for prototypes/tests; avoid in library code — prefer `?` or match.
Handle Option safely
match config.get("key") {
    Some(v) => println!("{v}"),
    None => println!("missing"),
}
// or: let v = opt.unwrap_or("default");
Combinators: `unwrap_or`, `unwrap_or_else`, `map`, `and_then`, `ok_or`.
panic! (unrecoverable)
panic!("unreachable state");
Unwinds the stack. Use for bugs/invariants, not expected errors.

Collections & Iterators

Vec (growable array)
let mut v: Vec<i32> = Vec::new();
v.push(1); v.push(2);
let v2 = vec![1, 2, 3];
let first = v2[0];          // panics if OOB
let safe = v2.get(0);       // Option<&i32>
`v.get(i)` returns Option (safe); indexing panics on out-of-bounds.
String vs &str
let owned: String = String::from("hi");
let slice: &str = "literal";
let joined = owned + "!";   // + takes String + &str
`String` = owned/growable heap; `&str` = borrowed view. `.to_string()` / `.into()` to convert.
HashMap
use std::collections::HashMap;
let mut m: HashMap<String, i32> = HashMap::new();
m.insert("a".into(), 1);
let v = m.get("a");                 // Option<&i32>
*m.entry("a".into()).or_insert(0) += 1;
`entry().or_insert()` is the idiomatic upsert pattern.
Iterator chains
let sum: i32 = v.iter()
    .filter(|&&x| x % 2 == 0)
    .map(|&x| x * 2)
    .sum();
Iterators are lazy — nothing runs until a consumer (`sum`, `collect`, `for`, `count`).
collect into a Vec
let doubled: Vec<i32> = v.iter().map(|x| x * 2).collect();
`collect()` often needs a type hint (turbofish `::<Vec<_>>()` or annotation).
enumerate / zip
for (i, val) in v.iter().enumerate() { }
for (a, b) in va.iter().zip(vb.iter()) { }

Advanced: Traits, Generics, Lifetimes, Concurrency, Async

Trait (shared behavior)
trait Summary {
    fn summarize(&self) -> String;
    fn preview(&self) -> String { "...".into() } // default impl
}
impl Summary for User {
    fn summarize(&self) -> String { self.name.clone() }
}
Traits = interfaces. `impl Trait for Type`. Default methods can be overridden.
Generics + trait bounds
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut max = &list[0];
    for x in list { if x > max { max = x; } }
    max
}
`<T: Bound>` or `where T: Bound`. Bounds let you call trait methods on generic types.
dyn / impl Trait
fn make() -> impl Summary { User::new("a".into()) }   // static
let items: Vec<Box<dyn Summary>> = vec![];            // dynamic dispatch
`impl Trait` = monomorphized (fast). `dyn Trait` = runtime vtable, needs Box/&.
Lifetimes
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
Lifetimes tie output ref validity to inputs; no runtime cost — purely compile-time checking.
Threads
use std::thread;
let h = thread::spawn(|| { /* work */ });
h.join().unwrap();
Move data in with `move ||`. Share with `Arc<Mutex<T>>` for mutable shared state.
Arc + Mutex (shared mutable)
use std::sync::{Arc, Mutex};
let data = Arc::new(Mutex::new(0));
let d = Arc::clone(&data);
thread::spawn(move || { *d.lock().unwrap() += 1; });
`Arc` = atomic refcount for sharing across threads; `Mutex` = interior mutability with lock.
Async (tokio)
// Cargo.toml: tokio = { version = "1", features = ["full"] }
#[tokio::main]
async fn main() {
    let res = fetch().await;
}
async fn fetch() -> u32 { 42 }
`async fn` returns a Future; nothing runs until `.await`ed on a runtime (tokio/async-std).
Declarative macros
macro_rules! square {
    ($x:expr) => { $x * $x };
}
let n = square!(4);   // 16
Macros expand at compile time. `!` marks a macro call (println!, vec!, format!).
External deps (Cargo.toml)
[dependencies]
serde = { version = "1", features = ["derive"] }
rand = "0.8"
`cargo add serde --features derive` edits Cargo.toml for you. Crates from crates.io.

Quick tips & gotchas

  • Run `cargo clippy` constantly — it teaches idiomatic Rust and catches the mistakes the compiler allows but shouldn't.
  • The borrow checker rule of thumb: many immutable borrows OR exactly one mutable borrow, never both simultaneously.
  • Move vs Copy: primitives (i32, bool, char, f64) copy implicitly; heap types (String, Vec, Box) MOVE — use `.clone()` when you genuinely need a second owner.
  • Avoid `.unwrap()`/`.expect()` in real code; propagate with `?` or handle with `match`. Reserve panics for true invariant violations.
  • `match` must be exhaustive — the compiler forces you to handle every enum variant, including `None`, which is how Rust kills null-pointer bugs.
  • Iterators are lazy: chains of `map`/`filter` do nothing until a consumer like `collect`, `sum`, or a `for` loop drives them.
  • Prefer `&str` for function params and `String` for owned/returned data; prefer `&[T]` over `&Vec<T>` for slice params.
  • String indexing (`s[0]`) is NOT allowed — strings are UTF-8; use `.chars()`, `.bytes()`, or slice byte ranges carefully.
