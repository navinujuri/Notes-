Intro Class 2- DSA — Cheat Sheet

Cohort 2.0 · 13 lessons · quick reference

Prefix Sums — build once, query O(1)

Standard 1-indexed prefix array (size n+1)
// prefix[i] = a[0]+...+a[i-1]
const prefix = [0];
for (let i = 0; i < a.length; i++)
  prefix.push(prefix[i] + a[i]);
Size n+1 with leading 0 removes the l===0 edge case.
Range sum of a[l..r] inclusive
const sum = prefix[r + 1] - prefix[l];
With 1-indexed prefix, no special-casing needed.
Running-prefix trick when you only need the answer (no array)
let run = 0;
for (const x of a) { run += x; /* use run */ }
If you never re-query a range, skip storing the full array.
P1 · Highest Altitude (LC 1732)
var largestAltitude = function(gain) {
  let alt = 0, max = 0;
  for (const g of gain) { alt += g; max = Math.max(max, alt); }
  return max;
};
Altitudes ARE the prefix sums of gain; start at 0, track the max.

Frequency Arrays — count in O(n), O(1) space for fixed alphabet

Lowercase letters -> size-26 array
const freq = new Array(26).fill(0);
for (const c of s) freq[c.charCodeAt(0) - 97]++;
'a'=97. Use 65 for uppercase, 48 for digits. Faster than a Map for small fixed sets.
Arbitrary / large keys -> Map
const freq = new Map();
for (const x of a) freq.set(x, (freq.get(x) || 0) + 1);
P2 · Most frequent character
const freq = new Array(26).fill(0);
for (const c of s) freq[c.charCodeAt(0) - 97]++;
let idx = 0;
for (let i = 1; i < 26; i++) if (freq[i] > freq[idx]) idx = i;
return String.fromCharCode(97 + idx);
First-seen ties win because > (not >=) never overwrites an equal max.

Difference Array — range updates + prefix (line sweep)

Add val to every index in [l, r] for many queries
const diff = new Array(n + 1).fill(0);
for (const [l, r, val] of queries) {
  diff[l] += val;
  diff[r + 1] -= val;   // needs size n+1
}
// recover final array with one prefix pass
for (let i = 1; i < n; i++) diff[i] += diff[i - 1];
Each query becomes O(1); rebuild in O(n) at the end. This is prefix sum run in reverse.
P3 · Maximum platforms needed (timeline / freq buckets)
const line = new Array(T + 2).fill(0);
for (let k = 0; k < n; k++) {
  line[arr[k]]++;        // train arrives -> +1
  line[dep[k] + 1]--;    // frees platform AFTER departure
}
let cur = 0, ans = 0;
for (let t = 0; t <= T; t++) { cur += line[t]; ans = Math.max(ans, cur); }
dep+1 because a train on the platform at its departure minute still occupies it.
P3 · Two-pointer variant (sort arr & dep)
arr.sort((a,b)=>a-b); dep.sort((a,b)=>a-b);
let plat = 0, ans = 0, i = 0, j = 0;
while (i < n) {
  if (arr[i] <= dep[j]) { plat++; i++; ans = Math.max(ans, plat); }
  else { plat--; j++; }
}
return ans;
Use <= (arrival == departure overlaps -> needs its own platform).

Prefix + HashMap — count subarrays by property

P4 · Subarray sums divisible by K (LC 974)
var subarraysDivByK = function(nums, k) {
  const count = new Map([[0, 1]]); // empty prefix, remainder 0
  let sum = 0, ans = 0;
  for (const x of nums) {
    sum += x;
    const r = ((sum % k) + k) % k; // normalize negatives
    const c = count.get(r) || 0;
    ans += c;                       // pair with every earlier same-remainder prefix
    count.set(r, c + 1);
  }
  return ans;
};
Two prefixes with equal remainder => the slice between them is divisible by k. Seed {0:1}.
Sibling pattern · subarray sum == k (LC 560)
const cnt = new Map([[0, 1]]);
let sum = 0, ans = 0;
for (const x of nums) {
  sum += x;
  ans += cnt.get(sum - k) || 0;
  cnt.set(sum, (cnt.get(sum) || 0) + 1);
}
Same skeleton: count how many earlier prefixes make the gap equal the target.

Sliding Window — contiguous subarray/substring templates

Fixed-size window (length k)
let sum = 0;
for (let r = 0; r < n; r++) {
  sum += a[r];
  if (r >= k) sum -= a[r - k];      // evict element leaving window
  if (r >= k - 1) { /* window [r-k+1 .. r] is complete */ }
}
Variable window (longest satisfying a condition)
let l = 0, ans = 0;
for (let r = 0; r < n; r++) {
  add(a[r]);
  while (!valid()) { remove(a[l]); l++; } // shrink until valid again
  ans = Math.max(ans, r - l + 1);
}
Use while (not if) to shrink — the window may need multiple removals.
P5 · K-Beauty of a Number (LC 2269)
var divisorSubstrings = function(num, k) {
  const s = String(num);
  let ans = 0;
  for (let i = 0; i + k <= s.length; i++) {
    const d = Number(s.substr(i, k));
    if (d !== 0 && num % d === 0) ans++;   // skip divide-by-zero
  }
  return ans;
};
Fixed window of k digits; a substring like '00' -> 0 must be excluded.
P6 · Substrings containing all of a, b, c (LC 1358)
var numberOfSubstrings = function(s) {
  const last = { a: -1, b: -1, c: -1 };
  let ans = 0;
  for (let i = 0; i < s.length; i++) {
    last[s[i]] = i;
    ans += 1 + Math.min(last.a, last.b, last.c); // 0 while any char unseen
  }
  return ans;
};
For each right end, all substrings starting <= min(last a,b,c) are valid. -1 sentinel makes the term 0.

Exactly-K via atMost(K) − atMost(K−1)

P7 · Count Nice Subarrays — sliding window (LC 1248)
var numberOfSubarrays = function(nums, k) {
  const atMost = (m) => {
    if (m < 0) return 0;
    let l = 0, cnt = 0, res = 0;
    for (let r = 0; r < nums.length; r++) {
      cnt += nums[r] & 1;
      while (cnt > m) cnt -= nums[l++] & 1;
      res += r - l + 1;              // # subarrays ending at r with <= m odds
    }
    return res;
  };
  return atMost(k) - atMost(k - 1);
};
"Exactly k" of a monotone count = atMost(k) − atMost(k−1). Treat odd as 1 (x & 1).
P7 · Same problem via prefix-count + Map
const cnt = new Map([[0, 1]]);
let odds = 0, ans = 0;
for (const x of nums) {
  odds += x & 1;
  ans += cnt.get(odds - k) || 0;
  cnt.set(odds, (cnt.get(odds) || 0) + 1);
}
return ans;
Map odd-count exactly like subarray-sum==k: odds is the prefix, k is the target.

Pattern Recognition — pick the tool fast

"Sum/answer over range [l..r]", static array
Prefix sum. Precompute once, answer each query in O(1).
"Count subarrays where sum/count has property P"
Prefix + HashMap of seen prefixes (== k, divisible by k, equal remainder). Seed {0:1}.
"Apply many range updates, then read"
Difference array: +val at l, −val at r+1, one prefix pass to materialize.
"Contiguous window with a constraint (longest/shortest/count)"
Sliding window. Fixed size -> evict a[r-k]; variable -> shrink with while(!valid).
"Exactly K ... " of a monotone quantity
atMost(K) − atMost(K−1). Turn a hard equality into two easy at-most windows.
General approach (the technique)
Brute force first -> spot repeated work -> replace it with prefix (cumulative) or window (incremental) so each step is O(1).

Quick tips & gotchas

  • Make prefix arrays size n+1 with a leading 0 — kills the l===0 special case and makes a[l..r] = prefix[r+1]-prefix[l].
  • Normalize negative remainders: ((sum % k) + k) % k. JS % keeps the sign of the dividend, which breaks the divisible-by-k Map.
  • Seed prefix-count maps with {0: 1} — it represents the empty prefix so subarrays starting at index 0 get counted.
  • In difference arrays and platform sweeps, size the array n+1 (or T+2) so the r+1 / dep+1 decrement never overflows.
  • Shrink variable windows with while, not if — one new right element can violate the constraint by more than one unit.
  • Maximum platforms: use <= when comparing arrival to departure; a train arriving exactly when another leaves still needs its own platform.
  • K-Beauty and any 'divides num' check must guard against d === 0 before num % d (e.g. substring '00').
  • 'Exactly k' is almost always atMost(k) - atMost(k-1); guard atMost(m) with m<0 -> return 0.
