Variables & Primitives

Declare (prefer const, then let)
const PI = 3.14;   // cannot reassign
let count = 0;     // reassignable
var old = 1;       // avoid: function-scoped
const/let are block-scoped; var is function-scoped + hoisted.
7 primitive types
"str"        // string
42, 3.14     // number (one type for int+float)
true/false   // boolean
undefined    // declared, no value
null         // intentional empty
10n          // bigint
Symbol('id') // symbol
typeof null === 'object' (historic bug).
Check type at runtime
typeof "hi"     // 'string'
typeof 42       // 'number'
typeof undefined// 'undefined'
typeof [] // 'object' — use Array.isArray([])
Template literals
const name = 'Sam';
`Hi ${name}, 1+1=${1+1}`;
Backticks; supports interpolation + multi-line.
String basics
'abc'.length        // 3
'abc'.toUpperCase() // 'ABC'
'a,b'.split(',')    // ['a','b']
'  x '.trim()       // 'x'

Properties of JS (how it behaves)

Single-threaded
// one call stack, one thing at a time
Async work is offloaded, results queued via the event loop.
Synchronous by default
Code runs top-to-bottom, line-by-line unless you use callbacks/promises/async.
Dynamically typed
let x = 5;   // number
x = 'five';  // now string — legal
Types attach to values, not variables. No compile-time type checks.
Interpreted + JIT compiled
No separate build step; engines (V8) JIT-compile hot paths for speed.
Loosely typed / coercion
'5' + 1   // '51'  (number -> string)
'5' - 1   // 4     (string -> number)
true + 1  // 2
+ favors string concat; other math operators coerce to number.
Falsy values (memorize)
false, 0, '', null, undefined, NaN
Everything else is truthy — including '0', [], {}.

Operators & Syntax

Strict vs loose equality
0 == '0'   // true  (coerces)
0 === '0'  // false (type + value)
Always use === / !== to avoid surprises.
Arithmetic
7 / 2     // 3.5
7 % 2     // 1  (remainder)
2 ** 10   // 1024 (power)
let n=1; n++; // 2
Logical + short-circuit
a && b    // b if a truthy, else a
a || b    // a if truthy, else b
!a        // negate
Nullish coalescing / optional chaining
user?.name          // undefined if user null
port ?? 3000        // 3000 only if port null/undefined
?? differs from || : keeps 0 and ''.
Ternary
const label = age >= 18 ? 'adult' : 'minor';
Statements end with ;
ASI auto-inserts semicolons, but be explicit to avoid edge-case bugs.

Control Flow & Loops

if / else if / else
if (x > 0) { }
else if (x < 0) { }
else { }
switch
switch (day) {
  case 'sat':
  case 'sun': return 'weekend';
  default: return 'weekday';
}
break each case or it falls through (used above intentionally).
Classic for
for (let i = 0; i < 5; i++) {
  console.log(i);
}
while / do-while
while (cond) { }
do { } while (cond);
for...of (values)
for (const item of [10,20,30]) { }
Use for arrays/strings/iterables.
for...in (keys)
for (const key in {a:1,b:2}) { }
Iterates object keys — avoid on arrays.

Functions

Declaration (hoisted)
function add(a, b) {
  return a + b;
}
Expression
const add = function(a, b) { return a + b; };
Arrow function
const add = (a, b) => a + b;
const sq = x => x * x;   // 1 arg, implicit return
No own `this` — inherits from enclosing scope.
Default parameters
function greet(name = 'guest') { }
Rest params + spread
function sum(...nums) { return nums.length; }
Math.max(...[1,2,3]);   // spread into args
Callbacks
[1,2,3].forEach(n => console.log(n));
A function passed as an argument to be called later.

Complex Types — Arrays

Create & access
const a = [1, 2, 3];
a[0]        // 1
a.length    // 3
a.at(-1)    // 3 (last)
Add / remove (mutating)
a.push(4);    // add end
a.pop();      // remove end
a.unshift(0); // add front
a.shift();    // remove front
a.splice(1,2);// remove 2 from idx 1
map — transform
[1,2,3].map(x => x * 2);   // [2,4,6]
Returns a NEW array; same length.
filter — keep some
[1,2,3,4].filter(x => x % 2 === 0); // [2,4]
reduce — fold to one value
[1,2,3].reduce((sum, x) => sum + x, 0); // 6
find / includes / indexOf
[1,2,3].find(x => x > 1);  // 2
[1,2,3].includes(2);       // true
Copy / merge with spread
const copy = [...a];
const all  = [...a, ...b];
Arrays are reference types — assignment copies the reference, not the data.

Complex Types — Objects & JSON

Create & access
const user = { name: 'Sam', age: 30 };
user.name       // dot
user['age']     // bracket (dynamic key)
Add / update / delete
user.email = 'a@b.com';
user.age = 31;
delete user.age;
Destructuring
const { name, age = 0 } = user;
const [first, ...rest] = [1,2,3];
Rename: const { name: n } = user;
Spread / merge objects
const patched = { ...user, age: 40 };
Shallow copy; later keys win.
Iterate object
Object.keys(user)    // ['name','age']
Object.values(user)  // ['Sam', 30]
Object.entries(user) // [['name','Sam'],...]
Nested / methods
const obj = {
  x: 1,
  greet() { return 'hi ' + this.x; }
};
Method shorthand; `this` = the object when called as obj.greet().
JSON convert
JSON.stringify(user)      // obj -> string
JSON.parse('{"a":1}')     // string -> obj
stringify drops undefined/functions.

Quick tips & gotchas

  • Use === / !== always; == coerces types and hides bugs (0 == '' is false but 0 == '0' is true).
  • Objects and arrays are reference types: `const b = a` copies the pointer, so mutating b changes a. Copy with {...a} / [...a].
  • const prevents reassignment, NOT mutation — you can still push to a const array or edit a const object's fields.
  • Prefer const by default; reach for let only when you must reassign; avoid var (function-scoped + hoisted).
  • Memorize the 6 falsy values: false, 0, '', null, undefined, NaN — everything else is truthy (even [] and {}).
  • map/filter/reduce return new arrays and don't mutate; push/pop/splice/sort mutate in place.
  • Arrow functions have no own `this`/`arguments` — don't use them as object methods that need `this`.
  • typeof null is 'object' and typeof [] is 'object' — use Array.isArray() to detect arrays.
