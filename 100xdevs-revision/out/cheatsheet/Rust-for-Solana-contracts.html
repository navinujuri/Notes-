Core Syntax: Variables, Types, Control Flow

Immutable by default; mut to reassign
let x = 5;          // immutable
let mut y = 5;      // mutable
y += 1;
let x = x + 1;      // shadowing (new binding, type may change)
Scalar + compound types
let n: u64 = 42;         // i8..i128, u8..u128, usize
let f: f64 = 3.14;
let b: bool = true;
let c: char = 'A';
let t: (i32, f64) = (1, 2.0);   // tuple -> t.0
let a: [u8; 3] = [1, 2, 3];      // fixed array
String vs &str
let owned: String = String::from("hi");
let slice: &str = &owned[0..1];
let n: u64 = "42".parse().unwrap();
String = heap-owned, growable; &str = borrowed view.
if / else are expressions
let sign = if n >= 0 { "pos" } else { "neg" };
Loops: loop / while / for
let v = loop { break 10; };          // loop returns value
for i in 0..5 { }                    // 0..=5 inclusive
for (i, x) in arr.iter().enumerate() { }
while cond { }
Functions
fn add(a: i32, b: i32) -> i32 { a + b }  // last expr = return, no ;
fn noop() {}                              // returns ()

Ownership, Borrowing, Lifetimes

Move semantics (heap values)
let s1 = String::from("x");
let s2 = s1;          // s1 MOVED, no longer valid
let s3 = s2.clone();  // deep copy if you need both
Copy types (i32, bool, char, fixed arrays of Copy) are copied, not moved.
Borrowing: shared (&) vs mutable (&mut)
fn len(s: &String) -> usize { s.len() }        // read-only borrow
fn push(s: &mut String) { s.push('!'); }       // mut borrow
Rule: any number of &, OR exactly one &mut, never both at once.
Lifetimes tie reference validity
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
Anchor account structs are generic over <'info>: Account<'info, T>.

Structs, Enums, Pattern Matching

Struct + impl (methods & associated fn)
struct Point { x: i32, y: i32 }
impl Point {
    fn new(x: i32, y: i32) -> Self { Self { x, y } }  // associated
    fn dist2(&self) -> i32 { self.x*self.x + self.y*self.y } // method
}
let p = Point::new(3, 4);
Enums with data
enum Shape {
    Circle(f64),
    Rect { w: f64, h: f64 },
    None,
}
match (exhaustive)
let area = match shape {
    Shape::Circle(r) => 3.14 * r * r,
    Shape::Rect { w, h } => w * h,
    Shape::None => 0.0,
};
_ => is the catch-all arm; compiler forces you to cover every variant.
if let / while let (concise match)
if let Some(v) = maybe { use_it(v); }
while let Some(x) = stack.pop() { }

Traits & Generics

Define + implement a trait
trait Area {
    fn area(&self) -> f64;
    fn name(&self) -> &str { "shape" }  // default method
}
impl Area for Point { fn area(&self) -> f64 { 0.0 } }
Derive common traits via macro
#[derive(Debug, Clone, Copy, PartialEq, Default)]
struct Config { retries: u8 }
#[derive(...)] auto-generates trait impls at compile time.
Generics + trait bounds
fn largest<T: PartialOrd + Copy>(list: &[T]) -> T {
    let mut m = list[0];
    for &x in list { if x > m { m = x; } }
    m
}
// where-clause form: fn f<T>(t: T) where T: Display + Clone {}
Generic struct / enum
struct Wrapper<T> { value: T }
enum Either<L, R> { Left(L), Right(R) }
let w = Wrapper { value: 5u64 };

Error Handling, Option, Iterators, Crates

Result + ? operator
fn read(s: &str) -> Result<u64, std::num::ParseIntError> {
    let n = s.parse::<u64>()?;   // early-return Err
    Ok(n * 2)
}
unwrap()/expect("msg") panic on Err; use ? to propagate instead.
Option enum
fn first(v: &[u8]) -> Option<u8> { v.first().copied() }
let x = first(&data).unwrap_or(0);
match first(&data) { Some(b) => .., None => .. }
Iterators (lazy; chain then collect)
let evens: Vec<i32> = (1..=10)
    .filter(|n| n % 2 == 0)
    .map(|n| n * n)
    .collect();
let total: u64 = nums.iter().sum();
let found = nums.iter().find(|&&n| n > 5);
iter() borrows, into_iter() consumes, iter_mut() mutably borrows.
External crates (Cargo.toml)
[dependencies]
borsh = "1"
serde = { version = "1", features = ["derive"] }
anchor-lang = "0.31"
cargo add <crate>; features gate optional functionality.

Serialization: Serde & Borsh

Serde (JSON / off-chain, human-facing)
use serde::{Serialize, Deserialize};
#[derive(Serialize, Deserialize)]
struct User { id: u64, name: String }
let j = serde_json::to_string(&u)?;
let u: User = serde_json::from_str(&j)?;
Borsh (on-chain, deterministic binary)
use borsh::{BorshSerialize, BorshDeserialize};
#[derive(BorshSerialize, BorshDeserialize)]
struct State { count: u64 }
let bytes = state.try_to_vec()?;
let s = State::try_from_slice(&bytes)?;
Solana/Anchor account & instruction data use Borsh — compact, no self-describing schema, order-dependent.

Solana Program: Anchor, Context, Accounts, IDL

Program skeleton (#[program])
use anchor_lang::prelude::*;
declare_id!("Fg6PaFpo...zPFsLnS");

#[program]
pub mod counter {
    use super::*;
    pub fn increment(ctx: Context<Increment>) -> Result<()> {
        let c = &mut ctx.accounts.counter;
        c.count = c.count.checked_add(1).unwrap();
        msg!("count = {}", c.count);
        Ok(())
    }
}
Context<T> — the accounts + metadata bag
// ctx.accounts.<name>   the validated accounts
// ctx.program_id        &Pubkey of this program
// ctx.bumps.<pda>       canonical PDA bump
// ctx.remaining_accounts extra AccountInfos
Context<T> is generic over your #[derive(Accounts)] struct.
Accounts validation struct
#[derive(Accounts)]
pub struct Increment<'info> {
    #[account(mut)]
    pub counter: Account<'info, Counter>,
    pub authority: Signer<'info>,
    pub system_program: Program<'info, System>,
}
init + PDA + space
#[account(
    init, payer = authority, space = 8 + 8,
    seeds = [b"counter", authority.key().as_ref()], bump
)]
pub counter: Account<'info, Counter>,
space = 8 (Anchor discriminator) + sum of field sizes.
State account (#[account] adds discriminator + Borsh)
#[account]
pub struct Counter { pub count: u64 }
Custom errors
#[error_code]
pub enum MyError {
    #[msg("value too large")] TooBig,
}
require!(v < 100, MyError::TooBig);
return err!(MyError::TooBig);
Transaction & IDL (concepts)
// Transaction: 1+ instructions, signed, executed ATOMICALLY
//   instruction = { program_id, accounts[], data (Borsh) }
// IDL: JSON at target/idl/<prog>.json describing
//   instructions, accounts, types -> drives TS client codegen
Client: new Program(idl, provider).methods.increment().accounts({..}).rpc().
Native (no Anchor) entrypoint
use solana_program::{entrypoint, entrypoint::ProgramResult,
    account_info::AccountInfo, pubkey::Pubkey};
entrypoint!(process_instruction);
fn process_instruction(id: &Pubkey, accounts: &[AccountInfo],
    data: &[u8]) -> ProgramResult { Ok(()) }

Quick tips & gotchas

  • Variables are immutable by default — reach for `let mut` only when you truly reassign; prefer shadowing (`let x = ...`) to change type.
  • Borrow checker mantra: many `&` XOR one `&mut`, and no reference may outlive its owner. Anchor account structs carry this as the `'info` lifetime.
  • On-chain data is Borsh, not Serde/JSON — order and types must match exactly across program and client, and Borsh has no self-describing schema.
  • Always budget 8 bytes for the Anchor discriminator: `space = 8 + <fields>`; forgetting it is the classic init size bug.
  • Do arithmetic with `checked_add`/`checked_mul` (or enable `overflow-checks = true`) — silent wraparound on lamports/counters is an exploit vector.
  • Use `msg!("...")` for on-chain logging; `println!` does nothing in a BPF program.
  • Propagate errors with `?` and typed `Result`; reserve `unwrap()`/`expect()` for truly-impossible cases since they panic (and abort the instruction).
  • After `anchor build`, run `anchor keys sync` so `declare_id!` and Anchor.toml match the deployed program keypair, then redeploy.
