Setup & Wallet Funding (devnet)

Install Solana CLI + spl-token
sh -c "$(curl -sSfL https://release.solana.com/stable/install)"
cargo install spl-token-cli   # or comes with solana tools
Anchor for programs: `cargo install --git https://github.com/coral-xyz/anchor anchor-cli`
Point to devnet & create keypair
solana config set --url devnet
solana-keygen new              # writes ~/.config/solana/id.json
solana address
Never fund a devnet keypair from mainnet — separate wallets per cluster.
Fund wallet (airdrop SOL)
solana airdrop 2
solana balance
Devnet only; mainnet has no faucet. Rate-limited — use faucet.solana.com if throttled.
Airdrop in code (@solana/web3.js)
const sig = await connection.requestAirdrop(pk, 2 * LAMPORTS_PER_SOL);
await connection.confirmTransaction(sig, "confirmed");
LAMPORTS_PER_SOL = 1e9. Confirm before assuming balance is spendable.
ETH tooling (Foundry / Hardhat)
curl -L https://foundry.paradigm.xyz | bash && foundryup
forge init myproj      # or: npx hardhat init
Fund testnets (Sepolia) via a faucet; mainnet costs real ETH for gas.

Solana vs Ethereum (mental model)

State model
ETH:    contract owns its storage (mapping balances)
Solana: programs are stateless; data lives in separate Accounts
On Solana you pass every account a tx touches explicitly — no implicit storage reads.
Native token & fees
ETH: ETH, gas priced in gwei, variable
SOL: SOL, fixed ~5000 lamports/sig, sub-cent
Solana fees are near-constant; ETH gas spikes with congestion.
Execution & speed
ETH: EVM, Solidity/Vyper, ~12s blocks
SOL: SVM, Rust (Anchor), ~400ms slots, parallel (Sealevel)
Solana parallelizes txs that don't touch the same accounts.
Token standard
ETH: ERC-20 = one contract per token
SOL: SPL Token = one shared program, tokens are Mint accounts
On Solana every holder needs an Associated Token Account (ATA) per mint.
Rent
Solana accounts must hold min SOL to be rent-exempt
No ETH equivalent — creating accounts (ATAs, mints) costs a refundable rent deposit.

AMM Recap (x·y = k)

Constant product invariant
x * y = k          // x = reserveIn, y = reserveOut
price = y / x      // spot price of X in terms of Y
Pool can never be fully drained — output asymptotes as input grows.
Swap output with 0.3% fee (Uniswap V2)
amountInWithFee = amountIn * 997
amountOut = (amountInWithFee * reserveOut) /
            (reserveIn * 1000 + amountInWithFee)
Fee (3/1000) stays in the pool, accruing to LPs. Larger trade = worse price (slippage).
Add liquidity (must match ratio)
amountB = amountA * reserveB / reserveA
LP_minted = min(dx/x, dy/y) * totalLP
Deposit off-ratio and the router refunds the excess of one side.
Price impact / slippage
impact = 1 - (amountOut / amountIn) / (reserveOut / reserveIn)
Set amountOutMin = quotedOut * (1 - slippageTolerance) to protect the trade.

Impermanent Loss

IL formula (vs just holding)
r = P_new / P_old              // price ratio of the volatile asset
valueRatio = 2 * sqrt(r) / (1 + r)
IL = valueRatio - 1            // always <= 0
'Impermanent' because it vanishes if price returns to entry; realized only on withdrawal.
Reference values
1.25x -> -0.6%    2x -> -5.7%
3x   -> -13.4%    4x -> -20.0%
5x   -> -25.5%    (same for equivalent drops)
Symmetric: a 4x pump and a 4x dump cause identical IL.
Net LP return
netReturn = feesEarned - IL
LPing is profitable only when accrued swap fees outweigh IL — favor correlated/stable pairs.

Create a Token — SPL (Solana CLI)

Mint a new token
spl-token create-token --decimals 6
# => Creating token <MINT_ADDRESS>
Decimals are fixed at creation. USDC uses 6; SOL/wrapped uses 9.
Create your ATA + mint supply
spl-token create-account <MINT>
spl-token mint <MINT> 1000000
Amount is in whole tokens; CLI scales by decimals. You must be the mint authority.
Transfer (auto-create recipient ATA)
spl-token transfer <MINT> 100 <RECIPIENT> --fund-recipient
Without --fund-recipient the tx fails if the recipient has no ATA for that mint.
Lock supply / revoke authority
spl-token authorize <MINT> mint --disable
Revoking mint authority makes supply fixed forever — common trust signal for launches.

Create a Token — Code (SPL & ERC-20)

@solana/spl-token
import { createMint, getOrCreateAssociatedTokenAccount, mintTo } from "@solana/spl-token";
const mint = await createMint(conn, payer, payer.publicKey, null, 6);
const ata  = await getOrCreateAssociatedTokenAccount(conn, payer, mint, payer.publicKey);
await mintTo(conn, payer, mint, ata.address, payer, 1_000_000 * 1e6);
4th arg (freezeAuthority)=null to disable freezing. Amount is in base units — multiply by 10**decimals.
ERC-20 via OpenZeppelin
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
contract MyToken is ERC20 {
  constructor() ERC20("MyToken", "MTK") {
    _mint(msg.sender, 1_000_000 * 10 ** decimals());
  }
}
decimals() defaults to 18. Deploy with `forge create` / Hardhat; verify on Etherscan.

Swap on a DEX

Uniswap V2 router (Solidity)
IERC20(tokenIn).approve(router, amountIn);
IUniswapV2Router(router).swapExactTokensForTokens(
  amountIn, amountOutMin, [tokenIn, tokenOut], msg.sender, block.timestamp + 300
);
ERC-20 swaps need approve() first (allowance). deadline guards against stuck txs.
Get a quote off-chain (V2)
const [ , out ] = await router.getAmountsOut(amountIn, [tokenIn, tokenOut]);
const amountOutMin = out.mul(995).div(1000);  // 0.5% slippage
Never pass amountOutMin=0 in prod — invites sandwich/MEV attacks.
Jupiter aggregator (Solana)
GET https://quote-api.jup.ag/v6/quote?inputMint=<A>&outputMint=<B>&amount=<n>&slippageBps=50
POST /v6/swap  { quoteResponse, userPublicKey }  -> signed & sent
Aggregates Raydium/Orca/etc for best route; slippageBps=50 means 0.5%.

Quick tips & gotchas

  • Devnet airdrops are rate-limited and flaky — retry or use a web faucet; never rely on them in tests without a fallback.
  • Impermanent loss is only realized when you withdraw. If price returns to entry, IL is zero — but fees earned are kept regardless.
  • On Solana every wallet needs an Associated Token Account (ATA) per mint before it can receive that token; use --fund-recipient / getOrCreateAssociatedTokenAccount.
  • SPL decimals and ERC-20 decimals are fixed at creation and differ (SPL often 6/9, ERC-20 usually 18) — always convert to base units before mint/transfer/swap math.
  • ERC-20 swaps require an approve() allowance to the router before swapExactTokensForTokens; SPL uses delegate/authority instead.
  • Always set amountOutMin (ETH) or slippageBps (Solana) — a zero/loose slippage bound gets sandwiched by MEV bots.
  • x·y=k means the pool never empties and price impact grows with trade size relative to reserves — split large orders or route through an aggregator.
  • Revoke mint authority (spl-token authorize --disable / no _mint path) once supply is final to prove the supply can't be inflated.
