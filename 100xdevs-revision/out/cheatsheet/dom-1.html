Basics of DOM — Cheat Sheet

Cohort 2.0 · 8 lessons · quick reference

DOM Fundamentals: static vs dynamic

What the DOM is
Browser parses HTML into a live in-memory tree of node objects (document -> html -> head/body -> ...). JS mutates this tree; the rendered page reflects it instantly. The DOM is NOT the HTML source file.
Root entry points
document            // whole page
document.documentElement  // <html>
document.head
document.body
document is a global; everything hangs off it.
Static HTML
<ul id="todos">
  <li>Buy milk</li>
</ul>
Markup shipped as-is from the server; content fixed at load, no JS mutation.
Dynamic HTML
const li = document.createElement('li');
li.textContent = 'Walk dog';
document.getElementById('todos').append(li);
JS creates/updates/removes nodes at runtime in response to data, events, timers, fetches.
Run JS after DOM is parsed
<script src="app.js" defer></script>
// or
document.addEventListener('DOMContentLoaded', init);
Selecting elements before they exist returns null. defer runs script after parse; inline scripts at end of <body> also work.

Fetching / selecting elements

By CSS selector (modern default)
document.querySelector('.card > h2');   // first match or null
document.querySelectorAll('li.done');    // static NodeList
querySelector takes ANY CSS selector. querySelectorAll snapshot does NOT auto-update.
By id / class / tag (older, faster)
document.getElementById('todos');          // Element or null
document.getElementsByClassName('item');   // live HTMLCollection
document.getElementsByTagName('li');       // live HTMLCollection
getElementById takes the id WITHOUT '#'. getElementsBy* return LIVE collections that update as the DOM changes.
Scoped search (within an element)
const card = document.querySelector('.card');
card.querySelector('button');   // searches only inside card
Iterate matches
document.querySelectorAll('li').forEach(li => {
  console.log(li.textContent);
});
NodeList has forEach. HTMLCollection does NOT — spread it: [...coll].forEach(...).
Traverse from a node
el.parentElement
el.children          // element-only, HTMLCollection
el.firstElementChild / el.lastElementChild
el.nextElementSibling / el.previousElementSibling
el.closest('.card')  // nearest ancestor matching selector
Prefer *Element* variants; childNodes/firstChild include text/whitespace nodes.

Reading / updating content

Text vs HTML
el.textContent = 'Hi <b>x</b>';  // literal text, safe
el.innerHTML  = '<b>Hi</b>';      // parses HTML
el.innerText;                      // rendered text (layout-aware, slower)
NEVER put untrusted input in innerHTML (XSS). Use textContent for user data.
Attributes
el.getAttribute('href');
el.setAttribute('href', '/x');
el.removeAttribute('disabled');
el.hasAttribute('checked');
Many map to properties: el.id, el.href, el.value, el.disabled = true.
data-* attributes
// <div data-user-id="42">
el.dataset.userId;        // "42"  (camelCased)
el.dataset.userId = '99';
kebab-case in HTML -> camelCase in dataset.
Classes
el.classList.add('active');
el.classList.remove('hidden');
el.classList.toggle('open');       // add/remove
el.classList.toggle('open', isOpen); // force state
el.classList.contains('active');
Prefer classList over overwriting el.className (which wipes all classes).
Inline styles
el.style.color = 'red';
el.style.backgroundColor = '#eee';  // camelCase props
el.style.cssText = 'color:red;margin:4px';
CSS 'background-color' -> JS 'backgroundColor'. Prefer toggling classes over per-property styling.
Form values
input.value;              // current value
input.checked;            // checkbox/radio
select.value;
value is a property, not an attribute — getAttribute('value') returns the initial default, not current input.

Adding elements

Create + insert (safe pattern)
const li = document.createElement('li');
li.textContent = 'New item';
li.className = 'todo';
list.append(li);           // add as last child
createElement makes a detached node; nothing renders until it's inserted.
Insertion methods
parent.append(node, 'text');   // end (multiple, allows strings)
parent.prepend(node);          // start
ref.before(node);              // as previous sibling
ref.after(node);               // as next sibling
ref.replaceWith(node);         // swap out
Modern & flexible. Older: appendChild(node) (single node only), insertBefore(new, ref).
insertAdjacentHTML / Element
el.insertAdjacentHTML('beforeend', '<li>x</li>');
// positions: beforebegin | afterbegin | beforeend | afterend
Parses HTML at a precise spot without clobbering existing children (unlike innerHTML +=). Watch XSS.
Clone a node
const copy = node.cloneNode(true);  // true = deep (with children)
cloneNode(false) copies the element only, no descendants. Clones have no event listeners.
Moving vs copying
Inserting an already-attached node MOVES it (a node can have only one parent). Clone first if you need it in two places.

Deleting elements

Remove a node
el.remove();                 // modern, self-removal
No args, removes el from its parent.
Remove a child (legacy)
parent.removeChild(childEl);
Returns the removed node; throws if not actually a child.
Clear all children
el.replaceChildren();        // modern, empties el
// or
el.innerHTML = '';           // simple but reparses
replaceChildren() with no args is the clean way. Loop-based while(el.firstChild) el.removeChild(el.firstChild) also works.
Replace children with new set
list.replaceChildren(li1, li2, li3);
Atomic swap of the entire child list.

More complex elements & patterns

Build nested structure
const card = document.createElement('div');
card.className = 'card';
card.innerHTML = `
  <h2>${title}</h2>
  <p>${body}</p>`;
container.append(card);
Template literals are handy but ${} of user data is an XSS hole — escape or use textContent.
DocumentFragment (batch inserts)
const frag = document.createDocumentFragment();
items.forEach(t => {
  const li = document.createElement('li');
  li.textContent = t;
  frag.append(li);
});
list.append(frag);   // single reflow
Fragment isn't rendered; appending it moves its children and empties it. Avoids per-item reflow in loops.
<template> for reusable markup
// <template id="row"><li class="row"></li></template>
const tpl = document.getElementById('row');
const node = tpl.content.cloneNode(true);
node.querySelector('.row').textContent = 'A';
list.append(node);
template.content is inert (not rendered/queried by page) until cloned + inserted.
Event delegation (dynamic lists)
list.addEventListener('click', e => {
  const li = e.target.closest('li');
  if (li) li.remove();
});
One listener on the parent handles current AND future children — no need to rebind after adding nodes.
Render a data array
list.replaceChildren(
  ...data.map(d => {
    const li = document.createElement('li');
    li.textContent = d.name;
    li.dataset.id = d.id;
    return li;
  })
);
Map data -> nodes, then replaceChildren once.

Quick tips & gotchas

  • querySelectorAll returns a STATIC NodeList; getElementsByClassName/TagName return LIVE HTMLCollections that mutate mid-loop — snapshot with [...coll] before iterating.
  • textContent for user-supplied data, innerHTML only for trusted markup. innerHTML += re-parses and destroys existing event listeners and input state.
  • Selecting elements before the DOM parses returns null — use defer, DOMContentLoaded, or place <script> at end of <body>.
  • input.value is a live property; getAttribute('value') returns only the initial HTML default, not what the user typed.
  • Inserting an existing node moves it (one parent max); cloneNode(true) first to duplicate. Clones do NOT carry event listeners.
  • Batch DOM writes with DocumentFragment or a single replaceChildren/append to avoid layout thrashing (reflow per insert).
  • Use event delegation on a stable parent for dynamically added children instead of rebinding listeners each time you add a node.
  • CSS hyphenated props become camelCase in JS: background-color -> el.style.backgroundColor; and prefer toggling classes over setting many inline styles.
