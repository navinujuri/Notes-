OpenAPI Spec — Cheat Sheet

Cohort 2.0 · 7 lessons · quick reference

Why & What

OpenAPI = language-agnostic contract for HTTP APIs
Single source of truth for endpoints, params, schemas, auth. Formerly 'Swagger' (spec renamed at 3.0).
What it buys you
Auto docs (Swagger UI / Redoc), typed client + server codegen, request/response validation, mock servers, contract testing.
Format: YAML or JSON
Both equivalent; YAML is the human-authored default. Versions: Swagger 2.0 (legacy) vs OpenAPI 3.0.x / 3.1.x (3.1 is a strict JSON Schema superset).
Design-first vs code-first
Design-first: write spec, generate stubs. Code-first: annotate code (e.g. Zod), emit spec. Hono+Zod is code-first.

Spec File Anatomy

Top-level skeleton
openapi: 3.0.3
info:
  title: Notes API
  version: 1.0.0
servers:
  - url: https://api.example.com/v1
paths: {}
components: {}
openapi, info, paths are effectively required. servers replaces 2.0's host/basePath/schemes.
info — metadata
title + version mandatory. Add description, license, contact for good docs.
paths — the endpoints
Keyed by URL template, then by HTTP method (get/post/put/delete/patch).
components — reusable definitions
Holds schemas, parameters, responses, requestBodies, securitySchemes. Reference with $ref.
$ref — reuse anything
schema:
  $ref: '#/components/schemas/Note'
Avoids duplication; can point to external files too ('./models.yaml#/Note').

Writing Paths, Schemas & Params

An operation with a JSON body + 201
paths:
  /notes:
    post:
      operationId: createNote
      summary: Create a note
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/NewNote'
      responses:
        '201':
          description: Created
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Note'
operationId must be unique — codegen uses it for function names.
Reusable schema in components
components:
  schemas:
    Note:
      type: object
      required: [id, title]
      properties:
        id: { type: string, format: uuid }
        title: { type: string, maxLength: 200 }
        done: { type: boolean, default: false }
format (uuid/date-time/email) is a hint; required is an array of prop names, not per-field.
Path / query params
    get:
      parameters:
        - name: id
          in: path
          required: true
          schema: { type: string }
        - name: limit
          in: query
          schema: { type: integer, default: 20 }
in: path params are always required: true.
Security scheme + apply
components:
  securitySchemes:
    bearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
security:
  - bearerAuth: []
Top-level security = global default; override per-operation. Empty [] = no scopes.

Hono + Zod + OpenAPI (code-first)

Install
npm i hono @hono/zod-openapi @hono/swagger-ui
@hono/zod-openapi re-exports a patched `z` — import z from it, not from 'zod'.
Define schema, name it for components
import { z } from '@hono/zod-openapi'

const Note = z.object({
  id: z.string().uuid().openapi({ example: 'a1b2' }),
  title: z.string().openapi({ example: 'Buy milk' }),
}).openapi('Note')
.openapi('Note') registers it under components/schemas so $ref reuse works.
createRoute — the typed contract
import { createRoute } from '@hono/zod-openapi'

const getNote = createRoute({
  method: 'get',
  path: '/notes/{id}',
  request: {
    params: z.object({
      id: z.string().openapi({ param: { name: 'id', in: 'path' } }),
    }),
  },
  responses: {
    200: { content: { 'application/json': { schema: Note } },
           description: 'A note' },
  },
})
Path uses {id} (OpenAPI style), not :id. Response keys are numeric status codes.
Wire handler with validated input
import { OpenAPIHono } from '@hono/zod-openapi'

const app = new OpenAPIHono()

app.openapi(getNote, (c) => {
  const { id } = c.req.valid('param')   // fully typed
  return c.json({ id, title: 'Buy milk' }, 200)
})
c.req.valid('param'|'query'|'json') returns parsed+typed data. 400 auto-returned on validation failure.
Register a security scheme
app.openAPIRegistry.registerComponent('securitySchemes', 'bearerAuth', {
  type: 'http', scheme: 'bearer',
})
Then add `security: [{ bearerAuth: [] }]` inside createRoute.

Serve the Spec & Swagger Page

Expose the generated JSON doc
app.doc('/doc', {
  openapi: '3.0.0',
  info: { version: '1.0.0', title: 'Notes API' },
})
GET /doc returns the full spec assembled from all createRoute definitions.
Mount Swagger UI
import { swaggerUI } from '@hono/swagger-ui'

app.get('/ui', swaggerUI({ url: '/doc' }))
url must point at the served spec route. Visit /ui for interactive 'try it out' docs.
Redoc alternative (static HTML)
<script src="https://cdn.redoc.ly/redoc/latest/bundles/redoc.standalone.js"></script>
<redoc spec-url="/doc"></redoc>
Redoc = read-only, cleaner docs; Swagger UI = interactive testing.
Handle servers behind a proxy
Set servers[].url in app.doc so 'try it out' hits the right base URL, not localhost.

Auto-Generated Clients

openapi-typescript — types only (lightweight)
npx openapi-typescript http://localhost:3000/doc -o ./src/api.d.ts
Emits TS types, no runtime. Pair with openapi-fetch for a typed fetch wrapper.
openapi-fetch — typed runtime client
import createClient from 'openapi-fetch'
import type { paths } from './api'

const client = createClient<paths>({ baseUrl: '/v1' })
const { data, error } = await client.GET('/notes/{id}', {
  params: { path: { id: 'a1b2' } },
})
~2kb, no codegen of methods — types drive path/param autocomplete.
Hono RPC (no external gen)
import { hc } from 'hono/client'
const client = hc<typeof app>('http://localhost:3000')
const res = await client.notes[':id'].$get({ param: { id: 'a1b2' } })
End-to-end types straight from the app type — skips OpenAPI entirely for Hono-to-Hono.
orval — full client + React Query hooks
npx orval --input ./doc.json --output ./src/gen/api.ts
Generates hooks/axios/fetch + MSW mocks. Config via orval.config.ts.
openapi-generator-cli — polyglot
npx @openapitools/openapi-generator-cli generate \
  -i doc.yaml -g typescript-axios -o ./client
~7 generators (typescript-axios, python, go, java...). Heavy but language-agnostic.

Quick tips & gotchas

  • Import `z` from '@hono/zod-openapi', never from 'zod' directly — only the patched instance carries the `.openapi()` method.
  • Give every operation a unique operationId: client generators derive function names from it; duplicates or missing ones produce ugly/colliding names.
  • OpenAPI path templates use {braces} ({id}); Hono's own router uses :colons (:id). In createRoute always use the brace form.
  • 3.1 aligns with JSON Schema (nullable via `type: [string, null]`); 3.0 uses `nullable: true`. Generators differ in support — pin your target version.
  • Name your Zod schemas with `.openapi('Name')` so they land in components/schemas and get $ref'd instead of inlined — cleaner docs and smaller generated clients.
  • `required` at the object level is a list of property names, not a per-property flag; forgetting it makes every field optional in generated types.
  • Regenerate clients in CI from the live /doc endpoint (or a committed spec) so drift between server and client is caught, not shipped.
  • Swagger UI 'Try it out' calls the servers[].url — set it explicitly for staging/prod, or requests silently hit localhost.
