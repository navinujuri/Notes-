installing-node-assignments — Cheat Sheet

Cohort 2.0 · 7 lessons · quick reference

Install Node.js

Recommended: install via nvm (manage multiple versions)
# macOS/Linux
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
nvm install --lts
nvm use --lts

# Windows -> use nvm-windows (installer .exe) or the node .msi
Prefer the LTS build for assignments; avoid mixing global installs across versions.
Verify install
node -v   # e.g. v22.x
npm -v    # npm ships with node
If not found, restart the terminal / re-source shell profile so PATH updates.
List / switch versions (nvm)
nvm ls
nvm install 20
nvm use 20
Bun as an alternative runtime (covered later in course)
curl -fsSL https://bun.sh/install | bash
bun run index.js
Not required for week 2; Node is the default for these assignments.

Run JS with Node & project setup

Run a single file
node index.js
node ./easy/palindrome.js
REPL (interactive scratchpad)
node
> [1,2,3].map(x => x*2)
> .exit   # or Ctrl+D
Great for testing one-liners without a file.
Init a project + package.json
npm init -y
Creates package.json so you can add deps and scripts.
Install deps (assignments often use jest to self-test)
npm install            # install from package.json
npm install --save-dev jest
node_modules is generated locally — never commit it; use .gitignore.
npm scripts
// package.json
"scripts": { "test": "jest", "start": "node index.js" }

// run
npm test
npm start
Auto-restart on save (dev loop)
npx nodemon index.js
Node 18+ also has built-in watch: node --watch index.js

Modules: export / import your solutions

CommonJS (Node default, what most week-2 assignments use)
// solution.js
function isPalindrome(str) { /* ... */ }
module.exports = isPalindrome;

// test file
const isPalindrome = require('./solution');
Multiple exports
module.exports = { add, sub };
const { add, sub } = require('./math');
ES Modules (opt in)
// package.json -> "type": "module"  (or use .mjs)
export function add(a, b) { return a + b; }
import { add } from './math.js';
With ESM you must include the .js extension in relative imports.

Easy JS assignments — core patterns

Sum / reduce an array
const total = arr.reduce((acc, n) => acc + n, 0);
Count vowels
const count = (str.match(/[aeiou]/gi) || []).length;
|| [] guards against null when there are no matches.
Palindrome check (normalize first)
const clean = str.toLowerCase().replace(/[^a-z0-9]/g, '');
const ok = clean === [...clean].reverse().join('');
Expenditure analysis (group/sum by key)
const byCategory = transactions.reduce((acc, t) => {
  acc[t.category] = (acc[t.category] || 0) + t.price;
  return acc;
}, {});
Find largest element
const max = Math.max(...arr);   // arr must be non-empty

Medium JS — strings, timing, classes

Anagram check (sorted-chars key)
const norm = s => [...s.toLowerCase()].sort().join('');
const isAnagram = (a, b) => norm(a) === norm(b);
Time a synchronous function
const start = Date.now();
calculateSum();
const ms = Date.now() - start;
For sub-ms precision use performance.now() (ms as float).
Class with state (e.g. Todo / Counter assignment)
class Todo {
  constructor() { this.todos = []; }
  add(t) { this.todos.push(t); }
  remove(i) { this.todos.splice(i, 1); }
  getAll() { return this.todos; }
}
module.exports = Todo;
Frequency map
const freq = {};
for (const ch of str) freq[ch] = (freq[ch] || 0) + 1;

Hard JS — filesystem & JSON

Read a file (sync, simplest for a one-off script)
const fs = require('fs');
const text = fs.readFileSync('./a.txt', 'utf-8');
Pass 'utf-8' or you get a raw Buffer instead of a string.
Count words in a file
const words = text.trim().split(/\s+/).filter(Boolean);
const count = words.length;
Clean a file: strip extra spaces, write back
const cleaned = text.replace(/\s+/g, ' ').trim();
fs.writeFileSync('./a.txt', cleaned);
Parse / stringify JSON
const data = JSON.parse(fs.readFileSync('./todos.json', 'utf-8'));
fs.writeFileSync('./todos.json', JSON.stringify(data, null, 2));
null, 2 pretty-prints with 2-space indent.

Easy async — callbacks & setTimeout

Defer work with setTimeout
setTimeout(() => console.log('ran after ~1s'), 1000);
Delay is a minimum, not a guarantee — it queues after current sync code finishes.
Callback-style function
function later(cb) {
  setTimeout(() => cb('done'), 500);
}
later(msg => console.log(msg));
Read file asynchronously (callback API)
const fs = require('fs');
fs.readFile('./a.txt', 'utf-8', (err, data) => {
  if (err) return console.error(err);
  console.log(data);
});
Error-first callback: always handle err before using data.
Print without blocking the event loop
console.log('1');
setTimeout(() => console.log('2'), 0);
console.log('3');
// -> 1, 3, 2
Even 0ms defers to after the synchronous stack drains.

Medium async — promises & async/await

Promisify a delay
const wait = ms => new Promise(res => setTimeout(res, ms));
await wait(1000);
fs Promises API + async/await
const fs = require('fs/promises');
async function main() {
  const data = await fs.readFile('./a.txt', 'utf-8');
  await fs.writeFile('./out.txt', data.toUpperCase());
}
main();
await only works inside an async function (or top-level in ESM).
Sequential vs parallel
// sequential (slow): each awaits the previous
const a = await f1();
const b = await f2();

// parallel (fast): start both, then wait
const [x, y] = await Promise.all([f1(), f2()]);
Error handling
try {
  const data = await fs.readFile('./missing.txt', 'utf-8');
} catch (err) {
  console.error('failed:', err.message);
}
Promise chains use .catch(); async/await uses try/catch.
Convert callback API to promise
const { promisify } = require('util');
const readFile = promisify(require('fs').readFile);
await readFile('./a.txt', 'utf-8');

Quick tips & gotchas

  • node -v / npm -v is your first sanity check — if the shell can't find node, reopen the terminal so PATH refreshes.
  • Assignments usually export a function via module.exports and are graded by jest — run npm test, don't add console.logs that break the return contract.
  • Return the value, don't console.log it. Test runners check what your function returns, not what it prints.
  • fs sync methods (readFileSync) block the event loop — fine for tiny CLI scripts, never for servers.
  • Always pass 'utf-8' to fs read calls or you'll get a Buffer instead of a string.
  • setTimeout delay is a floor, not exact; async code runs after the current synchronous stack finishes.
  • await must live inside an async function; forgetting await gives you a pending Promise, not the value.
  • Use Promise.all for independent async tasks to run them in parallel instead of awaiting one-by-one.
