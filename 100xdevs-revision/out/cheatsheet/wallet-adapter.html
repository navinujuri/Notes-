Client side apps in Solana, Wallet adapter — Cheat Sheet

Cohort 3.0 · 8 lessons · quick reference

Scaffold & Install

Create React app (Vite recommended)
npm create vite@latest my-dapp -- --template react
cd my-dapp && npm install
Install wallet-adapter + web3.js
npm i @solana/web3.js \
  @solana/wallet-adapter-base \
  @solana/wallet-adapter-react \
  @solana/wallet-adapter-react-ui \
  @solana/wallet-adapter-wallets
Buffer polyfill (Vite only — web3.js needs Node globals)
npm i buffer
// main.jsx (top of file)
import { Buffer } from 'buffer';
globalThis.Buffer = Buffer;
Without this you get "Buffer is not defined" on Vite. CRA already polyfills it.

Wrap app in providers

Provider tree: Connection > Wallet > WalletModal
import { ConnectionProvider, WalletProvider } from '@solana/wallet-adapter-react';
import { WalletModalProvider } from '@solana/wallet-adapter-react-ui';
import { clusterApiUrl } from '@solana/web3.js';
import { useMemo } from 'react';
import '@solana/wallet-adapter-react-ui/styles.css'; // required for default UI

function Root() {
  const endpoint = useMemo(() => clusterApiUrl('devnet'), []);
  return (
    <ConnectionProvider endpoint={endpoint}>
      <WalletProvider wallets={[]} autoConnect>
        <WalletModalProvider>
          <App />
        </WalletModalProvider>
      </WalletProvider>
    </ConnectionProvider>
  );
}
Order matters: ConnectionProvider outermost, WalletModalProvider innermost.
Endpoint options
clusterApiUrl('devnet')       // https://api.devnet.solana.com
clusterApiUrl('mainnet-beta') // rate-limited; use a paid RPC in prod
'https://api.devnet.solana.com' // or a raw string
wallets prop — usually just []
// Phantom/Solflare/Backpack implement the Wallet Standard
// and auto-register, so an empty array is fine.
<WalletProvider wallets={[]} autoConnect>

// Only add explicit adapters for wallets NOT auto-detected:
import { UnsafeBurnerWalletAdapter } from '@solana/wallet-adapter-wallets';
const wallets = useMemo(() => [new UnsafeBurnerWalletAdapter()], []);
Wrap adapter array in useMemo([]) so it isn't recreated every render.

Connect UI + core hooks

Drop-in connect / disconnect buttons
import { WalletMultiButton, WalletDisconnectButton }
  from '@solana/wallet-adapter-react-ui';

<WalletMultiButton />      // Connect / shows address when connected
<WalletDisconnectButton />
useConnection — RPC connection object
import { useConnection } from '@solana/wallet-adapter-react';
const { connection } = useConnection();
useWallet — everything about the connected wallet
import { useWallet } from '@solana/wallet-adapter-react';
const {
  publicKey,       // PublicKey | null (null when disconnected)
  connected,
  sendTransaction,
  signMessage,     // may be undefined if wallet lacks support
  signTransaction,
} = useWallet();
Always guard on publicKey before RPC calls; it's null until connected.

Requesting airdrop (faucet)

Request devnet SOL to the connected wallet
import { useConnection, useWallet } from '@solana/wallet-adapter-react';
import { LAMPORTS_PER_SOL } from '@solana/web3.js';

async function airdrop(amount) {
  const { connection } = useConnection();      // from component scope
  const { publicKey } = useWallet();
  if (!publicKey) throw new Error('Wallet not connected');
  const sig = await connection.requestAirdrop(
    publicKey,
    amount * LAMPORTS_PER_SOL
  );
  await connection.confirmTransaction(sig, 'confirmed');
}
LAMPORTS_PER_SOL
LAMPORTS_PER_SOL === 1_000_000_000 // 1 SOL = 1e9 lamports
web3.js works in lamports (integers). Always multiply/divide by this.

Showing user balance

Fetch balance (lamports → SOL)
const lamports = await connection.getBalance(publicKey);
const sol = lamports / LAMPORTS_PER_SOL;
Fetch in an effect, re-run on key change
import { useEffect, useState } from 'react';
const [bal, setBal] = useState(0);
useEffect(() => {
  if (!publicKey) return;
  connection.getBalance(publicKey).then(l => setBal(l / LAMPORTS_PER_SOL));
}, [publicKey, connection]);
Live updates via account subscription
const id = connection.onAccountChange(publicKey, (acc) =>
  setBal(acc.lamports / LAMPORTS_PER_SOL));
// cleanup: connection.removeAccountChangeListener(id);
Cheaper than polling; unsubscribe in the effect cleanup.

Sign a message

Sign arbitrary bytes (off-chain, no fee)
const { publicKey, signMessage } = useWallet();
if (!signMessage) throw new Error('Wallet does not support message signing');

const message = new TextEncoder().encode('Sign into my dapp');
const signature = await signMessage(message); // Uint8Array (64 bytes)
Verify the signature (ed25519)
import { ed25519 } from '@noble/curves/ed25519';
const ok = ed25519.verify(signature, message, publicKey.toBytes());
if (!ok) throw new Error('Invalid signature');
@noble/curves ships with web3.js. Signing proves ownership without spending gas — used for auth.

Sending SOL

Build + send a transfer transaction
import { useConnection, useWallet } from '@solana/wallet-adapter-react';
import {
  Transaction, SystemProgram, PublicKey, LAMPORTS_PER_SOL,
} from '@solana/web3.js';

async function sendSol(to, amountSol) {
  const { connection } = useConnection();
  const { publicKey, sendTransaction } = useWallet();

  const tx = new Transaction().add(
    SystemProgram.transfer({
      fromPubkey: publicKey,
      toPubkey: new PublicKey(to),
      lamports: amountSol * LAMPORTS_PER_SOL,
    })
  );

  const sig = await sendTransaction(tx, connection); // wallet signs + sends
  await connection.confirmTransaction(sig, 'confirmed');
  return sig;
}
sendTransaction auto-fills recent blockhash + feePayer and prompts the wallet to sign.
Assignment shape (tie it together)
// One component that: connects wallet, shows balance,
// has an Airdrop button, a Send-SOL form, and a Sign-Message button.
// Reuse the snippets above; guard every action on `publicKey`.

Quick tips & gotchas

  • Import '@solana/wallet-adapter-react-ui/styles.css' once — without it the WalletMultiButton renders unstyled.
  • publicKey is null until connected: guard every RPC/tx call with `if (!publicKey) return` to avoid runtime crashes.
  • Everything on-chain is in lamports (integers). Convert with LAMPORTS_PER_SOL (1e9); never send floats as lamports.
  • Devnet airdrops are heavily rate-limited — request 1-2 SOL max; if it 429s, use faucet.solana.com in the browser.
  • signMessage/signTransaction can be undefined depending on the wallet — check before calling, don't assume support.
  • Wrap the `wallets` array and `endpoint` in useMemo so providers don't reset the connection on every render.
  • Modern wallets (Phantom, Solflare, Backpack) auto-register via the Wallet Standard, so `wallets={[]}` is enough — you rarely need explicit adapters.
  • On Vite, polyfill Buffer (globalThis.Buffer = Buffer) or you'll hit 'Buffer is not defined' from web3.js.
  • Always confirmTransaction after airdrop/send before reading balance, or the UI shows stale numbers.
