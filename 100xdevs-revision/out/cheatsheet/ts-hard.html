More Typescript APIs — Cheat Sheet

Cohort 2.0 · 7 lessons · quick reference

Pre-requisites & Setup

Init a TS project
npm init -y
npm i -D typescript
npx tsc --init
tsc --init writes tsconfig.json with sane defaults.
Enable strict mode (needed for these utilities to be useful)
// tsconfig.json
{ "compilerOptions": { "strict": true, "target": "ES2020" } }
strict turns on strictNullChecks — key for Partial/optional props.
Compile & run
npx tsc          # emit JS
npx tsx index.ts  # run TS directly (no build)
Utility types are built-in
Pick/Partial/Readonly/Record/Exclude live in lib.es5.d.ts — no import needed.
Install zod for runtime validation + inference
npm i zod
zod needs strict + "strictNullChecks": true for correct inference.

Pick<T, K> — keep a subset of keys

Select specific properties into a new type
interface User { id: number; name: string; email: string; age: number }
type UserPreview = Pick<User, "id" | "name">;
// { id: number; name: string }
K is constrained to keyof T
type Bad = Pick<User, "phone">; // Error: "phone" not in keyof User
Implementation (mapped type)
type Pick<T, K extends keyof T> = { [P in K]: T[P] };
Omit = inverse of Pick (drop keys)
type NoEmail = Omit<User, "email">;
// Omit<T,K> = Pick<T, Exclude<keyof T, K>>
Omit key does NOT need to exist on T (no error) — unlike Pick.

Partial<T> — make everything optional

All props become optional (great for update/patch payloads)
function updateUser(id: number, patch: Partial<User>) { /* ... */ }
updateUser(1, { name: "neo" }); // ok, other fields skipped
Implementation
type Partial<T> = { [P in keyof T]?: T[P] };
Required<T> — inverse, strips the optional
type Config = { host?: string; port?: number };
type FullConfig = Required<Config>; // host & port now mandatory
Not deep
Partial is shallow — nested objects stay required. Write a recursive DeepPartial if you need it.

Readonly<T> — immutable properties

Freeze props at the type level
type ROUser = Readonly<User>;
const u: ROUser = { id: 1, name: "a", email: "x", age: 3 };
u.name = "b"; // Error: Cannot assign to 'name' (read-only)
Implementation
type Readonly<T> = { readonly [P in keyof T]: T[P] };
Compile-time only
No runtime protection — the emitted JS can still mutate. Use Object.freeze() for runtime immutability.
Readonly arrays / tuples
const nums: readonly number[] = [1, 2];
nums.push(3); // Error: push does not exist on readonly array
`as const` gives you a deeply readonly literal type.

Record<K, V> — typed key/value maps

Build an object type from a key union + value type
type Page = "home" | "about" | "contact";
type Nav = Record<Page, { title: string }>;
// { home: {...}; about: {...}; contact: {...} } — all keys required
Open-ended dictionary
type Scores = Record<string, number>;
const s: Scores = { alice: 10, bob: 7 };
Implementation
type Record<K extends keyof any, T> = { [P in K]: T };
keyof any = string | number | symbol.
Literal-key Records are exhaustive
Record<"a"|"b", V> requires BOTH keys; missing one is a compile error — good for exhaustive lookup tables.

Exclude<T, U> — remove members from a union

Drop union members assignable to U
type Keys = "a" | "b" | "c";
type NoA = Exclude<Keys, "a">; // "b" | "c"
Works on any union
type T = Exclude<string | number | boolean, boolean>; // string | number
Implementation (conditional + distributive)
type Exclude<T, U> = T extends U ? never : T;
Distributes over each union member; matches collapse to never.
Extract<T, U> — the inverse (keep matches)
type Onlyab = Extract<Keys, "a" | "b">; // "a" | "b"
NonNullable<T> = Exclude<T, null | undefined>
type X = NonNullable<string | null | undefined>; // string

Type inference in zod

Define a schema, derive the TS type with z.infer
import { z } from "zod";
const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
  email: z.string().email().optional(),
});
type User = z.infer<typeof UserSchema>;
// { id: number; name: string; email?: string }
Single source of truth: one schema → runtime validation + static type.
typeof is required
type U = z.infer<UserSchema>;        // Error: UserSchema is a value
type U = z.infer<typeof UserSchema>; // correct
Pass the schema's TYPE (typeof), not the value.
Validate at runtime
const user = UserSchema.parse(req.body);        // throws ZodError on fail
const r = UserSchema.safeParse(req.body);
if (!r.success) console.log(r.error.issues); else use(r.data);
input vs output types (transforms/defaults)
const S = z.object({ age: z.string().transform(Number) });
type In = z.input<typeof S>;   // { age: string }
type Out = z.output<typeof S>; // { age: number } (== z.infer)
z.infer === z.output. Use z.input for the pre-parse shape.
Combine with utility types
type NewUser = Omit<z.infer<typeof UserSchema>, "id">;

Quick tips & gotchas

  • Pick/Omit/Partial/Readonly are shallow (one level). For nested objects write recursive variants (DeepPartial, DeepReadonly).
  • Pick errors if a key isn't in keyof T; Omit silently ignores unknown keys — Omit is more forgiving but hides typos.
  • Readonly and readonly[] are compile-time only. Reach for Object.freeze() when you need runtime enforcement.
  • Record<K,V> with a literal-union K forces every key to be present — use it for exhaustive maps and to catch missing cases.
  • Exclude/Extract/NonNullable only operate on unions and rely on distributive conditional types — wrapping T in [T] disables distribution.
  • These utilities compose: Omit<T,K> is literally Pick<T, Exclude<keyof T, K>>; chain them freely (e.g. Partial<Pick<User,'name'>>).
  • In zod always write z.infer<typeof schema> — forgetting typeof passes a value where a type is expected.
  • When a zod schema has .transform()/.default(), z.infer gives the OUTPUT type; use z.input for the raw pre-parse shape.
