5 tools for senior engineers (Code without your mouse) — Cheat Sheet

Cohort 2.0 · 8 lessons · quick reference

Vim/vi — modes & motions (works on any server)

Modes: Normal (nav), Insert (type), Visual (select), Cmdline (:)
Esc            # back to Normal from anywhere
vi is POSIX — present on virtually every *nix box.
Enter Insert mode
i  a       # before / after cursor
I  A       # start / end of line
o  O       # new line below / above
Move by character / word / line
h j k l    # left down up right
w b e      # word fwd / back / end
0 ^ $      # col0 / first non-blank / EOL
Jump around the file
gg  G      # top / bottom
42G / :42  # go to line 42
Ctrl-d Ctrl-u  # half-page down / up
{  }       # prev / next paragraph
Find char on line, then repeat
f( t,      # jump to / before char
;  ,       # repeat forward / backward
Counts multiply any motion/operator
3w   5j   2dd   d3w
count + operator + motion is the whole grammar.

Vim — editing grammar (operator + text object)

Operator + motion
dw  d$  c$  y}  >i{
d=delete c=change y=yank >=indent.
Text objects (edit by structure, not position)
ciw   # change inner word
di(   # delete inside parens
ci"   # change inside quotes
dap   # delete a paragraph
cit   # change inside HTML/XML tag
i=inner, a=around (includes delimiters/space).
Whole-line ops
dd yy cc   # delete / yank / change line
>> <<      # indent / dedent
Paste, undo, repeat
p  P       # paste after / before
u  Ctrl-r  # undo / redo
.          # repeat last change
Visual select
v   V   Ctrl-v   # char / line / block
gv               # reselect last selection
Search & jump
/foo   n N   # search, next / prev
*            # search word under cursor
%            # jump to matching bracket
Substitute (find/replace)
:%s/old/new/g     # whole file
:%s/old/new/gc    # with confirm
:'<,'>s/x/y/g     # in visual range
Registers & marks
"ayy  "ap    # yank to / paste from reg a
ma  `a       # set mark a / jump to it
:reg  :marks # inspect
Save / quit
:w  :wq  :x  ZZ   # write / write+quit
:q!  ZQ           # quit discard

Neovim — config & IDE features

Config entry point (Lua)
~/.config/nvim/init.lua
$XDG_CONFIG_HOME/nvim; init.vim still works but Lua is idiomatic.
Options & leader
vim.g.mapleader = ' '
vim.opt.number = true
vim.opt.relativenumber = true
vim.opt.expandtab = true
vim.opt.shiftwidth = 2
Keymaps in Lua
vim.keymap.set('n', '<leader>w', ':w<CR>')
vim.keymap.set('i', 'jk', '<Esc>')
opts = { desc=..., silent=true, noremap=true }.
Plugin manager — lazy.nvim
require('lazy').setup({
  { 'neovim/nvim-lspconfig' },
  { 'nvim-treesitter/nvim-treesitter', build=':TSUpdate' },
})
:Lazy to manage; lockfile pins versions.
LSP (go-to-def, hover, rename)
vim.lsp.enable('lua_ls')  -- nvim 0.11+
-- or require('lspconfig').ts_ls.setup{}
Use :Mason to install servers; K=hover, gd=def, <leader>rn=rename.
Diagnostics & health
:checkhealth
:LspInfo  :Lazy  :Mason
:TSUpdate

Vimium — Vim motions in the browser

Install
Vimium (Chrome/Edge) · Vimium-FF (Firefox)
Type 'i' for native insert mode; Esc leaves it. Excluded on some pages (e.g. new-tab).
Scroll
j k        # down / up
d u        # half-page
gg  G      # top / bottom
Follow links via hints
f          # open link in current tab
F          # open link in new tab
Type the highlighted hint letters to activate.
Tabs
J K / gT gt  # prev / next tab
t            # new tab
x  X         # close / restore closed
^ (caret)    # last-visited tab
Navigate & open
o  O       # open URL/bookmark (curr / new tab)
b  B       # open bookmark
H  L       # history back / forward
Utility
gi         # focus first text input
yy         # copy current URL
/  n N     # find in page
?          # show all bindings

Vim in VSCode

Pick an extension
vscodevim.vim         # emulation, easy
asvetliakov.vscode-neovim  # real embedded nvim
vscode-neovim = full plugin/LSP parity; VSCodeVim = simpler, uses VSCode features.
Core settings (settings.json)
"vim.leader": "<space>",
"vim.useSystemClipboard": true,
"vim.hlsearch": true,
"editor.lineNumbers": "relative"
Escape without reaching for Esc
"vim.insertModeKeyBindings": [
  { "before": ["j","k"], "after": ["<Esc>"] }
]
Normal-mode custom binds
"vim.normalModeKeyBindingsNonRecursive": [
  { "before": ["<leader>","w"],
    "commands": ["workbench.action.files.save"] }
]
Non-recursive avoids remap loops; 'commands' calls VSCode actions.
Let VSCode keep some Ctrl keys
"vim.handleKeys": { "<C-d>": true, "<C-a>": false }
false = pass through to VSCode (e.g. Ctrl-a select-all).

tmux — persistent terminal sessions

Sessions
tmux new -s dev     # create named
tmux ls             # list
tmux a -t dev       # attach
tmux kill-session -t dev
Sessions survive SSH disconnects — attach again later.
Prefix key (default)
Ctrl-b   # then a command key
All bindings below are 'prefix, then key'.
Windows (tabs)
c        # new window
n p      # next / prev
0-9      # jump to window N
,        # rename
&        # kill window
Panes (splits)
%        # split vertical
"        # split horizontal
o / arrows  # move focus
z        # zoom/unzoom pane
x        # kill pane
Detach & copy mode
d        # detach (session keeps running)
[        # enter copy/scroll mode (q to exit)
Config ~/.tmux.conf
set -g prefix C-a        # rebind prefix
unbind C-b
set -g mouse on
set -g base-index 1
Reload live: prefix + : then `source-file ~/.tmux.conf`.

Window managers (tiling) & shell config

Tiling WM by OS
macOS : yabai + skhd  ·  AeroSpace
Linux : i3 / sway / Hyprland
Windows: komorebi  ·  FancyZones
Auto-arranges windows in a grid; drive focus/move/resize from the keyboard.
i3 config (~/.config/i3/config)
bindsym $mod+Return exec alacritty
bindsym $mod+j focus left
bindsym $mod+1 workspace 1
bindsym $mod+Shift+q kill
$mod is usually Super/Alt; reload with $mod+Shift+r.
Which shell rc file loads when
bash login   : ~/.bash_profile -> ~/.bashrc
bash non-login: ~/.bashrc
zsh (default): ~/.zshrc
login (both) : ~/.profile / ~/.zprofile
Interactive-but-non-login (new terminal tab) reads .bashrc/.zshrc — put aliases there.
Aliases, exports, functions
alias gs='git status'
export PATH="$HOME/.local/bin:$PATH"
export EDITOR=nvim
mkcd(){ mkdir -p "$1" && cd "$1"; }
Reload without new shell
source ~/.zshrc   # or: . ~/.zshrc
exec $SHELL -l for a clean login re-init.
Prompt & framework add-ons
oh-my-zsh · zinit · starship
eval "$(starship init zsh)"
starship is shell-agnostic (bash/zsh/fish/pwsh).

Quick tips & gotchas

  • vi/vim is guaranteed on almost every server (POSIX) — learn the motions once and you can edit anywhere over SSH.
  • Think grammar, not shortcuts: count + operator + text-object (e.g. 2ci") composes infinitely; text objects (ciw, di() beat raw motions.
  • The dot command '.' repeats your last change — pair it with n (search) for fast repetitive edits, and stay out of Visual mode when '.' would do.
  • Remap Caps Lock to Esc (or Ctrl) at the OS level — the single biggest ergonomic win for Vim and tmux.
  • Rebind the tmux prefix off Ctrl-b (Ctrl-a or Ctrl-Space) and enable mouse on while you learn — reload with source-file.
  • Know your rc file: aliases/exports go in ~/.bashrc or ~/.zshrc (interactive), PATH/env for login shells in ~/.profile/~/.zprofile.
  • Keep dotfiles in a git repo and symlink them (GNU stow or a bootstrap script) so every machine is one clone away.
  • In VSCode prefer vscode-neovim for real plugin/LSP parity; use vim.handleKeys to hand specific Ctrl-combos back to VSCode.
