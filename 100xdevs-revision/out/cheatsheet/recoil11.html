Recoil — Cheat Sheet

Cohort 3.0 · 7 lessons · quick reference

Setup & RecoilRoot

Install
npm install recoil
# or
yarn add recoil
Wrap app in RecoilRoot (required provider)
import { RecoilRoot } from 'recoil';

function App() {
  return (
    <RecoilRoot>
      <Todos />
    </RecoilRoot>
  );
}
Every component using Recoil hooks must be inside a RecoilRoot. You can nest roots; inner root shadows outer state.
Optional: initialize atoms via initializeState
<RecoilRoot initializeState={({ set }) => set(countAtom, 10)}>
Handy for SSR / tests to seed atom values before first render.

The Context API problem Recoil solves

Plain Context: create + provide + consume
const CountContext = createContext(null);

function Provider({ children }) {
  const [count, setCount] = useState(0);
  return (
    <CountContext.Provider value={{ count, setCount }}>
      {children}
    </CountContext.Provider>
  );
}

const { count, setCount } = useContext(CountContext);
Context is for passing data down without prop-drilling — it is NOT a state manager.
Gotcha: ANY context value change re-renders ALL consumers
Every component calling useContext re-renders when the provider value changes, even if it only reads an unrelated field. This is the core pain Recoil fixes with fine-grained subscriptions.
Object value = new reference every render
// BAD: new object each render -> all consumers re-render
<Ctx.Provider value={{ count, setCount }}>
Wrap in useMemo to stabilize, but you still can't do per-field subscriptions like Recoil atoms.

Recoil vs Context API

Subscription granularity
Context: all consumers re-render on any change. Recoil: only components subscribed to the specific atom/selector that changed re-render.
Derived state
Context has no built-in derivation. Recoil selectors compute derived values with automatic caching + dependency tracking.
Async
Context: roll your own. Recoil: async selectors integrate with Suspense out of the box.
Boilerplate / mental model
Context = passing values through the tree. Recoil = a graph of atoms (state) and selectors (derived), consumed via hooks anywhere under RecoilRoot.
When to just use Context
Rarely-changing global values (theme, locale, auth user). Reach for Recoil when many components read/write shared, frequently-changing state.

Atoms — units of state

Define an atom
import { atom } from 'recoil';

const countAtom = atom({
  key: 'countAtom',   // globally unique string
  default: 0,
});
key MUST be unique across the whole app; duplicate keys throw in dev.
useRecoilState — read + write (like useState)
const [count, setCount] = useRecoilState(countAtom);
setCount(c => c + 1);
useRecoilValue — read only (subscribes)
const count = useRecoilValue(countAtom);
useSetRecoilState — write only (NO re-render on change)
const setCount = useSetRecoilState(countAtom);
Use in components that update state but don't display it — avoids unnecessary re-renders.
useResetRecoilState — reset to default
const reset = useResetRecoilState(countAtom);
Object/array default
const todosAtom = atom({
  key: 'todos',
  default: [],
});
setTodos(prev => [...prev, newTodo]);
Treat atom state as immutable — always return a new array/object, never mutate in place.

Selectors — derived state

Read-only selector (get)
import { selector } from 'recoil';

const doubledCount = selector({
  key: 'doubledCount',
  get: ({ get }) => {
    const c = get(countAtom);   // tracks dependency
    return c * 2;
  },
});
Recompiles only when a get() dependency changes; result is cached (memoized) per input.
Consume a selector
const doubled = useRecoilValue(doubledCount);
useRecoilValue works for atoms AND selectors interchangeably.
Writable selector (get + set)
const tempF = selector({
  key: 'tempF',
  get: ({ get }) => get(tempC) * 9/5 + 32,
  set: ({ set }, newF) =>
    set(tempC, (newF - 32) * 5/9),
});
const [f, setF] = useRecoilState(tempF);
set receives ({set,get,reset}, newValue). Use for two-way derived state.
Async selector (Promise get)
const userQuery = selector({
  key: 'userQuery',
  get: async ({ get }) => {
    const id = get(userIdAtom);
    const r = await fetch(`/api/user/${id}`);
    return r.json();
  },
});
Suspends the component; wrap consumer in <Suspense fallback={...}>. Use useRecoilValueLoadable to handle loading/error without Suspense.
Handle async without Suspense
const loadable = useRecoilValueLoadable(userQuery);
if (loadable.state === 'loading') return <Spinner/>;
if (loadable.state === 'hasError') throw loadable.contents;
return <div>{loadable.contents.name}</div>;
loadable.state is 'hasValue' | 'loading' | 'hasError'.

Re-render control: memo & alternate approach

React.memo to stop prop-driven re-renders
const Row = React.memo(function Row({ item }) {
  return <li>{item.text}</li>;
});
Only skips re-render if props are shallow-equal. Recoil already scopes re-renders per subscribed atom; combine both to isolate expensive subtrees.
Write-only hook = the 'alternate approach' to avoid re-renders
// Button re-renders on every count change (bad):
const [count, setCount] = useRecoilState(countAtom);

// Button never re-renders on count change (good):
const setCount = useSetRecoilState(countAtom);
Split display vs. mutation components so the setter component doesn't subscribe.
Selector to isolate the changing slice
// Instead of subscribing to a big object atom,
// derive just the field a component needs:
const titleSel = selector({
  key: 'title',
  get: ({ get }) => get(formAtom).title,
});
Component re-renders only when title changes, not on every formAtom update.
atomFamily / selectorFamily — parameterized state
import { atomFamily } from 'recoil';

const todoItem = atomFamily({
  key: 'todoItem',
  default: id => ({ id, done: false }),
});
const [item, setItem] = useRecoilState(todoItem(id));
Each id gets its own atom instance -> editing one row re-renders only that row (classic list-perf fix vs one giant array atom).

Quick tips & gotchas

  • Every atom/selector key must be globally unique — a stray duplicate throws 'Duplicate atom key' in dev.
  • useRecoilValue subscribes (re-renders); useSetRecoilState does NOT. Pick the narrowest hook for the job.
  • All Recoil hooks must run under a <RecoilRoot>, or you get 'This component must be used inside a RecoilRoot'.
  • Never mutate atom state in place — spread/return new objects so Recoil detects the change.
  • Prefer selectors over computing derived values in render: selectors are cached and only recompute when a get() dependency changes.
  • Async selectors throw Promises — either wrap consumers in <Suspense> or read them with useRecoilValueLoadable.
  • For long lists, use atomFamily(id) instead of one array atom so a single item update re-renders only that item.
  • Context is for low-frequency global values (theme/auth); reach for Recoil when shared state changes often and many components read different slices.
