End to end contract with FE — Cheat Sheet

Cohort 3.0 · 5 lessons · quick reference

Setup, compile & deploy (Hardhat)

Init project + compile
npm i -D hardhat && npx hardhat init
npx hardhat compile   # -> artifacts/contracts/X.sol/X.json (abi + bytecode)
Compile emits BOTH the ABI (FE needs it) and bytecode (goes on-chain).
Local chain for dev
npx hardhat node        # JSON-RPC on http://127.0.0.1:8545, funded accounts
Point MetaMask at localhost:8545, chainId 31337.
Deploy to a testnet
npx hardhat run scripts/deploy.js --network sepolia
Deploy returns the contract ADDRESS — the third thing the FE needs (address + ABI + chain).
Deploy script (ethers v6)
const f = await ethers.getContractFactory("Crowdfund");
const c = await f.deploy(ethers.parseEther("10"));
await c.waitForDeployment();
console.log(await c.getAddress());

Step 1 — Writing the contract (Solidity)

Skeleton: state, events, modifiers
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
contract Crowdfund {
  address public owner;
  uint256 public goal;
  uint256 public totalRaised;
  event Contributed(address indexed from, uint256 amount);
  constructor(uint256 _goal){ owner = msg.sender; goal = _goal; }
  modifier onlyOwner(){ require(msg.sender == owner, "not owner"); _; }
}
payable state-changing fn (costs gas)
function contribute() external payable {
  require(msg.value > 0, "zero");
  totalRaised += msg.value;
  emit Contributed(msg.sender, msg.value);
}
Emit an event so the FE can react to the change without polling.
view/pure fn (free read)
function getStatus() external view returns (uint256, uint256) {
  return (totalRaised, goal);
}
view = reads state, pure = touches no state. Both are free via eth_call.
Access control + safe ETH send
function withdraw() external onlyOwner {
  (bool ok,) = owner.call{value: address(this).balance}("");
  require(ok, "transfer failed");
}
Checks-Effects-Interactions: update state BEFORE external calls to avoid reentrancy.

Writing the FE — provider vs signer

Connect wallet (MetaMask)
import { ethers } from "ethers";
const provider = new ethers.BrowserProvider(window.ethereum);
await provider.send("eth_requestAccounts", []);
const signer = await provider.getSigner();
window.ethereum is injected by MetaMask; guard for undefined (no wallet).
Two Contract instances: read vs write
import abi from "./abi/Crowdfund.json"; // artifact.abi array
const read  = new ethers.Contract(ADDR, abi, provider); // eth_call, free
const write = new ethers.Contract(ADDR, abi, signer);   // tx, costs gas
provider = read-only (works even disconnected); signer = writes (needs wallet).
Read state
const [raised, goal] = await read.getStatus();
Returns BigInt in ethers v6 — format with ethers.formatEther().
Write + await receipt
const tx = await write.contribute({ value: ethers.parseEther("0.1") });
const receipt = await tx.wait(); // resolves only once MINED
The send resolves with a tx hash immediately; state is stale until tx.wait().

React path — wagmi + viem

Read hook (auto-refetch, cached)
const { data, refetch } = useReadContract({
  address: ADDR, abi, functionName: "getStatus",
});
Write hook
const { writeContract, data: hash } = useWriteContract();
writeContract({ address: ADDR, abi,
  functionName: "contribute", value: parseEther("0.1") });
Wait for the tx in React
const { isLoading, isSuccess } = useWaitForTransactionReceipt({ hash });
Invalidate/refetch reads on isSuccess so the UI reflects new state.
Wallet connection state
const { address, isConnected, chainId } = useAccount();
wagmi wraps viem + handles connection lifecycle, caching, re-renders.

Add the ABI + address (the glue layer)

Address keyed by chainId
const CONTRACTS = {
  11155111: "0xSepolia...",
  1:        "0xMainnet...",
};
const ADDR = CONTRACTS[Number(chainId)];
Wrong chain -> wrong/nonexistent address. Pick address from the wallet's live chainId.
Import ABI straight from artifacts
import artifact from "../artifacts/contracts/Crowdfund.sol/Crowdfund.json";
const abi = artifact.abi;
Re-copy ABI on EVERY redeploy — a stale ABI silently breaks encode/decode.
Prompt a network switch
await window.ethereum.request({
  method: "wallet_switchEthereumChain",
  params: [{ chainId: "0xaa36a7" }], // Sepolia = 11155111
});
chainId in wallet RPC is hex; on-chain/JS values are decimal. Catch 4902 = chain not added.

Events & error handling

Live updates via events
read.on("Contributed", (from, amount) => refresh());
Use a WebSocket provider for push; still re-read on load — events can be missed on reorg/disconnect.
Query past logs
const logs = await read.queryFilter("Contributed", fromBlock, "latest");
Handle rejection & revert
try { const r = await (await write.contribute(...)).wait(); }
catch (e) {
  if (e.code === "ACTION_REJECTED") {/* user cancelled */}
  if (e.code === "CALL_EXCEPTION") {/* require() revert */}
}
User rejection, revert, out-of-gas, and pending state only surface in the real end-to-end loop.

Quick tips & gotchas

  • Three things wire FE to a contract: ADDRESS + ABI + correct chainId. Get one wrong and calls silently fail or hit the wrong contract.
  • Reads (view/pure) go through a provider via eth_call — free, no wallet needed. Writes need a signer, cost gas, and are async transactions.
  • A state-changing call resolves with a tx HASH, not the new value. Always await tx.wait() (or useWaitForTransactionReceipt) before showing success.
  • Re-copy the ABI on every redeploy and re-check the address — a stale ABI causes silent decode failures; a stale address calls the old contract.
  • Never trust the FE for validation: enforce every invariant on-chain with require/revert. An attacker crafts raw txs that bypass your UI entirely.
  • ethers v6 returns BigInt, not number — use parseEther/formatEther and BigInt math, never Number().
  • Handle the wrong-network case explicitly: read wallet chainId, prompt wallet_switchEthereumChain, and only enable actions once on the right chain.
  • ethers/viem = low-level (providers, signers, encoding); wagmi = React hooks on top. Use wagmi in React to kill connection/caching boilerplate.
