ETH on the frontend. — Cheat Sheet

Cohort 3.0 · 10 lessons · quick reference

Setup & Mental Model

Install the stack (Wagmi v2 needs Viem + TanStack Query as peers)
npm i wagmi viem @tanstack/react-query
The three layers, top to bottom
Viem = low-level TS Ethereum client (RPC, encoding, types). Wagmi = React hooks over Viem. TanStack Query = caching/dedup/refetch layer Wagmi reads/writes are built on.
"Hookifying Viem" = what Wagmi is
Every Wagmi read hook wraps a Viem publicClient action in a useQuery; every write wraps a walletClient action in a useMutation. That's why you get data/isLoading/error/refetch for free.
Wallet = an injected EIP-1193 provider
window.ethereum.request({ method: 'eth_requestAccounts' })
MetaMask/Rabby inject window.ethereum. Wagmi connectors abstract this; only reach for window.ethereum raw when debugging.

Config (chains + transports + connectors)

createConfig — the single source of truth
import { http, createConfig } from 'wagmi'
import { mainnet, sepolia } from 'wagmi/chains'
import { injected, metaMask, walletConnect } from 'wagmi/connectors'

export const config = createConfig({
  chains: [mainnet, sepolia],
  connectors: [
    injected(),
    walletConnect({ projectId: process.env.NEXT_PUBLIC_WC_ID! }),
  ],
  transports: {
    [mainnet.id]: http(), // public RPC
    [sepolia.id]: http('https://sepolia.infura.io/v3/KEY'),
  },
})
TS: register config globally for typed hooks
declare module 'wagmi' {
  interface Register { config: typeof config }
}
A chain must be in config.chains
Reads/writes against a chain not listed here throw. Add every network you target, incl. testnets.

Context / Provider wiring

Wagmi is React Context — wrap the app once
import { WagmiProvider } from 'wagmi'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import { config } from './wagmi'

const queryClient = new QueryClient()

export function Providers({ children }) {
  return (
    <WagmiProvider config={config}>
      <QueryClientProvider client={queryClient}>
        {children}
      </QueryClientProvider>
    </WagmiProvider>
  )
}
Order matters: WagmiProvider OUTSIDE QueryClientProvider
Wagmi hooks call useQuery internally — they need the QueryClient in scope. Reverse the nesting and every hook throws "No QueryClient set".
Next.js App Router
Provider file needs 'use client' at top; render <Providers> in root layout.

Connecting to the wallet

Connect / disconnect / read account
import { useAccount, useConnect, useDisconnect } from 'wagmi'

const { address, isConnected, chainId, status } = useAccount()
const { connectors, connect } = useConnect()
const { disconnect } = useDisconnect()

if (isConnected) return <button onClick={() => disconnect()}>{address}</button>
return connectors.map((c) => (
  <button key={c.uid} onClick={() => connect({ connector: c })}>{c.name}</button>
))
Use connector.uid as the React key
Not c.id — uids are unique per instance; ids can collide across injected wallets.
status values
'connecting' | 'reconnecting' | 'connected' | 'disconnected'. Prefer status/isConnected over checking address for undefined.
Switch network
import { useSwitchChain } from 'wagmi'
const { chains, switchChain } = useSwitchChain()
switchChain({ chainId: sepolia.id })

Reading state (balance + contracts)

Native ETH balance
import { useBalance } from 'wagmi'
const { data, isLoading } = useBalance({ address })
// data => { value: bigint, decimals: 18, symbol: 'ETH', formatted: '1.5' }
Format bigint for display
import { formatEther, formatUnits } from 'viem'
formatEther(data.value)            // '1.5'
formatUnits(raw, 6)                 // USDC (6 decimals)
ERC-20 balance via useReadContract
import { useReadContract } from 'wagmi'
import { erc20Abi } from 'viem'

const { data: bal } = useReadContract({
  address: TOKEN,
  abi: erc20Abi,
  functionName: 'balanceOf',
  args: [address!],
  query: { enabled: !!address },
})
Gate a query on inputs
useBalance({ address, query: { enabled: !!address, refetchInterval: 5000 } })
If address is undefined the query auto-disables. Use query.enabled for explicit control; query.refetchInterval replaces v1's watch: true.

Initiating a transaction

Send ETH
import { useSendTransaction } from 'wagmi'
import { parseEther } from 'viem'

const { sendTransaction, data: hash, isPending } = useSendTransaction()
sendTransaction({ to: '0xRecipient', value: parseEther('0.01') })
Call a contract (state-changing)
import { useWriteContract } from 'wagmi'

const { writeContract, data: hash } = useWriteContract()
writeContract({
  address: TOKEN,
  abi: erc20Abi,
  functionName: 'transfer',
  args: [to, parseEther('1')],
})
Wait for confirmation
import { useWaitForTransactionReceipt } from 'wagmi'
const { isLoading: confirming, isSuccess } = useWaitForTransactionReceipt({ hash })
hash is available immediately after submit (user signed); isSuccess flips once mined. Show a pending spinner between the two.
parseEther / parseUnits for inputs
parseEther('0.05')      // 50000000000000000n
parseUnits('100', 6)     // USDC amount as bigint
Never build value from a float. All amounts on-chain are bigint wei.

Dropping to Viem directly

Grab Wagmi's clients inside components
import { usePublicClient, useWalletClient } from 'wagmi'
const publicClient = usePublicClient()          // reads
const { data: walletClient } = useWalletClient() // signing
Standalone read client (no React)
import { createPublicClient, http } from 'viem'
import { mainnet } from 'viem/chains'

const client = createPublicClient({ chain: mainnet, transport: http() })
const bal = await client.getBalance({ address: '0x...' })
Standalone wallet client from the browser
import { createWalletClient, custom } from 'viem'
const wallet = createWalletClient({ chain: mainnet, transport: custom(window.ethereum) })
const [account] = await wallet.requestAddresses()
await wallet.sendTransaction({ account, to, value })
When to use raw Viem vs hooks
Hooks for anything rendered (they cache + re-render). Raw clients for scripts, event handlers, or one-off imperative calls outside the React tree.

Quick tips & gotchas

  • Everything on-chain is bigint (wei). Convert at the edges only: parseEther/parseUnits going in, formatEther/formatUnits coming out. Never do float math on balances.
  • WagmiProvider must wrap QueryClientProvider, not the other way around — Wagmi hooks depend on the TanStack QueryClient being in context.
  • Wagmi v2 dropped the `watch: true` prop. For live data use `query: { refetchInterval }` or watch blocks; and pass `query: { enabled }` to defer a read until inputs exist.
  • A submitted tx gives you a hash the moment the user signs — that is NOT confirmation. Pipe the hash into useWaitForTransactionReceipt and only treat isSuccess as done.
  • walletConnect() requires a projectId from cloud.walletconnect.com; injected()/metaMask() do not. Missing projectId silently breaks WC connections.
  • Key connector lists by connector.uid, not id — multiple injected wallets can share an id and cause React key collisions.
  • Reads/writes only work for chains listed in config.chains and with a matching entry in transports; forget the testnet there and hooks throw at runtime.
  • Prefer useAccount().status / isConnected over `if (address)` — during reconnect the address can be briefly undefined while the wallet is still connected.
