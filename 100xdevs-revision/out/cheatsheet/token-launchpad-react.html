Token Launchpad in React — Cheat Sheet

Cohort 3.0 · 10 lessons · quick reference

Core Concepts: Instructions vs Transactions

Instruction
Single operation aimed at one program: { programId, keys (accounts), data }. Does nothing on its own.
Transaction
Atomic bundle of 1+ instructions. All-or-nothing: if any instruction fails, the whole tx reverts.
Required tx fields
tx.feePayer = wallet.publicKey;
tx.recentBlockhash = (await connection.getLatestBlockhash()).blockhash;
Without feePayer + recentBlockhash the tx can't be signed/sent.
Signers
Wallet signs as feePayer via sendTransaction; any NEW account (e.g. mint Keypair) must partialSign the tx before sending.
Why bundle instructions
createAccount + initMint + initMetadata in ONE tx = mint is never left half-initialized.

Node Polyfill (Vite)

Install plugin
npm i -D vite-plugin-node-polyfills
Browser lacks Buffer/process/global that @solana/web3.js expects. Without this you get 'Buffer is not defined'.
vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { nodePolyfills } from 'vite-plugin-node-polyfills'

export default defineConfig({
  plugins: [react(), nodePolyfills()],
})
Manual alternative
Some setups just alias global->globalThis and inject window.Buffer = Buffer from 'buffer'; the plugin is cleaner.

Wallet Adapter Context (Providers)

Import UI styles once
import '@solana/wallet-adapter-react-ui/styles.css';
Skipping this leaves the wallet modal/button unstyled.
Wrap app in providers
import { ConnectionProvider, WalletProvider } from '@solana/wallet-adapter-react';
import { WalletModalProvider } from '@solana/wallet-adapter-react-ui';

<ConnectionProvider endpoint={'https://api.devnet.solana.com'}>
  <WalletProvider wallets={[]} autoConnect>
    <WalletModalProvider>
      <TokenLaunchpad />
    </WalletModalProvider>
  </WalletProvider>
</ConnectionProvider>
Order matters: Connection > Wallet > WalletModal. wallets={[]} works because standard wallets (Phantom etc.) auto-register.
Connect / disconnect buttons
import { WalletMultiButton, WalletDisconnectButton }
  from '@solana/wallet-adapter-react-ui';
<WalletMultiButton />
<WalletDisconnectButton />
Access connection + wallet
import { useConnection, useWallet } from '@solana/wallet-adapter-react';
const { connection } = useConnection();
const wallet = useWallet(); // wallet.publicKey, wallet.sendTransaction
Hooks only work inside the providers. Guard on wallet.publicKey before acting.

Inspecting createMint / Why go manual

Server-style helper (needs a Keypair)
import { createMint } from '@solana/spl-token';
const mint = await createMint(
  connection, payer /* Keypair */, mintAuthority, freezeAuthority, 9
);
createMint requires a payer Keypair to sign — browser wallets never expose a secret key, so this can't run client-side.
Client pattern instead
Build the raw instructions (createAccount + initMint) yourself and sign via wallet.sendTransaction. That's what the launchpad does.
Token program choice
import { TOKEN_2022_PROGRAM_ID } from '@solana/spl-token';
Use Token-2022 for built-in on-chain metadata (metadata pointer extension). Legacy TOKEN_PROGRAM_ID needs Metaplex for metadata.

Creating the Mint (Token-2022 + Metadata)

Imports
import { Keypair, SystemProgram, Transaction } from '@solana/web3.js';
import {
  TOKEN_2022_PROGRAM_ID, ExtensionType, getMintLen,
  createInitializeMetadataPointerInstruction,
  createInitializeMintInstruction, TYPE_SIZE, LENGTH_SIZE
} from '@solana/spl-token';
import { createInitializeInstruction, pack } from '@solana/spl-token-metadata';
Generate mint keypair + metadata
const mintKeypair = Keypair.generate();
const metadata = {
  mint: mintKeypair.publicKey,
  name: 'KIRA', symbol: 'KIR',
  uri: 'https://cdn.100xdevs.com/metadata.json',
  additionalMetadata: [],
};
Size + rent (mint space + metadata)
const mintLen = getMintLen([ExtensionType.MetadataPointer]);
const metadataLen = TYPE_SIZE + LENGTH_SIZE + pack(metadata).length;
const lamports = await connection
  .getMinimumBalanceForRentExemption(mintLen + metadataLen);
createAccount space is ONLY mintLen; but fund rent for mintLen + metadataLen (metadata realloc's the account later).
Build + partial-sign tx
const tx = new Transaction().add(
  SystemProgram.createAccount({
    fromPubkey: wallet.publicKey,
    newAccountPubkey: mintKeypair.publicKey,
    space: mintLen, lamports, programId: TOKEN_2022_PROGRAM_ID,
  }),
  createInitializeMetadataPointerInstruction(
    mintKeypair.publicKey, wallet.publicKey,
    mintKeypair.publicKey, TOKEN_2022_PROGRAM_ID),
  createInitializeMintInstruction(
    mintKeypair.publicKey, 9, wallet.publicKey, null,
    TOKEN_2022_PROGRAM_ID),
  createInitializeInstruction({
    programId: TOKEN_2022_PROGRAM_ID,
    mint: mintKeypair.publicKey, metadata: mintKeypair.publicKey,
    name: metadata.name, symbol: metadata.symbol, uri: metadata.uri,
    mintAuthority: wallet.publicKey, updateAuthority: wallet.publicKey,
  }),
);
tx.feePayer = wallet.publicKey;
tx.recentBlockhash = (await connection.getLatestBlockhash()).blockhash;
tx.partialSign(mintKeypair);
await wallet.sendTransaction(tx, connection);
Metadata pointer points the mint at ITSELF, so metadata: mintKeypair.publicKey. initMint must run before initMetadata.

Create ATA + Actually Mint Tokens

Derive Associated Token Account
import { getAssociatedTokenAddressSync } from '@solana/spl-token';
const ata = getAssociatedTokenAddressSync(
  mintKeypair.publicKey, wallet.publicKey, false, TOKEN_2022_PROGRAM_ID
);
Pass TOKEN_2022_PROGRAM_ID — omitting it derives the wrong (legacy) ATA address.
Create the ATA
import { createAssociatedTokenAccountInstruction } from '@solana/spl-token';
const tx2 = new Transaction().add(
  createAssociatedTokenAccountInstruction(
    wallet.publicKey, ata, wallet.publicKey,
    mintKeypair.publicKey, TOKEN_2022_PROGRAM_ID)
);
await wallet.sendTransaction(tx2, connection);
ATA is the account that actually holds a user's balance for this mint.
Mint supply into the ATA
import { createMintToInstruction } from '@solana/spl-token';
const tx3 = new Transaction().add(
  createMintToInstruction(
    mintKeypair.publicKey, ata, wallet.publicKey,
    1_000_000_000, [], TOKEN_2022_PROGRAM_ID)
);
await wallet.sendTransaction(tx3, connection);
Amount is in base units: 1 token at 9 decimals = 1e9. Signer/authority = wallet (the mint authority set above).

Quick tips & gotchas

  • Use devnet (https://api.devnet.solana.com) and airdrop SOL first — creating a mint + ATA costs rent + fees.
  • 'Buffer is not defined' / 'global is not defined' => the node-polyfills plugin isn't wired into vite.config.js.
  • Any freshly generated account (the mint Keypair) MUST partialSign the tx; the wallet only signs as feePayer.
  • Instruction order is load-bearing: createAccount -> initMetadataPointer -> initMint -> initMetadata. initMetadata fails if the mint isn't initialized yet.
  • Fund rent for mintLen + metadataLen but set createAccount space = mintLen only; Token-2022 reallocs when metadata is written.
  • Mint amounts are in base units — multiply human amount by 10^decimals or you'll mint 1e-9 of a token.
  • Match the program ID everywhere (mint, ATA derivation, mintTo). Mixing TOKEN_PROGRAM_ID with TOKEN_2022 gives 'incorrect program id' / wrong ATA.
  • Guard every handler on wallet.publicKey (and connected) so hooks don't throw before a wallet is connected.
