Contract Anatomy

Minimal contract skeleton
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

contract Counter {
    uint256 public count;              // auto getter count()

    function increment() external {
        count += 1;
    }
}
SPDX + pragma are compiler-required conventions. `public` state vars auto-generate a getter.
Constructor (runs once at deploy)
address public owner;
constructor(uint256 _start) {
    owner = msg.sender;
    count = _start;
}
No `function` keyword. Args passed at deploy time, baked into the deploy tx.
State vs local vs global
uint256 stored;                 // storage (persists on-chain, costs gas)
function f() external view {
    uint256 tmp = 1;            // memory/stack (transient)
    address who = msg.sender;   // global
}
storage writes are expensive (~20k gas cold). Minimize them.
Globals cheat
msg.sender   // caller address
msg.value    // wei sent (needs payable)
block.timestamp
block.number
address(this).balance
block.timestamp is miner-influenceable — don't use for strong randomness.

Types (and how they differ from JS)

Statically typed, no `let`/`var`
uint256 n = 5;      // unsigned int, no floats
int256  s = -5;
bool    ok = true;
address a = 0xAbC...;
bytes32 h;
string  name = "gm";
JS has one `number`; Solidity has sized ints (uint8..uint256 in steps of 8). uint = uint256.
NO floating point
uint256 price = 1e18;   // represent 1.0 as 1e18 wei
// use fixed-point manually; multiply before divide
Biggest JS-dev trap. Integer division truncates: 7/2 == 3.
Overflow reverts (since 0.8)
uint8 x = 255;
x += 1;              // reverts (was silent wrap pre-0.8)
unchecked { x += 1; } // opt out for gas
No NaN/Infinity like JS. Arithmetic errors revert the whole tx.
Reference types need data location
function g(uint256[] memory arr) external {}
mapping(address => uint256) public balances;
struct User { string name; uint256 age; }
memory / storage / calldata must be declared for arrays, structs, strings, bytes. calldata is cheapest for external fn args.
mapping — no keys, no length, no iteration
mapping(address => uint256) balances;
balances[msg.sender] += 100;
Unlike a JS object: can't enumerate keys or get .length. Missing key returns the zero value, not undefined.
Default values (no undefined/null)
// uint->0, bool->false, address->address(0), string->""
There is no null/undefined. Everything is zero-initialized.

Functions, Visibility & Modifiers

Visibility (required on every fn)
function a() public {}    // internal + external
function b() external {}  // only outside calls
function c() internal {}  // this + derived
function d() private {}   // this contract only
external calldata args are cheaper than public memory args.
State mutability
function get() external view returns (uint256) { return count; } // reads only
function add(uint a, uint b) external pure returns (uint) { return a+b; } // no state
function pay() external payable {}  // can receive ETH
view/pure calls are free when called off-chain (eth_call), no tx needed.
Named/multiple returns
function stats() external view returns (uint256 c, address o) {
    return (count, owner);
}
Custom errors + require (gas-cheap reverts)
error NotOwner(address caller);
function admin() external {
    if (msg.sender != owner) revert NotOwner(msg.sender);
    require(count > 0, "empty");   // older style
}
Custom errors (0.8.4+) are cheaper than require strings. revert rolls back ALL state changes in the tx.
Modifiers (reusable guards)
modifier onlyOwner() {
    require(msg.sender == owner, "!owner");
    _;                 // placeholder = fn body runs here
}
function setX(uint v) external onlyOwner { x = v; }
Events (off-chain logs, indexed for filtering)
event Transfer(address indexed from, address indexed to, uint256 amount);
function send(address to, uint256 amt) external {
    emit Transfer(msg.sender, to, amt);
}
Up to 3 `indexed` params are searchable/filterable by dapps. Cheaper than storage but NOT readable from other contracts.

ABI & Bytecode

What compile produces
// solc → two artifacts:
//  1. bytecode (.bin)  → EVM machine code deployed on-chain
//  2. ABI (.json)      → interface spec used by clients
Bytecode = what runs. ABI = how off-chain code knows how to talk to it. You need BOTH to interact.
ABI = JSON interface descriptor
[{"type":"function","name":"increment",
  "inputs":[],"outputs":[],"stateMutability":"nonpayable"},
 {"type":"function","name":"count",
  "inputs":[],"outputs":[{"type":"uint256"}],"stateMutability":"view"}]
Describes functions, args, return types, events. The EVM does NOT read this — clients do, to encode calldata.
Function selector (first 4 bytes of calldata)
// selector = bytes4(keccak256("increment()"))
cast sig "transfer(address,uint256)"   // → 0xa9059cbb
Calldata = selector (4 bytes) + ABI-encoded args (32-byte padded). This is how the EVM dispatches to the right function.
Encode / decode calldata
cast calldata "transfer(address,uint256)" 0xabc... 100
cast abi-decode "balanceOf(address)(uint256)" 0x...
In Solidity: abi.encodeWithSelector(sel, args), abi.decode(data, (uint256)).
Use ABI + address in ethers.js
import { Contract } from "ethers";
const c = new Contract(address, abi, signer);
await c.increment();
const n = await c.count();
The ABI tells ethers how to encode the call into bytecode calldata under the hood.
Interface = ABI in Solidity form
interface IERC20 {
    function transfer(address to, uint256 amt) external returns (bool);
}
IERC20(token).transfer(to, 100);
Lets one contract call another using just the signature — no source/bytecode needed.
Get on-chain bytecode
cast code 0xContractAddress --rpc-url $RPC
Empty (0x) code means it's an EOA/wallet, not a contract.

Solidity vs JavaScript — mindset shifts

Execution cost = gas
Every op costs gas paid in ETH. There is no free loop. JS runs free on your CPU; Solidity runs on thousands of nodes.
Deterministic & sandboxed
No fetch/HTTP, no fs, no Date.now() as truth, no Math.random(). All nodes must reach identical results. Use oracles (Chainlink) for external data.
Immutable after deploy
No hot-reload. Deployed bytecode is permanent. Bugs need a redeploy or upgrade proxy. `git push` mentality does not apply.
State is global & public
uint256 private secret;  // still readable via storage slots!
`private` hides from other contracts, NOT from the public blockchain. Never store secrets.
Sync only, no async/promises
uint256 n = other.getValue();   // external call returns synchronously
No await/Promise. But external calls can re-enter your contract — guard against reentrancy.
Whole-tx atomicity
A revert anywhere undoes everything in the tx (like a DB rollback). JS throws leave partial mutations.
== vs no === , no truthy coercion
if (x) {}   // ERROR: x must be bool
if (x != 0) {}
No loose/strict equality distinction, no truthy/falsy. Conditions must be strictly bool.
Compiled, not interpreted
solc → EVM bytecode ahead of time. Type errors caught at compile, not runtime like JS.

Quick tips & gotchas

  • Always pin the pragma to a tested range (^0.8.24) and include the SPDX line — the compiler warns without it.
  • You need BOTH the ABI and the contract address to call a deployed contract; the ABI encodes args into calldata, the address routes the tx.
  • Follow checks-effects-interactions: update state BEFORE making external calls to avoid reentrancy (the classic DAO hack).
  • Prefer custom errors (error Foo()) over require("string") — they cost less gas and deployment size.
  • Money is uint256 in wei; 1 ETH = 1e18 wei. There are no decimals/floats — scale by 1e18 and multiply before dividing.
  • `private`/`internal` only restrict contract-to-contract access; all state is publicly readable on-chain — never store secrets.
  • Since 0.8, arithmetic auto-reverts on overflow/underflow; only wrap in `unchecked{}` when you've proven it's safe for gas savings.
  • Use `calldata` for external function reference-type args and minimize `storage` writes (~20k gas each) — gas is the real constraint, not CPU.
