React — Cheat Sheet

Cohort 3.0 · 24 lessons · quick reference

Setup & Project Bootstrap

Create app with Vite (recommended, fast)
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev
Use react-ts template for TypeScript. Vite dev server has HMR out of the box.
Create React App (legacy, slower)
npx create-react-app my-app
npm start
CRA is deprecated; prefer Vite or a framework (Next.js) for new projects.
Entry point wiring
// main.jsx
import { createRoot } from 'react-dom/client';
import App from './App';

createRoot(document.getElementById('root')).render(<App />);
React 18+ uses createRoot, not the old ReactDOM.render.
StrictMode double-invokes in dev
createRoot(el).render(<StrictMode><App /></StrictMode>);
Renders/effects run twice in dev only to surface impure logic. Not in production.

Components, Props, Children, Fragments

Function component + destructured props
function Card({ title, onClick }) {
  return <div onClick={onClick}>{title}</div>;
}
// usage: <Card title="Hi" onClick={() => {}} />
Props are read-only. Component names MUST be capitalized or React treats them as HTML tags.
Default props via destructuring defaults
function Btn({ variant = 'primary', children }) { ... }
children — content between tags
function Panel({ children }) {
  return <div className="panel">{children}</div>;
}
// <Panel><h1>Hi</h1></Panel>
children is a special prop; enables composition/slot patterns.
Fragment — group without extra DOM node
return (
  <>
    <td>A</td>
    <td>B</td>
  </>
);
Use <React.Fragment key={id}> when you need a key in a list.
Spread + rest props
function Input({ label, ...rest }) {
  return <input {...rest} />;
}

State & Effects (useState, useEffect)

useState — local reactive state
const [count, setCount] = useState(0);
setCount(count + 1);
State updates are async & batched; the variable doesn't change until re-render.
Functional update (avoid stale state)
setCount(c => c + 1);
Use updater form when new state depends on previous, esp. in loops/handlers firing rapidly.
Never mutate — replace objects/arrays
setUser(u => ({ ...u, name: 'Al' }));
setList(l => [...l, item]);
Mutating in place won't trigger a re-render; React compares by reference.
Lazy initial state (expensive init)
const [v] = useState(() => heavyCompute());
Pass a function so it runs only on first render, not every render.
useEffect — side effects after render
useEffect(() => {
  const id = setInterval(tick, 1000);
  return () => clearInterval(id); // cleanup
}, []);
[] = run once on mount. [dep] = run when dep changes. No array = run every render.
Data fetching in effect
useEffect(() => {
  let active = true;
  fetch(`/api/${id}`).then(r => r.json())
    .then(d => active && setData(d));
  return () => { active = false; };
}, [id]);
Guard against race conditions / setting state after unmount.

Rendering: Conditional, Lists, Styles, Re-renders

Conditional rendering
{isLoading && <Spinner />}
{user ? <Profile /> : <Login />}
{count > 0 ? <List /> : null}
Beware {count && <X/>} — if count is 0, React renders the literal 0. Use count > 0.
Lists + keys
{items.map(item => (
  <li key={item.id}>{item.name}</li>
))}
key must be stable & unique among siblings. NEVER use array index if list reorders/filters.
Inline styling (object, camelCase)
<div style={{ backgroundColor: 'red', fontSize: 14 }} />
Values are strings or numbers (px assumed). Prefer className/CSS for anything reusable.
className, not class
<div className="card active" />
class is reserved in JS; htmlFor replaces the label 'for' attribute.
Tracking re-renders (debug)
console.log('Rendered:', Component.name);
// or React DevTools -> Profiler / 'Highlight updates'
A component re-renders when its state changes, its parent re-renders, or its context changes.
Prevent unnecessary re-renders
const Memoized = React.memo(Child);
const cb = useCallback(() => f(a), [a]);
const val = useMemo(() => calc(x), [x]);
memo skips re-render if props are shallow-equal. Pair with useCallback/useMemo for reference stability.

Class Components, Lifecycle, Error Boundaries

Class component (legacy)
class Counter extends React.Component {
  state = { n: 0 };
  inc = () => this.setState(s => ({ n: s.n + 1 }));
  render() { return <button onClick={this.inc}>{this.state.n}</button>; }
}
Prefer function components + hooks. Classes still needed for error boundaries.
Lifecycle -> hook mapping
componentDidMount    -> useEffect(fn, [])
componentDidUpdate   -> useEffect(fn, [deps])
componentWillUnmount -> return cleanup in useEffect
One useEffect can express mount+update+unmount together.
Error boundary (class only)
class ErrorBoundary extends React.Component {
  state = { hasError: false };
  static getDerivedStateFromError() { return { hasError: true }; }
  componentDidCatch(err, info) { log(err, info); }
  render() {
    return this.state.hasError ? <Fallback/> : this.props.children;
  }
}
Catches render-phase errors in children only. NOT event handlers, async, or SSR.

Routing, Layouts & SPA (react-router-dom v6+)

Install & set up router
npm i react-router-dom
SPA = one HTML page; router swaps components client-side without full reloads.
Route config
<BrowserRouter>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/users/:id" element={<User />} />
    <Route path="*" element={<NotFound />} />
  </Routes>
</BrowserRouter>
Navigation (no page reload)
import { Link, useNavigate } from 'react-router-dom';
<Link to="/about">About</Link>
const nav = useNavigate(); nav('/login');
Use <Link>/useNavigate, NOT <a href> — <a> triggers a full reload and loses state.
Read route params / search
const { id } = useParams();
const [params] = useSearchParams();
Layout via nested routes + Outlet
<Route element={<Layout />}>
  <Route path="/" element={<Home />} />
  <Route path="/settings" element={<Settings />} />
</Route>
// Layout: <nav/> + <Outlet /> renders the matched child
Shared chrome (navbar/sidebar) stays mounted while <Outlet/> swaps the page.
Lazy-load routes
const About = lazy(() => import('./About'));
<Suspense fallback={<Loading/>}><About/></Suspense>
Code-splitting keeps initial bundle small.

useRef & Custom Hooks

useRef — DOM access
const inputRef = useRef(null);
<input ref={inputRef} />;
inputRef.current.focus();
useRef — mutable value that survives renders
const renders = useRef(0);
renders.current++;
Changing ref.current does NOT trigger a re-render (unlike state).
Custom hook — extract reusable stateful logic
function useToggle(init = false) {
  const [on, setOn] = useState(init);
  const toggle = useCallback(() => setOn(v => !v), []);
  return [on, toggle];
}
Must start with 'use' and may call other hooks. Shares logic, not state — each call gets its own state.
Rules of Hooks
// ✅ top level of component/hook
// ❌ not inside if/for/nested fns
Call hooks in the same order every render so React can track them.

State Sharing: Lift, Prop Drilling, Context, Recoil

Lift state up (rolling up state)
// Move shared state to closest common parent,
// pass value + setter down as props.
Lifting can cause a big subtree to re-render — split components / memoize to limit blast radius.
Prop drilling (the problem)
<A user={user}>
  <B user={user}>
    <C user={user} /> // passed through B just to reach C
Passing props through intermediate components that don't use them → Context/state lib.
Context API — avoid drilling
const UserCtx = createContext(null);

<UserCtx.Provider value={user}>
  <App />
</UserCtx.Provider>

const user = useContext(UserCtx);
Every consumer re-renders when the Provider value changes. Memoize the value object.
Context re-render gotcha (testing/perf)
const value = useMemo(() => ({ user, setUser }), [user]);
<Ctx.Provider value={value}>
New object literal each render = all consumers re-render even if data is unchanged.
Recoil — atom-level state (granular re-renders)
npm i recoil

const countAtom = atom({ key: 'count', default: 0 });

// wrap app: <RecoilRoot><App/></RecoilRoot>
const [c, setC] = useRecoilState(countAtom);
const v = useRecoilValue(countAtom);   // read only
const set = useSetRecoilState(countAtom); // write only
Only components subscribed to a specific atom re-render — fixes Context's over-rendering. selector() for derived state.

Quick tips & gotchas

  • State updates are async and batched — read the new value after re-render or use the functional updater setX(prev => ...).
  • Never mutate state/props directly; always create a new object/array so React's reference comparison detects the change.
  • {value && <Comp/>} renders a literal 0 when value is 0 — use value > 0 && ... or a ternary.
  • useEffect with a missing/wrong dependency array is the #1 bug source: [] runs once, [deps] on change, no array every render. Return a cleanup fn for subscriptions/timers.
  • List keys must be stable and unique; using the array index breaks when items are reordered, inserted, or filtered.
  • Context Provider re-renders ALL consumers when its value changes — wrap the value in useMemo; reach for Recoil/Zustand when re-render granularity matters.
  • Error boundaries only catch errors during rendering of children — not in event handlers, async code, or SSR (wrap those in try/catch).
  • Use <Link>/useNavigate for SPA navigation; a plain <a href> forces a full page reload and wipes React state.
