GRPC — Cheat Sheet

Cohort 2.0 · 8 lessons · quick reference

RPC fundamentals

RPC = call a function that runs on another machine
Client invokes a local stub; args are serialized, sent over the network, executed server-side, result returned. Hides transport so a remote call looks like a local one.
Dumb RPC over plain HTTP/JSON
// server: one endpoint = one 'procedure'
app.post('/rpc', (req, res) => {
  const { method, params } = req.body;
  if (method === 'sum') res.json({ result: params.a + params.b });
});
// client
fetch('/rpc', { method:'POST',
  body: JSON.stringify({ method:'sum', params:{a:1,b:2} }) });
Contrast with REST: RPC is action/verb-centric (/getUser), REST is resource/noun-centric (GET /users/1). The 'dumb' version is just JSON-over-HTTP with a method field.
Common RPC protocols
JSON-RPC (JSON over HTTP/WS, human-readable), XML-RPC (legacy, verbose), SOAP (XML + WSDL contract, enterprise), gRPC (Protobuf over HTTP/2, binary, fastest + typed contract). Choose gRPC for internal high-throughput service-to-service.

Protocol Buffers (.proto)

proto3 file skeleton
syntax = "proto3";
package todo;

message Todo {
  int32 id = 1;
  string title = 2;
  bool done = 3;
}
Field numbers (=1,=2) are the wire identity — NEVER change/reuse them once shipped. Names can change; numbers cannot.
Scalar types
double float
int32 int64 uint32 uint64
sint32 sint64   // zigzag, better for negatives
fixed32 fixed64 sfixed32 sfixed64
bool string bytes
proto3 defaults: numbers=0, string="", bool=false. Unset vs default is indistinguishable unless you wrap or use 'optional'.
Repeated (arrays) & maps
repeated string tags = 4;
map<string, int32> scores = 5;
repeated = list; map keys must be integral or string. Maps are unordered on the wire.
Nested messages, enums, optional
enum Status { UNKNOWN = 0; ACTIVE = 1; }
message User {
  message Address { string city = 1; }
  Address addr = 1;
  optional string nickname = 2;   // presence tracking
}
First enum value MUST be 0 (the default). 'optional' brings back has-field presence in proto3.
oneof (tagged union)
message Event {
  oneof payload { string msg = 1; int32 code = 2; }
}
Setting one field clears the others. Good for variant types.

Defining a gRPC service

service + rpc definitions
service TodoService {
  rpc CreateTodo (Todo) returns (Todo);
  rpc GetTodos   (Empty) returns (TodoList);
}
message Empty {}
message TodoList { repeated Todo todos = 1; }
Every rpc takes exactly one request message and returns one response message — wrap primitives in a message.
Four call types
rpc Unary(Req) returns (Res);              // 1<->1
rpc ServerStream(Req) returns (stream Res);// 1->N
rpc ClientStream(stream Req) returns (Res);// N->1
rpc BiDi(stream Req) returns (stream Res); // N<->N
'stream' keyword on either side. Streaming rides HTTP/2 multiplexing — no new connections per message.

Node.js: @grpc/grpc-js setup

Install
npm i @grpc/grpc-js @grpc/proto-loader
npm i -D grpc-tools ts-proto   # for codegen / TS types
grpc-js is pure JS (no native addon). proto-loader parses .proto at runtime — no build step needed.
Load proto at runtime
import * as grpc from '@grpc/grpc-js';
import * as protoLoader from '@grpc/proto-loader';
const def = protoLoader.loadSync('todo.proto', {
  keepCase: true, longs: String, enums: String, defaults: true, oneofs: true,
});
const proto = grpc.loadPackageDefinition(def) as any;
keepCase:true keeps proto field names as-is; otherwise they're camelCased in JS.
Implement + start server
const server = new grpc.Server();
server.addService(proto.todo.TodoService.service, {
  CreateTodo: (call, cb) => cb(null, { ...call.request, id: 1 }),
  GetTodos:   (call, cb) => cb(null, { todos: [] }),
});
server.bindAsync('0.0.0.0:50051',
  grpc.ServerCredentials.createInsecure(),
  () => server.start());
Handler signature: (call, callback). call.request holds the deserialized message. bindAsync replaces the deprecated bind+start.
Client (unary)
const client = new proto.todo.TodoService(
  'localhost:50051', grpc.credentials.createInsecure());
client.CreateTodo({ title: 'ship', done: false }, (err, res) => {
  console.log(res.id);
});
Callback-style API. Wrap in util.promisify or a Promise for async/await.
Server-streaming handler
GetTodos: (call) => {
  call.write({ id: 1, title: 'a' });
  call.write({ id: 2, title: 'b' });
  call.end();
}
// client
const stream = client.GetTodos({});
stream.on('data', t => console.log(t));
stream.on('end', () => {});
Streaming handlers get NO callback — use call.write/end and listen to 'data'/'end'/'error' events.

Adding types (TypeScript)

Generate TS types with ts-proto
protoc \
  --plugin=./node_modules/.bin/protoc-gen-ts_proto \
  --ts_proto_out=./src/gen \
  --ts_proto_opt=outputServices=grpc-js,esModuleInterop=true \
  ./proto/todo.proto
Generates typed message interfaces + typed service definitions so call.request and responses are checked.
Type the handler with generated types
import { TodoServiceServer, Todo } from './gen/todo';
const impl: TodoServiceServer = {
  createTodo: async (call): Promise<Todo> => ({ ...call.request, id: 1 }),
};
Runtime proto-loader gives you `any`; codegen (ts-proto / grpc-tools) is what gives real compile-time safety.
grpc.web / codegen for browsers
protoc --js_out=import_style=commonjs \
  --grpc-web_out=import_style=typescript,mode=grpcwebtext:./out todo.proto
Browsers can't speak raw HTTP/2 gRPC — you need grpc-web + an Envoy proxy or a compatible gateway.

Errors, metadata & deadlines

Return a gRPC status error
cb({ code: grpc.status.NOT_FOUND, message: 'todo missing' });
Use canonical codes (OK, INVALID_ARGUMENT, NOT_FOUND, UNAUTHENTICATED, DEADLINE_EXCEEDED...) — not HTTP codes.
Metadata (headers) + deadline
const md = new grpc.Metadata();
md.set('authorization', 'Bearer x');
client.GetTodos({}, md, { deadline: Date.now() + 2000 }, cb);
Deadlines are absolute timestamps and propagate across hops. Always set one to avoid hung calls.

Quick tips & gotchas

  • Field numbers are forever: reserve removed ones with `reserved 3, 5;` and `reserved "oldName";` so nobody reuses the tag and corrupts data.
  • proto3 can't tell 'unset' from 'default 0/""/false' — use `optional` for presence, or a wrapper/oneof when zero is a meaningful value.
  • gRPC needs HTTP/2 end-to-end; browsers can't do it directly — use grpc-web behind Envoy, or a REST gateway (grpc-gateway).
  • proto-loader = zero-build but untyped (`any`); ts-proto / protoc codegen = real TypeScript types. Pick codegen for anything non-trivial.
  • Streaming handlers have no callback: drive them with call.write()/call.end() and handle 'data'/'end'/'error' events — mixing the two styles is a common bug.
  • Always set a deadline on client calls; without one a stuck server hangs the client indefinitely. Deadlines propagate to downstream services automatically.
  • Use ServerCredentials.createInsecure() only in dev — production needs TLS via createSsl(); mismatched insecure/secure creds fail with cryptic UNAVAILABLE errors.
  • Enum's first value must be 0 and acts as the default/UNKNOWN sentinel; don't assign business meaning to the zero value.
