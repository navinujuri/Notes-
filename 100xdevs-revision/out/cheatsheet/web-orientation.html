Web Dev + Devops Orientation — Cheat Sheet

Cohort 2.0 · 8 lessons · quick reference

Orientation & Study Strategy

Avoid the spikes — consistency > cramming
Don't binge 8h then vanish for a week. 1-2 focused hours daily beats weekend spikes; the dropout risk is the gap, not the pace.
"What would I do if I was you?" — build, don't just watch
Passive video-watching ≈ 0 retention. Pause, type every snippet yourself, then rebuild it from scratch without the video.
Select your level — Beginner vs Advanced/DevOps track
Beginner: start HTML/CSS/JS fundamentals. If you already know JS + React, skip ahead and start at the backend/DevOps level. Don't sit through material you know.
Good things to hash out (before Day 1)
# accounts to create upfront
- GitHub (auth, portfolio, deploys)
- Vercel / Netlify (free frontend hosting)
- Discord (cohort community + doubt support)
Get the boring setup out of the way so nothing blocks you mid-lesson.
Take notes + ship projects publicly
Push every exercise to GitHub. A green contribution graph + deployed links is your interview proof.

HTML — Boilerplate & Document Structure

HTML5 skeleton
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Page</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <!-- content -->
  <script src="script.js"></script>
</body>
</html>
viewport meta is mandatory for responsive/mobile. Put <script> at end of body (or use defer).
Semantic layout elements
<header>, <nav>, <main>, <section>,
<article>, <aside>, <footer>
Use these over generic <div> — better a11y + SEO. One <main> per page.
Headings & text
<h1>..<h6>  <p>  <strong>  <em>
<br>  <hr>  <span>
Exactly one <h1> per page; don't skip heading levels for styling.
Links & images
<a href="https://x.com" target="_blank" rel="noopener">Link</a>
<img src="pic.jpg" alt="describe it" width="300">
alt is required for a11y. target=_blank needs rel=noopener for security.
Lists
<ul><li>item</li></ul>      <!-- unordered -->
<ol><li>step</li></ol>      <!-- ordered -->

HTML — Forms & Inputs

Form skeleton
<form action="/submit" method="POST">
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>
  <button type="submit">Send</button>
</form>
name is the key sent to the server. Match label for= to input id= for a11y + click-to-focus.
Input types
type="text | email | password | number"
type="checkbox | radio | date | file"
<textarea></textarea>
<select><option value="1">One</option></select>
Radios sharing the same name= act as one group.
Validation attributes
required  minlength="3"  maxlength="20"
min="0"  max="100"  pattern="[0-9]+"
placeholder="hint"

CSS — Selectors & Box Model

Attach CSS
<link rel="stylesheet" href="style.css">  <!-- external, preferred -->
<style> p { color: red; } </style>       <!-- internal -->
<p style="color:red">                    <!-- inline, avoid -->
Selectors
*            /* all */
tag          /* p, div */
.class       /* .btn */
#id          /* #nav */
.a .b        /* descendant */
.a > .b      /* direct child */
a:hover      /* pseudo-class */
input:focus
Specificity (who wins)
inline > #id > .class > tag > *
!important overrides all — avoid it; it's a debugging trap. More specific selector wins ties by source order (last one applies).
Box model — always reset
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
border-box makes width include padding+border — sane sizing. Default content-box adds them on top.
Units
px    /* fixed */
rem   /* relative to root font-size (16px) */
em    /* relative to parent font-size */
%     /* relative to parent */
vw/vh /* viewport width/height */
Use rem for font-size/spacing so it scales; px for borders.

CSS — Flexbox (1D layout)

Enable flex on the container
.container {
  display: flex;
  flex-direction: row;   /* or column */
  gap: 16px;
}
Align along main + cross axis
.container {
  justify-content: center;   /* main axis: start|center|space-between|space-around */
  align-items: center;       /* cross axis: start|center|stretch */
}
justify-content follows flex-direction. Swap to column and the axes flip.
Common centering (both axes)
display: flex;
justify-content: center;
align-items: center;
The classic "center a div" — 3 lines.
Item growth & wrapping
.item { flex: 1; }        /* grow to fill equally */
.container { flex-wrap: wrap; }

CSS — Grid, Positioning & Responsive

Grid (2D layout)
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
1fr = 1 fraction of free space. Use grid for 2D (rows+cols), flex for 1D.
Auto-responsive grid (no media query)
grid-template-columns:
  repeat(auto-fit, minmax(200px, 1fr));
Cards auto-reflow: as many 200px+ columns as fit.
position
position: relative;  /* anchor for children */
position: absolute;  /* vs nearest positioned ancestor */
position: fixed;     /* vs viewport (sticky navbar) */
position: sticky; top: 0;
absolute needs a relative ancestor or it anchors to <body>.
Media query (mobile-friendly)
@media (max-width: 768px) {
  .container { flex-direction: column; }
}
Prefer mobile-first with min-width. Requires the viewport meta tag in HTML.
Colors & background
color: #333;
background: #f5f5f5;
color: rgb(255 0 0 / 0.5);   /* alpha */

Quick tips & gotchas

  • Consistency beats intensity: short daily sessions avoid the 'spike then quit' trap that kills most cohort learners.
  • Type every snippet by hand and rebuild from scratch — watching lessons without coding retains almost nothing.
  • Skip to your real level. If you know HTML/CSS/JS already, don't re-watch fundamentals; start where it's new.
  • Always add box-sizing: border-box on * and reset margin/padding — the #1 source of unexpected layout width.
  • Set the <meta name="viewport"> tag or media queries and responsive layouts silently do nothing on mobile.
  • Flexbox = one dimension (row OR column), Grid = two dimensions (rows AND cols). Pick by the problem, not habit.
  • Never fight specificity with !important — write a more specific selector instead; !important compounds into unmaintainable CSS.
  • Use Live Server (auto-reload) + Prettier format-on-save from day one so you focus on code, not manual refresh/formatting.
