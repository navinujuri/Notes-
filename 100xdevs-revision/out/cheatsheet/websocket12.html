Websockets — Cheat Sheet

Cohort 3.0 · 5 lessons · quick reference

Protocol & Concepts

What it is
Full-duplex, persistent TCP connection over a single socket. Bidirectional — server can push without client polling.
Handshake = HTTP Upgrade
GET /chat HTTP/1.1
Host: server.example.com
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
Sec-WebSocket-Version: 13

// 101 Switching Protocols -> connection becomes a ws
Starts as HTTP, then switches protocols. After 101, it's no longer HTTP.
Schemes
ws://  -> plaintext (port 80)
wss:// -> TLS       (port 443)
Use wss:// in production. Browsers block ws:// from https:// pages (mixed content).
vs HTTP polling / SSE
Polling = repeated requests (wasteful). SSE = server->client only, text only, auto-reconnect. WS = both directions, binary + text.
Frame types
text (UTF-8), binary, ping, pong, close. Ping/pong are control frames for keepalive.

Node.js Server Setup (ws)

Install
npm install ws          # server lib
npm i -D @types/ws      # TS types
Standalone server
const { WebSocketServer } = require('ws');
const wss = new WebSocketServer({ port: 8080 });

wss.on('connection', (ws, req) => {
  ws.send('welcome');
  ws.on('message', (data, isBinary) => {
    ws.send(isBinary ? data : `echo: ${data}`);
  });
  ws.on('close', (code, reason) => {});
  ws.on('error', console.error);
});
Attach to existing HTTP/Express server
const http = require('http');
const server = http.createServer(app);   // app = express()
const wss = new WebSocketServer({ server });
server.listen(3000);
Lets HTTP routes and WS share one port. wss listens for upgrade events automatically.
noServer mode — auth on upgrade
const wss = new WebSocketServer({ noServer: true });
server.on('upgrade', (req, socket, head) => {
  const token = new URL(req.url, 'http://x').searchParams.get('token');
  if (!verify(token)) return socket.destroy();   // reject before handshake
  wss.handleUpgrade(req, socket, head, (ws) => {
    wss.emit('connection', ws, req);
  });
});
The ONLY reliable place to authenticate — browsers can't set custom headers on WebSocket(). Reject via socket.destroy().

Server Patterns

Broadcast to all clients
import { WebSocket } from 'ws';
function broadcast(msg) {
  wss.clients.forEach((c) => {
    if (c.readyState === WebSocket.OPEN) c.send(msg);
  });
}
Always guard on readyState === OPEN (1); a closing socket throws on send.
Rooms / channels (no built-in support)
const rooms = new Map();  // roomId -> Set<ws>
function join(ws, id) {
  if (!rooms.has(id)) rooms.set(id, new Set());
  rooms.get(id).add(ws);
  ws.on('close', () => rooms.get(id)?.delete(ws));
}
function toRoom(id, msg) {
  rooms.get(id)?.forEach((c) =>
    c.readyState === 1 && c.send(msg));
}
ws has no rooms — you track membership yourself. socket.io gives rooms out of the box.
Message framing (JSON convention)
ws.on('message', (raw) => {
  const { type, payload } = JSON.parse(raw);
  switch (type) {
    case 'chat': handleChat(payload); break;
    case 'join': join(ws, payload.room); break;
  }
});
ws.send(JSON.stringify({ type: 'ack', payload: {} }));
WS gives you a byte stream, not RPC. Define your own {type,payload} envelope. Wrap JSON.parse in try/catch.
Heartbeat — detect dead connections
function heartbeat() { this.isAlive = true; }
wss.on('connection', (ws) => {
  ws.isAlive = true;
  ws.on('pong', heartbeat);
});
const iv = setInterval(() => {
  wss.clients.forEach((ws) => {
    if (!ws.isAlive) return ws.terminate(); // no pong -> kill
    ws.isAlive = false;
    ws.ping();
  });
}, 30000);
wss.on('close', () => clearInterval(iv));
TCP won't tell you a client vanished (dead router, closed laptop). Ping/pong is the only reliable liveness check. terminate() is immediate; close() is graceful.
Backpressure check
if (ws.bufferedAmount < 1e6) ws.send(bigChunk);
bufferedAmount = bytes queued but not yet sent. Growing = slow client; drop or throttle.

Client-side (Browser)

Connect & handle events
const ws = new WebSocket('wss://api.example.com/chat?token=abc');
ws.onopen  = () => ws.send(JSON.stringify({ type: 'join' }));
ws.onmessage = (e) => console.log(JSON.parse(e.data));
ws.onerror = (e) => console.error('ws error', e);
ws.onclose = (e) => console.log(e.code, e.reason, e.wasClean);
No custom headers allowed on the constructor — pass auth via query string or subprotocol, then validate on upgrade.
readyState constants
WebSocket.CONNECTING // 0
WebSocket.OPEN       // 1  <- only state where send() is safe
WebSocket.CLOSING    // 2
WebSocket.CLOSED     // 3
if (ws.readyState === WebSocket.OPEN) ws.send(data);
Send / receive binary
ws.binaryType = 'arraybuffer';   // default 'blob'
ws.send(new Uint8Array([1,2,3]).buffer);
ws.send(JSON.stringify({ a: 1 }));
send() accepts string | ArrayBuffer | Blob | TypedArray.
Close cleanly
ws.close(1000, 'user left');  // 1000 = normal
Codes: 1000 normal, 1001 going away, 1006 abnormal (no close frame), 1011 server error. Custom app codes: 4000-4999.
Subprotocol negotiation
const ws = new WebSocket(url, ['graphql-ws', 'json']);
// server: new WebSocketServer({ handleProtocols: () => 'graphql-ws' })
ws.protocol; // chosen one

Resilience (client reconnect)

Auto-reconnect with exponential backoff
let retries = 0;
function connect() {
  const ws = new WebSocket(URL);
  ws.onopen = () => { retries = 0; };
  ws.onclose = () => {
    const delay = Math.min(1000 * 2 ** retries++, 30000);
    setTimeout(connect, delay + Math.random() * 1000); // + jitter
  };
}
connect();
Browsers do NOT auto-reconnect WebSockets (unlike EventSource/SSE). Add jitter to avoid thundering-herd reconnects after a server restart.
App-level heartbeat from client
setInterval(() => ws.readyState === 1 &&
  ws.send(JSON.stringify({ type: 'ping' })), 25000);
Proxies/LBs kill idle connections (~60s). Keep traffic flowing under the idle timeout.
Resume state after reconnect
Reconnection = fresh connection. Re-auth, re-join rooms, and request a delta since lastSeenId (server should support message replay/cursor).

Scaling WS Servers

Core problem: connections are stateful
Each socket is pinned to ONE process. Client on server A can't be reached by server B — unlike stateless HTTP behind a round-robin LB.
Sticky sessions at the load balancer
# nginx: hash client to a fixed backend
upstream ws { ip_hash; server a:8080; server b:8080; }
Keeps a client pinned to its process. Needed if any per-process in-memory state exists. L4 (TCP) LBs also work well for WS.
nginx proxy — required upgrade headers
location /ws {
  proxy_pass http://ws;
  proxy_http_version 1.1;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection "upgrade";
  proxy_read_timeout 3600s;   # else idle conns dropped
}
Without Upgrade/Connection headers the handshake fails (101 never happens).
Pub/Sub backplane for cross-node broadcast
const sub = redis.duplicate(); await sub.subscribe('chat');
sub.on('message', (ch, msg) => broadcastLocal(msg));
// on receive from a client:
redis.publish('chat', msg);   // fan-out to ALL server instances
The key scaling pattern: each node publishes to Redis; every node's subscriber pushes to ITS local clients. Now server A can reach clients on server B.
socket.io + Redis adapter
const { createAdapter } = require('@socket.io/redis-adapter');
io.adapter(createAdapter(pubClient, subClient));
io.to('room1').emit('msg', data); // works across all nodes
socket.io handles rooms, reconnection, fallbacks, and cross-node fan-out for you. Trade-off: heavier + custom protocol (not raw WS).
Capacity & tuning
# raise open file descriptor limit (each conn = 1 fd)
ulimit -n 100000
Bottleneck is memory/fds per connection, not CPU. ~tens of thousands of idle conns per node. Offload TLS to the LB; scale horizontally + backplane, not vertically.

Quick tips & gotchas

  • send() throws if readyState !== OPEN. Always guard with `ws.readyState === WebSocket.OPEN` (=== 1) before sending.
  • Authenticate during the HTTP upgrade (noServer mode / server 'upgrade' event) — browsers can't set custom headers, so pass tokens via query string or subprotocol and validate BEFORE handshake.
  • Browsers never auto-reconnect a WebSocket. You must implement reconnect with exponential backoff + jitter yourself.
  • TCP can't detect a silently-dead peer. Implement ping/pong heartbeat + terminate() on missed pong, or you leak zombie connections.
  • Proxies/LBs drop idle connections (~60s default). Send app-level or protocol pings under that timeout, and raise proxy_read_timeout.
  • Horizontal scaling needs a Redis (or similar) pub/sub backplane — connections are pinned to one process, so a plain round-robin LB can't route pushes across nodes.
  • Use wss:// (TLS) always in prod; ws:// from an https page is blocked as mixed content. Terminate TLS at the LB to save app-server CPU.
  • WS gives you raw bytes, not RPC — define a {type, payload} JSON envelope and wrap JSON.parse in try/catch to survive malformed frames.
