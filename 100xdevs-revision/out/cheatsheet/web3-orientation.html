Setup & Tooling (Node.js)

Built-in crypto module (no install)
const crypto = require('crypto');
// or ESM: import crypto from 'node:crypto';
Elliptic-curve keys/signatures (secp256k1, Bitcoin's curve)
npm i elliptic
const EC = require('elliptic').ec;
const ec = new EC('secp256k1');
secp256k1 is the exact curve Bitcoin & Ethereum use.
Quick REPL to experiment with hashes
node -e "console.log(require('crypto').createHash('sha256').update('hi').digest('hex'))"

Hashing (SHA-256)

Hash any string to hex
const hash = crypto.createHash('sha256')
  .update('hello world')
  .digest('hex');
// b94d27b9934d3e08a52e52d7da7dabfac484efe37a5380ee9088f7ace2efcde9
digest() empties the hash object — call it once per createHash().
Hash structured data deterministically
const h = crypto.createHash('sha256')
  .update(JSON.stringify(obj))
  .digest('hex');
Same input -> same 64-char (256-bit) hex, always. Key ordering must be stable.
Core properties to memorize
Deterministic, one-way (irreversible), avalanche (1-bit change -> totally different hash), fixed-length output, collision-resistant.
Hex vs raw length
// 256 bits = 32 bytes = 64 hex chars
crypto.createHash('sha256').update('').digest('hex').length // 64
SHA-256 output is always 64 hex chars regardless of input size.

Proof of Work (mining)

Find a nonce so hash starts with N zeros
function mine(input, difficulty) {
  const prefix = '0'.repeat(difficulty);
  let nonce = 0;
  while (true) {
    const hash = crypto.createHash('sha256')
      .update(input + nonce).digest('hex');
    if (hash.startsWith(prefix)) return { nonce, hash };
    nonce++;
  }
}
The only way to find the nonce is brute-force guessing — that's the 'work'.
Difficulty = number of leading zeros required
Each extra leading zero makes it ~16x harder (hex). Verifying is instant; finding is expensive — asymmetric by design.
Verify PoW (cheap)
const ok = crypto.createHash('sha256')
  .update(input + nonce).digest('hex')
  .startsWith('0000');

Build a Blockchain / New Currency

Block structure
class Block {
  constructor(index, data, prevHash = '') {
    this.index = index;
    this.timestamp = Date.now();
    this.data = data;        // transactions
    this.prevHash = prevHash;
    this.nonce = 0;
    this.hash = this.computeHash();
  }
  computeHash() {
    return crypto.createHash('sha256')
      .update(this.index + this.timestamp +
        JSON.stringify(this.data) + this.prevHash + this.nonce)
      .digest('hex');
  }
}
prevHash is what chains blocks — tampering with block N breaks every block after it.
Mine a block (embed PoW)
mineBlock(difficulty) {
  const prefix = '0'.repeat(difficulty);
  while (!this.hash.startsWith(prefix)) {
    this.nonce++;
    this.hash = this.computeHash();
  }
}
Chain + genesis block + validation
class Chain {
  constructor() { this.blocks = [new Block(0, 'genesis')]; }
  add(data, difficulty = 4) {
    const prev = this.blocks[this.blocks.length - 1];
    const b = new Block(this.blocks.length, data, prev.hash);
    b.mineBlock(difficulty);
    this.blocks.push(b);
  }
  isValid() {
    for (let i = 1; i < this.blocks.length; i++) {
      const cur = this.blocks[i], prev = this.blocks[i - 1];
      if (cur.hash !== cur.computeHash()) return false;
      if (cur.prevHash !== prev.hash) return false;
    }
    return true;
  }
}
A 'currency' is just a chain whose data is a ledger of signed transactions.
Transaction shape
const tx = { from: senderPubKey, to: recipientPubKey, amount: 10 };
'from' = null for coinbase/mining reward (money creation).

Digital Signatures & Keys

Generate a keypair (your wallet)
const key = ec.genKeyPair();
const publicKey  = key.getPublic('hex'); // your address
const privateKey = key.getPrivate('hex'); // NEVER share
Public key = who you are; private key = proof you're you. Ownership = knowing the private key.
Sign a transaction
const txHash = crypto.createHash('sha256')
  .update(JSON.stringify(tx)).digest('hex');
const sig = key.sign(txHash).toDER('hex');
Anyone can verify with only the public key
const pub = ec.keyFromPublic(publicKey, 'hex');
const valid = pub.verify(txHash, sig); // true/false
Signature proves the sender authorized it without revealing the private key. Can't be forged, can't be repudiated.

Bitcoin Essentials

Double SHA-256 (Bitcoin's actual hash)
const dsha = (buf) => crypto.createHash('sha256')
  .update(crypto.createHash('sha256').update(buf).digest())
  .digest('hex');
Bitcoin hashes block headers with SHA256(SHA256(x)).
Hard-limit supply & issuance
21M BTC cap. Reward halves every 210,000 blocks (~4 yrs): 50 -> 25 -> 12.5 -> 6.25 -> 3.125 BTC (2024).
Timing & difficulty
~10 min target block time. Difficulty re-adjusts every 2016 blocks (~2 weeks) to hold that 10-min average.
UTXO model (not account balances)
You spend whole 'unspent transaction outputs' and get change back; balance = sum of your UTXOs. Ethereum uses accounts instead.
Units
1 BTC = 100,000,000 satoshis (1e-8). Smallest unit = 1 sat.
Merkle root
All txs in a block hashed pairwise into one root hash stored in the header — lets you prove a tx is included without the full block.
Coinbase transaction
First tx in a block; has no input, creates the block reward + fees for the miner. This is how new BTC is minted.

Why Blockchains — Core Concepts

The problem it solves
Trust without a central authority (bank/gov). No single party can rewrite history or double-spend.
Immutability
Each block commits to prevHash, so editing old data forces re-mining every subsequent block faster than the network — practically impossible.
Consensus (Nakamoto / longest chain)
Honest nodes accept the chain with the most cumulative PoW. Attacker needs >50% of hash power to rewrite it.
Decentralization vs distribution
Every full node keeps a full copy of the ledger; anyone can verify independently. No trusted intermediary.
Double-spend prevention
Ordering txs in mined blocks + signatures makes spending the same coin twice detectable and rejectable.

Quick tips & gotchas

  • digest() consumes the hash object — you must createHash() fresh for every hash; reusing one throws.
  • Higher difficulty = exponentially more work: adding one leading hex zero multiplies expected attempts by ~16.
  • Never log or commit a private key. Losing it = losing funds; leaking it = losing funds. There is no reset.
  • JSON.stringify is only deterministic if key order is stable — sort keys before hashing transactions or two nodes will disagree.
  • PoW is asymmetric: finding a valid nonce is brute-force expensive, but any node verifies it with a single hash.
  • Changing any field in a block changes its hash, which breaks prevHash of the next block — that cascade is what makes tampering detectable.
  • Bitcoin uses double SHA-256 and the secp256k1 curve specifically — match those if you're mimicking real Bitcoin behavior.
  • A 51% attack lets you reorder/reverse recent txs, but not steal coins you have no private key for or mint beyond the reward schedule.
