MongoDB, Zod, Error handling and Input validation — Cheat Sheet

Cohort 3.0 · 13 lessons · quick reference

Setup & Atlas Connection

Install stack (Express + Mongoose + Zod + bcrypt + JWT)
npm install express mongoose zod bcrypt jsonwebtoken
Connect to MongoDB Atlas (call before app.listen)
const mongoose = require("mongoose");
await mongoose.connect(process.env.MONGO_URL);
app.listen(3000);
await/catch the connect — never hardcode creds; keep the URI in .env.
Atlas SRV connection string format
mongodb+srv://<user>:<pass>@cluster0.xxxxx.mongodb.net/todo-app
The path segment after .net/ is the DB name — omit it and you write to `test`. URL-encode special chars in the password.
Atlas free tier checklist
Create M0 cluster -> Database Access (add user/pass) -> Network Access (allow 0.0.0.0/0 for dev) -> Connect -> copy driver URI.

Schema & Model (Mongoose)

Define schema + model
const { Schema, model } = require("mongoose");
const UserSchema = new Schema({
  email:    { type: String, required: true, unique: true },
  password: { type: String, required: true },
  name:     String
});
const User = model("User", UserSchema);
Collection name is pluralized+lowercased automatically: `User` -> `users`.
Common field options
age: { type: Number, min: 0, default: 0 }
role: { type: String, enum: ["user","admin"], default: "user" }
createdAt: { type: Date, default: Date.now }
`unique: true` builds an index — it is NOT a validator; duplicate insert throws E11000.
Auto timestamps
new Schema({ ... }, { timestamps: true });
Adds managed `createdAt` / `updatedAt` fields.

CRUD Operations

Create
await User.create({ email, password });
// bulk seed
await Todo.insertMany([{ title: "a" }, { title: "b" }]);
Read
await User.find({ role: "admin" });      // array
await User.findOne({ email });            // one or null
await User.findById(id);                  // by _id
await Todo.find({}).select("title done"); // projection
Queries are thenables — always `await` (or `.exec()`). No match returns [] / null, not an error.
Update
await Todo.updateOne({ _id: id }, { $set: { done: true } });
await Todo.findByIdAndUpdate(id, { title }, { new: true });
`{ new: true }` returns the updated doc (default returns the pre-update one).
Delete
await Todo.deleteOne({ _id: id });
await Todo.deleteMany({ userId: uid });
Query operators
await Todo.find({ priority: { $gt: 3 } });
await Todo.find({ status: { $in: ["open","wip"] } });
await Todo.find({ $or: [{ a: 1 }, { b: 2 }] });

Input Validation (Zod)

Define a schema
const { z } = require("zod");
const SignupSchema = z.object({
  email:    z.string().email(),
  password: z.string().min(6),
  name:     z.string().optional()
});
safeParse — no throw (preferred in routes)
const parsed = SignupSchema.safeParse(req.body);
if (!parsed.success) {
  return res.status(400).json({ errors: parsed.error.format() });
}
const { email, password } = parsed.data;
Use parsed.data (coerced/typed) downstream — do not re-read req.body.
parse — throws ZodError (let error middleware catch it)
const data = SignupSchema.parse(req.body);
Handy validators
z.number().int().positive()
z.string().min(3).max(20)
z.enum(["user","admin"])
z.array(z.string())
z.string().datetime()
z.coerce.number()  // "5" -> 5 (query params)
Validate as reusable middleware: `(schema) => (req,res,next) => {...}` to keep routes clean.

Password Hashing (bcrypt)

Hash on signup
const bcrypt = require("bcrypt");
const hash = await bcrypt.hash(password, 10); // 10 = salt rounds
await User.create({ email, password: hash });
Never store plaintext. Salt rounds ~10-12; higher = slower = safer.
Verify on signin
const user = await User.findOne({ email });
const ok = user && await bcrypt.compare(password, user.password);
if (!ok) return res.status(403).json({ msg: "Invalid creds" });
Return the same generic message for bad email AND bad password (avoid user enumeration).
Issue JWT after auth
const token = jwt.sign({ id: user._id }, process.env.JWT_SECRET);

Error Handling (Express)

try/catch per async route
app.post("/todo", async (req, res) => {
  try {
    const todo = await Todo.create(req.body);
    res.json(todo);
  } catch (e) { res.status(500).json({ msg: "Server error" }); }
});
Un-awaited/unhandled rejections in async handlers do NOT auto-reach error middleware in Express 4.
Global error middleware (4 args, register LAST)
app.use((err, req, res, next) => {
  if (err instanceof z.ZodError)
    return res.status(400).json({ errors: err.format() });
  if (err.code === 11000)
    return res.status(409).json({ msg: "Duplicate" });
  console.error(err);
  res.status(500).json({ msg: "Something broke" });
});
Must have exactly 4 params or Express treats it as a normal handler.
Async wrapper (forward errors to middleware)
const wrap = fn => (req, res, next) =>
  Promise.resolve(fn(req, res, next)).catch(next);
app.post("/todo", wrap(async (req, res) => { ... }));
404 fallback
app.use((req, res) => res.status(404).json({ msg: "Not found" }));

Relationships in Mongo

Referenced (normalized) — store ObjectId + ref
const TodoSchema = new Schema({
  title:  String,
  userId: { type: Schema.Types.ObjectId, ref: "User" }
});
`ref` string must match the model name passed to model("User").
populate — join the referenced doc
await Todo.find({ userId }).populate("userId", "email name");
2nd arg = field projection on the joined doc. Adds a query round-trip.
Embedded (denormalized) — subdocuments
const UserSchema = new Schema({
  addresses: [{ city: String, zip: String }]
});
Embed for data read together & bounded in size; reference for shared/unbounded/large relations.
Filter by owner (auth scoping)
await Todo.find({ userId: req.userId });
Always scope queries to the authenticated user — never trust an id from the request body.

Quick tips & gotchas

  • MongoDB is schema-less at the DB level — Mongoose schemas + Zod are what actually enforce structure. Validate at the edge (Zod) and at the model (Mongoose).
  • `unique: true` is an index, not validation: it only errors on write (E11000 code 11000), and only after the index is built. Handle that error code explicitly.
  • Every Mongoose query returns a thenable but does nothing until awaited — a forgotten `await` gives you a Query object, not data.
  • Never return raw Mongo/Zod errors to clients — they leak schema and internals. Map to clean status codes (400/403/409/500) in error middleware.
  • Hash passwords with bcrypt BEFORE saving and strip `password` from any response (`.select('-password')`).
  • Store secrets (MONGO_URL, JWT_SECRET) in .env + process.env; add .env to .gitignore. A leaked Atlas URI = full DB access.
  • `findByIdAndUpdate` returns the OLD doc unless you pass `{ new: true }`; validators don't run on updates unless `{ runValidators: true }`.
  • Prefer `safeParse` in request handlers (branch on `.success`) and reserve throwing `parse` for code paths funneled through global error middleware.
