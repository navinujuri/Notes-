Node.js Runtime — Cheat Sheet

Cohort 3.0 · 9 lessons · quick reference

Node.js & Bun — what they are

Node.js
C++ runtime embedding Google's V8 JS engine + libuv event loop. Runs JS outside the browser (servers, CLIs, tooling). Single-threaded event loop, non-blocking async I/O.
Check version / run a file
node -v          # runtime version
node script.js   # execute a file
node             # REPL
Bun
All-in-one JS/TS runtime (uses JavaScriptCore, not V8). Drop-in Node replacement + bundler + test runner + package manager. Runs .ts directly, much faster installs.
Run with Bun
bun -v
bun run index.ts   # native TS, no tsc
bun index.ts       # shorthand
bun install        # fast npm-compatible installs
Runtime globals (no import needed)
process, console, __dirname, __filename,
setTimeout, setInterval, Buffer, global
__dirname/__filename are CJS-only; in ESM use import.meta.url / import.meta.dirname.

Start a project

Init package.json interactively
npm init
Init with all defaults (fast)
npm init -y
Minimal package.json
{
  "name": "my-app",
  "version": "1.0.0",
  "main": "index.js",
  "type": "commonjs",
  "scripts": { "start": "node index.js" }
}
Set "type": "module" to use ESM import/export instead of require.
Run a script
npm run start   # runs scripts.start
npm start       # start/test/stop are special: 'run' optional
Custom scripts always need 'run' (e.g. npm run build).
Gitignore node_modules
echo node_modules/ > .gitignore
Never commit node_modules — reinstall from lockfile instead.

npm — install & manage

Install all deps from package.json
npm install   # or: npm i
Add a runtime dependency
npm install express
Goes under "dependencies"; writes exact tree to package-lock.json.
Add a dev-only dependency
npm install -D typescript nodemon
-D / --save-dev → "devDependencies", excluded in `npm i --production`.
Install a specific version
npm install lodash@4.17.21
npm install react@latest
Global install (CLIs)
npm install -g nodemon
Prefer npx for one-off use to avoid polluting global env.
Run a package binary without installing
npx cowsay "hi"
npx create-react-app my-app
Remove / update / audit
npm uninstall express
npm update
npm outdated
npm audit fix

Modules — internal (built-in)

Import built-ins (CommonJS)
const fs = require('fs');
const path = require('path');
const os = require('os');
Use the 'node:' prefix to be explicit: require('node:fs').
Import built-ins (ESM)
import fs from 'node:fs';
import { readFile } from 'node:fs/promises';
fs — read/write files
const fs = require('fs');
fs.readFileSync('a.txt', 'utf8');           // sync
fs.promises.writeFile('a.txt', 'hi');       // async/await
Always pass 'utf8' or you get a raw Buffer.
path — build safe paths
path.join(__dirname, 'data', 'a.txt');
path.resolve('./a.txt');
path.extname('a.tar.gz');   // '.gz'
process — env, args, exit
process.argv;        // CLI args array
process.env.PORT;     // environment vars
process.cwd();        // working dir
process.exit(1);      // exit with code
Other common built-ins
http, https, url, crypto, events,
stream, util, child_process, readline

Modules — external & your own

Your own file (relative path)
// CommonJS
module.exports = { add };
const { add } = require('./math');
Must start with ./ or ../ for local files; bare names resolve to node_modules.
Your own file (ESM)
// math.mjs / "type":"module"
export const add = (a, b) => a + b;
import { add } from './math.js';
ESM requires the file extension in the import path.
External package (installed)
const express = require('express');
// or ESM:
import express from 'express';
Resolution order
Local ./path → core module → node_modules (up the dir tree). Directory imports use its package.json "main" or index.js.

package-lock.json & versioning

What package-lock.json is
Auto-generated. Pins the EXACT version + integrity hash of every dep AND transitive dep so installs are reproducible. Commit it.
Semver ranges in package.json
"express": "^4.18.2"   // caret: 4.x.x  (minor+patch)
"express": "~4.18.2"   // tilde: 4.18.x (patch only)
"express": "4.18.2"    // exact
^ allows minor bumps, ~ only patch. Lockfile records what was actually installed within that range.
Clean reproducible install (CI)
npm ci
Installs strictly from lockfile, deletes node_modules first, fails if package.json & lock disagree. Faster than `npm i`.
dependencies vs devDependencies
Runtime code (express, dotenv) → dependencies. Build/test tooling (typescript, jest, nodemon) → devDependencies.

Assignments — build a CLI

Read CLI arguments
// node app.js add 5
const [,, cmd, ...args] = process.argv;
console.log(cmd, args); // 'add' ['5']
argv[0]=node path, argv[1]=script path, real args start at index 2.
Basic command router
switch (cmd) {
  case 'add':  addTodo(args.join(' ')); break;
  case 'list': listTodos(); break;
  default: console.log('Unknown command');
}
Persist data to a JSON file
const fs = require('fs');
const FILE = './todos.json';
const load = () =>
  fs.existsSync(FILE) ? JSON.parse(fs.readFileSync(FILE,'utf8')) : [];
const save = (d) => fs.writeFileSync(FILE, JSON.stringify(d, null, 2));
Interactive prompts (readline)
const readline = require('readline');
const rl = readline.createInterface({ input: process.stdin, output: process.stdout });
rl.question('Name? ', (name) => { console.log(name); rl.close(); });
Make it an executable bin
// top of index.js
#!/usr/bin/env node
Add "bin": { "mycli": "index.js" } to package.json, then `npm link` to run `mycli` globally.
Parse flags cleanly (Node 18.3+)
const { values } = require('node:util')
  .parseArgs({ options: { name: { type: 'string' } } });
Built-in flag parser; for anything richer use commander or yargs.

Quick tips & gotchas

  • process.argv real args start at index 2 — argv[0] is the node binary and argv[1] is the script path.
  • Commit package-lock.json; never commit node_modules. Use `npm ci` in CI for exact, reproducible installs.
  • `^` (caret) allows minor+patch upgrades, `~` (tilde) only patch. The lockfile records the exact resolved version.
  • require() is synchronous CommonJS; import is ESM. Mixing them needs "type":"module" or the .mjs/.cjs extensions — pick one per file.
  • Local imports must start with ./ or ../; a bare name like 'fs' or 'express' resolves to a core module or node_modules.
  • fs.readFileSync returns a Buffer unless you pass an encoding like 'utf8'.
  • Node's I/O is async/non-blocking — prefer fs.promises / await over the Sync variants in servers to avoid blocking the event loop.
  • Use npx to run a package binary once (npx create-react-app) instead of installing it globally.
  • Bun runs TypeScript and installs deps natively with no extra tooling — `bun index.ts`, `bun install` — and is largely npm/Node compatible.
