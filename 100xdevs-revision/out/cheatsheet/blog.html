Blogging website — Cheat Sheet

Cohort 2.0 · 15 lessons · quick reference

The Stack & Backend Init (Steps 1-2)

Stack: serverless edge backend + shared types + React FE
Cloudflare Workers + Hono (backend), PostgreSQL + Prisma (DB), Zod (validation, shared via npm), JWT (auth), React + Vite + react-router-dom + Tailwind (FE), TypeScript everywhere.
Scaffold the Hono backend on Cloudflare Workers
npm create hono@latest backend
# template: cloudflare-workers
cd backend && npm i
npm run dev   # local at http://localhost:8787
Minimal Hono app entry (src/index.ts)
import { Hono } from 'hono'
const app = new Hono()
app.get('/', (c) => c.text('Hello Hono!'))
export default app
Export default the app — Workers runtime calls its fetch handler. No app.listen().
Type Bindings (env) + Variables (context state) generics
const app = new Hono<{
  Bindings: { DATABASE_URL: string; JWT_SECRET: string };
  Variables: { userId: string };
}>()
Bindings type c.env.*; Variables type c.get()/c.set(). Without them TS errors on env/context access.

Prisma + DB (Step 4)

Install Prisma + Accelerate (connection pooling for edge)
npm i prisma --save-dev
npm i @prisma/client @prisma/extension-accelerate
npx prisma init
schema.prisma — User + Post models
model User {
  id       String @id @default(uuid())
  email    String @unique
  name     String?
  password String
  posts    Post[]
}
model Post {
  id        String  @id @default(uuid())
  title     String
  content   String
  published Boolean @default(false)
  author    User    @relation(fields: [authorId], references: [id])
  authorId  String
}
Migrate (uses DIRECT url in .env) then generate edge client
npx prisma migrate dev --name init
npx prisma generate --no-engine
--no-engine drops the query-engine binary (Workers can't run it); queries go through Accelerate. Re-run generate after every schema change.
Instantiate PrismaClient INSIDE each handler (edge client)
import { PrismaClient } from '@prisma/client/edge'
import { withAccelerate } from '@prisma/extension-accelerate'

const prisma = new PrismaClient({
  datasourceUrl: c.env.DATABASE_URL,
}).$extends(withAccelerate())
Import from '@prisma/client/edge'. Create per-request — Workers are stateless and have no long-lived global connection.
Two DB URLs
Runtime (wrangler.toml): pooled Accelerate string prisma://accelerate.prisma-data.net/?api_key=... . Migrations (.env): direct postgresql://... connection.

Handlers, Routing & Middleware (Steps 3, 5, 6)

Split routers per resource (src/routes/user.ts)
import { Hono } from 'hono'
export const userRouter = new Hono<{ Bindings: {...} }>()
userRouter.post('/signup', async (c) => { /* ... */ })
userRouter.post('/signin', async (c) => { /* ... */ })
Mount sub-routers with app.route (src/index.ts)
import { userRouter } from './routes/user'
import { blogRouter } from './routes/blog'
app.route('/api/v1/user', userRouter)
app.route('/api/v1/blog', blogRouter)
Paths inside a router are relative to the mount prefix.
JWT auth middleware guarding /blog/*
import { verify } from 'hono/jwt'
blogRouter.use('/*', async (c, next) => {
  const jwt = c.req.header('authorization') || ''
  const user = await verify(jwt, c.env.JWT_SECRET)
  if (!user) { c.status(403); return c.json({ message: 'unauthorized' }) }
  c.set('userId', String(user.id))
  await next()
})
Course sends the raw token (no 'Bearer ' prefix). If you prefix, strip it before verify.
Enable CORS (needed once FE calls the API)
import { cors } from 'hono/cors'
app.use('/*', cors())

Auth & Blog Routes (Step 7)

Signup — create user, return signed JWT
import { sign } from 'hono/jwt'
const body = await c.req.json()
const user = await prisma.user.create({
  data: { email: body.email, password: body.password, name: body.name }
})
const jwt = await sign({ id: user.id }, c.env.JWT_SECRET)
return c.json({ jwt })
Demo hashes nothing — in real apps hash passwords (bcrypt/argon).
Signin — verify creds, return JWT
const user = await prisma.user.findFirst({
  where: { email: body.email, password: body.password }
})
if (!user) { c.status(403); return c.json({ message: 'invalid' }) }
return c.json({ jwt: await sign({ id: user.id }, c.env.JWT_SECRET) })
Create blog (author = c.get('userId'))
const authorId = c.get('userId')
const blog = await prisma.post.create({
  data: { title: body.title, content: body.content, authorId }
})
return c.json({ id: blog.id })
Update / get one / list all (bulk)
await prisma.post.update({ where: { id: body.id }, data: {...} })
await prisma.post.findFirst({ where: { id: c.req.param('id') } })
await prisma.post.findMany({
  select: { id:true, title:true, content:true, author:{ select:{ name:true } } }
})
Use c.req.param('id') for /:id routes. Prefer select to control payload shape.

Zod + Common Package (Steps 10-12)

Init the shared common package
mkdir common && cd common && npm init -y
npm i zod
npx tsc --init   # set "declaration": true, "rootDir":"./src", "outDir":"./dist"
Define schemas + inferred types (common/src/index.ts)
import { z } from 'zod'
export const signupInput = z.object({
  email: z.string().email(),
  password: z.string().min(6),
  name: z.string().optional()
})
export type SignupInput = z.infer<typeof signupInput>
Export both the schema (runtime validation) and z.infer type (compile-time). Single source of truth for FE + BE.
Build + publish to npm (scoped, public)
tsc -b
npm login
npm publish --access public   # name e.g. @yourname/medium-common
After each schema change: npm version patch, tsc -b, npm publish; then npm update the package in FE & BE.
Consume in backend with safeParse (never trust input)
import { signupInput } from '@yourname/medium-common'
const { success } = signupInput.safeParse(body)
if (!success) { c.status(411); return c.json({ message: 'Inputs incorrect' }) }
safeParse returns { success } instead of throwing like parse().

Deploy — Cloudflare Workers (Steps 8-9)

wrangler.toml — env vars for runtime
name = "backend"
main = "src/index.ts"
compatibility_date = "2024-01-01"

[vars]
DATABASE_URL = "prisma://accelerate.prisma-data.net/?api_key=..."
JWT_SECRET = "super-secret"
For real secrets prefer: npx wrangler secret put JWT_SECRET (keeps them out of the repo).
Login + deploy
npx wrangler login
npm run deploy   # wrangler deploy
Gives a *.workers.dev URL. Use that as the FE's BACKEND_URL.
Test with Postman before FE (Step 8)
POST {{url}}/api/v1/user/signup   body: raw JSON
POST {{url}}/api/v1/blog          header: Authorization: <jwt>
Copy jwt from signup/signin response into the Authorization header for /blog/* calls.

Frontend — Vite + React Router (Steps 13-15)

Scaffold FE + Tailwind + router + common
npm create vite@latest frontend -- --template react-ts
cd frontend && npm i
npm i -D tailwindcss postcss autoprefixer && npx tailwindcss init -p
npm i react-router-dom axios @yourname/medium-common
Routes in App.tsx
import { BrowserRouter, Routes, Route } from 'react-router-dom'
<BrowserRouter>
  <Routes>
    <Route path="/signup" element={<Signup />} />
    <Route path="/signin" element={<Signin />} />
    <Route path="/blogs" element={<Blogs />} />
    <Route path="/blog/:id" element={<Blog />} />
  </Routes>
</BrowserRouter>
Auth call — store JWT, navigate
import axios from 'axios'
import { useNavigate } from 'react-router-dom'
const navigate = useNavigate()
const res = await axios.post(`${BACKEND_URL}/api/v1/user/signup`, inputs)
localStorage.setItem('token', res.data.jwt)
navigate('/blogs')
Authenticated fetch — read :id param + send token
import { useParams } from 'react-router-dom'
const { id } = useParams()
const res = await axios.get(`${BACKEND_URL}/api/v1/blog/${id}`, {
  headers: { Authorization: localStorage.getItem('token') }
})
Match the header format the backend middleware expects (raw token vs 'Bearer ').
Reusable components
Auth (shared signup/signin form), LabelledInput, Quote (split-screen), Appbar, BlogCard, BlogSkeleton, Avatar. Extract data-fetching into custom hooks (useBlog, useBlogs).

Quick tips & gotchas

  • Create PrismaClient inside each handler, not globally — Cloudflare Workers are stateless per request and share no connection; use the '@prisma/client/edge' import + withAccelerate for pooling.
  • Access secrets via c.env.X on Workers (not process.env), and type them in the Hono<{ Bindings }> generic or TS will error.
  • Keep two DB URLs: the pooled Accelerate prisma:// URL for runtime (wrangler.toml) and a direct postgresql:// URL in .env for prisma migrate/generate.
  • Re-run `npx prisma generate --no-engine` after every schema change, or the client types/queries go stale.
  • Common package is the single source of truth: after editing schemas, npm version patch + publish, then npm update it in BOTH frontend and backend.
  • Use zod safeParse (returns { success }) over parse (throws); return 411 on invalid input.
  • Middleware sends the raw JWT in Authorization (no 'Bearer ' prefix in the course) — keep FE and verify() consistent or auth silently 403s.
  • Enable CORS on the backend (hono/cors) before the React app can call it, and set BACKEND_URL to the deployed *.workers.dev origin.
