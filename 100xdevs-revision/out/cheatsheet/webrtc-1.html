WebRTC Basic implementation, advance discussion — Cheat Sheet

Cohort 2.0 · 9 lessons · quick reference

Why WebRTC & Core Jargon

Why WebRTC
Browser-native, plugin-free P2P real-time media + arbitrary data. Sub-500ms latency over UDP (SRTP/DTLS), encrypted by default. Alternatives (WebSocket/HLS) add server hop + latency.
Signaling
How peers exchange SDP + ICE candidates BEFORE the P2P link exists. NOT part of the WebRTC spec — you build it (WebSocket, HTTP, even a QR code).
SDP (Session Description Protocol)
Text blob describing codecs, resolutions, IPs, encryption keys. Exchanged as offer/answer.
ICE
Interactive Connectivity Establishment — framework that gathers candidate address:port pairs and probes them to find a working path through NATs.
STUN
Lightweight server that tells a peer its public IP:port (server-reflexive candidate). Free (e.g. stun:stun.l.google.com:19302). Handles ~80% of NATs.
TURN
Relay server that forwards media when direct P2P fails (symmetric NAT). Bandwidth-heavy, must be authenticated. Needed for ~15-20% of connections.
Candidate types
host (local LAN) < srflx (STUN/public) < relay (TURN). ICE picks the best working candidate-pair.

Frontend Setup: Media + PeerConnection

Capture camera/mic
const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: true });
localVideo.srcObject = stream;
Requires a secure context (HTTPS or localhost). Returns a rejected promise otherwise.
Screen share
const screen = await navigator.mediaDevices.getDisplayMedia({ video: true, audio: false });
Cannot be combined with getUserMedia in one call — capture separately.
Create the peer connection
const pc = new RTCPeerConnection({
  iceServers: [
    { urls: 'stun:stun.l.google.com:19302' },
    { urls: 'turn:turn.example.com:3478', username: 'user', credential: 'pass' }
  ]
});
Add local tracks (send media)
stream.getTracks().forEach(track => pc.addTrack(track, stream));
Adding a track fires onnegotiationneeded — do this before/around offer creation.
Receive remote media
pc.ontrack = ({ streams: [remoteStream] }) => {
  remoteVideo.srcObject = remoteStream;
};
Watch connection health
pc.onconnectionstatechange = () => console.log(pc.connectionState);
States: new → connecting → connected → disconnected → failed → closed. Treat 'failed' as fatal (restart ICE).

Connecting: Offer / Answer / ICE

Caller — create & send offer
const offer = await pc.createOffer();
await pc.setLocalDescription(offer);
signaling.send({ type: 'offer', sdp: pc.localDescription });
Callee — answer the offer
await pc.setRemoteDescription(offer);
const answer = await pc.createAnswer();
await pc.setLocalDescription(answer);
signaling.send({ type: 'answer', sdp: pc.localDescription });
Caller — apply the answer
await pc.setRemoteDescription(answer);
Trickle ICE — send candidates
pc.onicecandidate = ({ candidate }) => {
  if (candidate) signaling.send({ type: 'candidate', candidate });
};
Send each candidate as it's found — don't wait for full gathering. null candidate = gathering complete.
Add remote candidates
await pc.addIceCandidate(new RTCIceCandidate(candidate));
Can arrive before setRemoteDescription — buffer them in an array and flush after remote desc is set, or use perfect negotiation.
Perfect negotiation (glare-proof)
let makingOffer = false;
pc.onnegotiationneeded = async () => {
  makingOffer = true;
  await pc.setLocalDescription();
  signaling.send({ description: pc.localDescription });
  makingOffer = false;
};
Designate one peer 'polite' (rolls back on collision) and one 'impolite' to resolve simultaneous offers.

Data Channels

Open a channel (initiator)
const dc = pc.createDataChannel('chat', { ordered: true });
dc.onopen = () => dc.send('hello');
dc.onmessage = (e) => console.log('recv', e.data);
Receive a channel (other side)
pc.ondatachannel = ({ channel }) => {
  channel.onmessage = (e) => console.log(e.data);
};
Unreliable / low-latency mode
pc.createDataChannel('game', { ordered: false, maxRetransmits: 0 });
UDP-like semantics for game state / cursors — skips retransmits, avoids head-of-line blocking.
Backpressure
if (dc.bufferedAmount > 16 * 1024 * 1024) return; // pause sends
dc.bufferedAmountLowThreshold = 1024 * 1024;
dc.onbufferedamountlow = () => resumeSending();
Sending faster than the link drains bloats the buffer — gate on bufferedAmount.

Backend: Signaling Server

Minimal WebSocket relay (Node ws)
import { WebSocketServer } from 'ws';
const wss = new WebSocketServer({ port: 8080 });
wss.on('connection', (ws) => {
  ws.on('message', (msg) => {
    // relay to everyone else (naive: use rooms in prod)
    wss.clients.forEach((c) => {
      if (c !== ws && c.readyState === 1) c.send(msg.toString());
    });
  });
});
Server only shuttles offer/answer/candidate JSON — it never touches media in a P2P topology.
Room-scoped routing
In production key sockets by roomId and forward only within the room. Track peer join/leave to trigger renegotiation.
Run coturn (STUN+TURN)
turnserver -a -u user:pass -r example.com \
  --min-port=49152 --max-port=65535 \
  --listening-port=3478 --fingerprint
Self-hosting TURN is the reliability fix for corporate/symmetric NATs. Use TLS (turns:) + short-lived credentials in prod.
Force relay (test TURN)
new RTCPeerConnection({ iceServers, iceTransportPolicy: 'relay' });
Drops host/srflx candidates so only TURN paths are used — verifies your TURN server actually works.

Stats, Bitrate & Track Control

Read live stats
const stats = await pc.getStats();
stats.forEach((r) => {
  if (r.type === 'inbound-rtp' && r.kind === 'video')
    console.log(r.framesPerSecond, r.packetsLost, r.jitter);
});
Key report types: inbound-rtp, outbound-rtp, candidate-pair (chosen path + RTT), remote-inbound-rtp (round-trip + loss reported by peer).
Round-trip time
if (r.type === 'candidate-pair' && r.state === 'succeeded')
  console.log('RTT', r.currentRoundTripTime);
Cap send bitrate
const sender = pc.getSenders().find(s => s.track?.kind === 'video');
const p = sender.getParameters();
p.encodings[0].maxBitrate = 500_000; // 500 kbps
await sender.setParameters(p);
Switch track (screen share toggle)
const sender = pc.getSenders().find(s => s.track.kind === 'video');
await sender.replaceTrack(screenTrack);
replaceTrack swaps media WITHOUT renegotiation — smoother than remove/add.
Cleanup (release camera)
stream.getTracks().forEach(t => t.stop());
pc.close();
Forgetting t.stop() leaves the camera light on and the device locked.

Libraries & Topologies

simple-peer (thin P2P wrapper)
import Peer from 'simple-peer';
const peer = new Peer({ initiator: true, stream });
peer.on('signal', (data) => signaling.send(data)); // pass to other side
peer.signal(remoteData);                            // feed remote signal in
peer.on('stream', (s) => { remoteVideo.srcObject = s; });
Bundles offer/answer/ICE into one opaque 'signal' blob. You still ship it over your own channel.
Other P2P libs
PeerJS (hosted broker + client), Daily/Twilio (managed SDKs). Fine for 1:1 or tiny groups.
Mesh
Every peer ↔ every peer: N·(N-1)/2 connections, each uploads N-1 copies. Zero server media cost but dies past ~4-5 participants (uplink + CPU).
SFU (Selective Forwarding Unit)
Each peer uploads ONCE to the server; server forwards streams to others. Scales to dozens+. Servers: mediasoup, LiveKit, ion-sfu, Janus, Jitsi Videobridge.
Simulcast (SFU)
pc.addTransceiver(track, { sendEncodings: [
  { rid: 'h', maxBitrate: 900_000 },
  { rid: 'm', maxBitrate: 300_000, scaleResolutionDownBy: 2 },
  { rid: 'l', maxBitrate: 100_000, scaleResolutionDownBy: 4 }
] });
Send multiple quality layers; SFU forwards the layer each receiver can handle.
MCU (Multipoint Control Unit)
Server decodes + mixes all streams into one composite. Lowest client bandwidth/CPU, highest server cost. Used for recording, PSTN bridges, legacy endpoints.

Quick tips & gotchas

  • getUserMedia/getDisplayMedia only work in a secure context — HTTPS or localhost. Deploying to plain HTTP silently breaks camera access.
  • Signaling is your responsibility — WebRTC defines no wire protocol for exchanging SDP/candidates. Any transport (WS/HTTP/SSE) works.
  • ICE candidates often arrive before setRemoteDescription resolves. Buffer them and flush after remote desc is set, or adopt the perfect-negotiation pattern.
  • Always deploy TURN for production. STUN alone fails behind symmetric/corporate NATs (~15-20% of users). Test with iceTransportPolicy: 'relay'.
  • Mesh topology does not scale — uplink is O(N) per peer. Switch to an SFU (mediasoup/LiveKit) beyond ~4 participants.
  • Handle simultaneous offers (glare) with perfect negotiation: one polite peer that rolls back, one impolite peer that ignores the incoming offer.
  • Use replaceTrack() to swap camera↔screen or change quality without a full renegotiation round-trip.
  • On teardown, call track.stop() on every track AND pc.close() — closing the connection alone leaves the camera/mic hardware active.
