Middlwares in next — Cheat Sheet

Cohort 2.0 · 5 lessons · quick reference

What & Where — Setup

Middleware = code that runs BEFORE a request is completed
Runs on every matching request; can rewrite, redirect, block, or mutate headers/cookies before the route handler/page renders.
Single file at project root (or inside src/)
my-app/
  middleware.ts   <- here (NOT in app/)
  app/
  src/middleware.ts  <- OR here if using src/
One middleware.ts per project. It's the top-level file, sibling to app/ (or inside src/).
Minimal skeleton
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  return NextResponse.next()
}

export const config = { matcher: '/dashboard/:path*' }
Export a function named `middleware` (or `export default`). `config.matcher` scopes it.
Runtime
Edge runtime by default (no Node APIs like fs). Next 15.2+ can opt into Node via `export const config = { runtime: 'nodejs' }` (experimental).

Core APIs — NextRequest / NextResponse

NextResponse.next() — let request continue
return NextResponse.next()
Redirect (changes URL in browser)
return NextResponse.redirect(new URL('/login', request.url))
Pass an absolute URL; build with `new URL(path, request.url)`.
Rewrite (proxy — URL stays, content differs)
return NextResponse.rewrite(new URL('/proxy', request.url))
Browser bar unchanged; internally serves another path.
Return JSON / block early
return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
Read request URL parts
const { pathname, searchParams } = request.nextUrl
searchParams.get('token')
Read headers / cookies
request.headers.get('authorization')
request.cookies.get('session')?.value
Geo / IP (Next 15+)
import { geolocation, ipAddress } from '@vercel/functions'
const { country } = geolocation(request)
`request.geo` and `request.ip` were REMOVED in Next 15 — use @vercel/functions helpers.

Code — Auth guard

Protect routes, redirect unauthenticated users
export function middleware(request: NextRequest) {
  const token = request.cookies.get('token')?.value

  if (!token) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  return NextResponse.next()
}

export const config = {
  matcher: ['/dashboard/:path*', '/settings/:path*'],
}
Preserve intended destination after login
const url = new URL('/login', request.url)
url.searchParams.set('from', request.nextUrl.pathname)
return NextResponse.redirect(url)

Code #2 — Rewrite, headers & cookies

Inject a response header
const res = NextResponse.next()
res.headers.set('x-custom-header', 'hello')
return res
Set a cookie on the response
const res = NextResponse.next()
res.cookies.set('theme', 'dark', { path: '/', maxAge: 60 * 60 })
return res
Forward/modify REQUEST headers to downstream
const headers = new Headers(request.headers)
headers.set('x-user-id', '123')
return NextResponse.next({ request: { headers } })
Pass headers to route handlers/server components; read via `headers()`.
A/B test via rewrite
const bucket = Math.random() < 0.5 ? 'a' : 'b'
return NextResponse.rewrite(new URL(`/home-${bucket}`, request.url))

Selectively running middleware

Single path with wildcard
export const config = { matcher: '/about/:path*' }
`:path*` = zero-or-more segments. `:path` = exactly one.
Multiple matchers (array)
export const config = {
  matcher: ['/dashboard/:path*', '/api/:path*'],
}
Match everything EXCEPT static/api (most common)
export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
}
Negative lookahead skips assets & _next internals — huge perf win.
Conditional matcher (has / missing)
export const config = {
  matcher: [{
    source: '/api/:path*',
    has: [{ type: 'header', key: 'authorization' }],
    missing: [{ type: 'cookie', key: 'session' }],
  }],
}
type: 'header' | 'cookie' | 'query' | 'host'.
In-code branching (when matcher isn't enough)
export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl
  if (pathname.startsWith('/admin')) return checkAdmin(request)
  return NextResponse.next()
}

Quick tips & gotchas

  • Matcher values MUST be static string literals — they're analyzed at build time, so you can't compute them at runtime or import them from variables.
  • Always return a NextResponse (or nothing). Forgetting to `return NextResponse.next()` on the pass-through path silently breaks the request.
  • Edge runtime: no Node.js APIs (fs, net, most crypto). Keep middleware light — it runs on EVERY matching request, so no DB calls or heavy work.
  • Exclude _next/ and static assets in your matcher, otherwise middleware fires for every image/JS chunk and tanks performance.
  • Set cookies/headers on the RESPONSE object you return — mutating `request.cookies` only affects downstream reads, not the browser.
  • Build redirect/rewrite URLs with `new URL(path, request.url)` — a bare string like '/login' throws (needs absolute URL).
  • Only ONE middleware.ts per project. Compose multiple concerns by branching on `request.nextUrl.pathname` inside it (or chain helper functions).
  • `redirect` changes the browser URL (client sees it); `rewrite` keeps the URL but serves different content (invisible proxy). Pick deliberately.
