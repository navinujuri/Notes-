Promises and async, await — Cheat Sheet

Cohort 2.0 · 10 lessons · quick reference

Classes & Inheritance (JS OOP basics)

Define a class + constructor
class Person {
  constructor(name, age) {
    this.name = name;   // instance fields
    this.age = age;
  }
  greet() { return `Hi, I'm ${this.name}`; }  // prototype method
}
const p = new Person("Ada", 36);
Methods live on the prototype (shared); fields set in constructor are per-instance. `new` is mandatory.
static members & getters/setters
class Circle {
  static PI = 3.14159;
  constructor(r) { this._r = r; }
  get area() { return Circle.PI * this._r ** 2; }
  set radius(v) { this._r = v; }
  static of(r) { return new Circle(r); }  // factory
}
Circle.PI;            // access on class, not instance
Circle.of(5).area;    // getter called w/o ()
`static` = called on the class itself. Getters/setters are accessed like properties (no parens).
Inheritance: extends + super
class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} makes a noise`; }
}
class Dog extends Animal {
  constructor(name, breed) {
    super(name);          // MUST call before using `this`
    this.breed = breed;
  }
  speak() { return `${super.speak()} — woof`; }  // super.method()
}
In a subclass constructor, `super(...)` must run before any `this` access or it throws ReferenceError.
Private fields (#)
class Counter {
  #count = 0;
  inc() { this.#count++; }
  get value() { return this.#count; }
}
// obj.#count from outside => SyntaxError
`#name` is truly private (enforced by engine), unlike the `_convention`.
instanceof / method check
p instanceof Person;   // true
new Dog("Rex") instanceof Animal;  // true (walks prototype chain)

Promise class — creating & consuming

Create a Promise (executor runs immediately)
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve(42), 1000);
  // reject(new Error("boom"))  on failure
});
3 states: pending → fulfilled (resolve) | rejected (reject). Settles once; later resolve/reject calls are ignored.
Consume with .then/.catch/.finally
p.then(val => console.log(val))
 .catch(err => console.error(err))
 .finally(() => console.log("done"));
`.then` returns a NEW promise → chainable. Return a value to pass down; return a promise to wait for it.
Chaining (flatten async steps)
fetch(url)
  .then(res => res.json())   // returned promise is awaited
  .then(data => data.id)
  .then(id => console.log(id))
  .catch(err => /* one catch handles whole chain */);
A single trailing .catch catches rejections from any prior step.
Instant promises
Promise.resolve(1);           // already-fulfilled
Promise.reject(new Error("x")); // already-rejected

Promise combinators (concurrency)

Promise.all — wait for all, fail fast
const [a, b] = await Promise.all([fetchA(), fetchB()]);
Rejects immediately if ANY input rejects; resolves to array of values in input order.
Promise.allSettled — never rejects
const results = await Promise.allSettled([p1, p2]);
// [{status:'fulfilled',value}, {status:'rejected',reason}]
Use when you want every outcome regardless of individual failures.
Promise.race — first to settle wins
await Promise.race([fetch(url), timeout(5000)]);
Settles (fulfill OR reject) as soon as the first input settles. Classic for timeouts.
Promise.any — first fulfilled wins
await Promise.any([mirror1(), mirror2()]);
Ignores rejections; rejects with AggregateError only if ALL reject.

async / await syntax

async function always returns a Promise
async function getUser() {
  return { id: 1 };   // wrapped => Promise<{id:1}>
}
getUser().then(u => ...);
A thrown error inside becomes a rejected promise. `return x` == `Promise.resolve(x)`.
await unwraps a promise
async function main() {
  const res = await fetch(url);   // pauses until settled
  const data = await res.json();
  return data;
}
`await` only valid inside `async` fns (or top-level in ES modules). Awaiting a non-promise just returns the value.
try/catch = your .catch
try {
  const data = await getUser();
} catch (err) {
  console.error(err);   // catches rejections + throws
} finally {
  cleanup();
}
Arrow & method forms
const f = async (x) => x * 2;
const obj = { async load() { await ...; } };
Run in parallel, not serial
// SLOW (sequential):
const a = await fa(); const b = await fb();
// FAST (concurrent):
const [a, b] = await Promise.all([fa(), fb()]);
Awaiting one-by-one when calls are independent needlessly serializes them.

Callback hell → Promises → async/await

Callback hell (the problem)
getUser(id, (err, user) => {
  if (err) return cb(err);
  getPosts(user, (err, posts) => {
    if (err) return cb(err);
    getComments(posts[0], (err, c) => { /* ...deeper... */ });
  });
});
Rightward drift + repeated error checks + no composability. Hard to read & maintain.
Same logic with async/await (flat)
const user  = await getUser(id);
const posts = await getPosts(user);
const c     = await getComments(posts[0]);
Linear, readable, single try/catch handles all errors.
Promisify a callback API
const readFileP = (path) => new Promise((resolve, reject) => {
  fs.readFile(path, 'utf8', (err, data) =>
    err ? reject(err) : resolve(data));
});
// or: const readFileP = util.promisify(fs.readFile);
Node: `import { promisify } from 'util'` wraps err-first callbacks automatically.

Defining your own async fn & error-first callbacks vs reject

Write a promise-returning helper (delay)
function delay(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}
await delay(1000);  // sleep
No built-in sleep in JS — this is the idiom.
Error-first callback convention (Node classic)
function loadUser(id, callback) {
  db.find(id, (err, row) => {
    if (err) return callback(err);       // err is 1st arg
    callback(null, row);                 // null err on success
  });
}
loadUser(1, (err, user) => { if (err) {...} });
Convention: callback(err, result). Success => err is null/undefined. Caller must check err every time.
Promise reject equivalent
function loadUser(id) {
  return new Promise((resolve, reject) => {
    db.find(id, (err, row) =>
      err ? reject(err) : resolve(row));
  });
}
// consume: try { await loadUser(1) } catch(err){...}
reject(err) ≈ passing err to error-first callback; resolve(v) ≈ callback(null, v). Errors propagate up automatically instead of manual re-passing.
Reject with Error objects, not strings
reject(new Error("not found"));   // ✅ stack trace
// reject("not found")            // ❌ no stack, harder to debug

Gotchas & patterns cheatsheet

Forgetting await = you get the Promise, not the value
const data = getUser();        // Promise { <pending> }
const data = await getUser();  // actual value
await inside .map returns array of promises
const users = await Promise.all(ids.map(id => getUser(id)));
`arr.map(async ...)` yields Promise[]; wrap in Promise.all. `forEach` does NOT await — avoid it.
Sequential loop (when order/backpressure matters)
for (const id of ids) {
  await process(id);   // one at a time
}
`for...of` with await runs serially; use when each step depends on the last.
this binding in class methods passed as callbacks
btn.onclick = this.handle.bind(this);
// or arrow class field: handle = () => { this.x }
Passing `obj.method` loses `this`. Bind or use an arrow class field.

Quick tips & gotchas

  • async function ALWAYS returns a Promise; `throw` inside becomes a rejection, `return x` becomes a resolved value.
  • One trailing `.catch()` (or a single try/catch) handles rejections from the whole chain — don't wrap every step.
  • Independent awaits run serially and waste time; batch them with Promise.all for concurrency.
  • A Promise settles exactly once — extra resolve/reject calls are silently ignored.
  • Always reject/throw `new Error(...)`, never a raw string, so you keep a stack trace.
  • `await` is only legal inside `async` functions (or top-level of an ES module) — otherwise SyntaxError.
  • `Array.forEach(async ...)` does NOT wait; use `for...of` with await (serial) or `map` + `Promise.all` (parallel).
  • In a subclass, call `super(...)` before touching `this`, and remember arrow functions don't get their own `this`.
