DOM Part 2 — Cheat Sheet

Cohort 3.0 · 8 lessons · quick reference

Create & Insert Elements

Create a node
const div = document.createElement("div");
Set visible text (safe, no HTML parsing)
div.textContent = todo.title;
Prefer over innerHTML for user input — avoids XSS injection.
Set inner HTML (parses tags)
el.innerHTML = `<span>${title}</span>`;
Unsafe with untrusted data; slower on repeated writes.
Append / prepend children
parent.appendChild(div);      // single node
parent.append(a, b, "text");   // multiple + strings
parent.prepend(div);           // insert first
Insert HTML at a position without reparsing parent
el.insertAdjacentHTML("beforeend", `<li>${t}</li>`);
positions: beforebegin | afterbegin | beforeend | afterend
Attributes & data-*
btn.setAttribute("data-id", todo.id);
btn.dataset.id = todo.id;        // same thing
const id = btn.dataset.id;       // read back (string!)
Classes
el.classList.add("done");
el.classList.remove("done");
el.classList.toggle("done");
el.classList.contains("done");

Select & Traverse

By id (fastest)
const el = document.getElementById("todos");
CSS selector — first / all
document.querySelector(".todo");        // first match | null
document.querySelectorAll(".todo");     // static NodeList
Loop a NodeList
document.querySelectorAll(".todo")
  .forEach(el => el.remove());
NodeList has forEach; HTMLCollection (.children) does not — spread it: [...el.children]
Walk the tree
el.parentElement;
el.children;              // element children only
el.closest(".card");     // nearest ancestor (or self) matching
Match a selector on an element
if (e.target.matches(".delete-btn")) { /* ... */ }

State-Derived Rendering (core mental model)

Single source of truth: UI is a function of state
let todos = [];   // the ONLY thing you mutate
// UI = render(todos). Never hand-patch the DOM ad hoc.
Change state -> call render(). Don't surgically edit DOM in event handlers.
render(): wipe + rebuild from state
function render() {
  const parent = document.getElementById("todos");
  parent.innerHTML = "";              // clear
  for (const todo of todos) {
    parent.appendChild(makeRow(todo));
  }
}
Simplest correct pattern. Every render reflects current state exactly.
Build one row (factory fn)
function makeRow(todo) {
  const div = document.createElement("div");
  div.textContent = todo.title;

  const del = document.createElement("button");
  del.textContent = "Delete";
  del.dataset.id = todo.id;
  del.onclick = () => deleteTodo(todo.id);  // closure over id

  div.appendChild(del);
  return div;
}
Batch inserts with a fragment (perf)
const frag = document.createDocumentFragment();
for (const t of todos) frag.appendChild(makeRow(t));
parent.replaceChildren(frag);   // one reflow, clears + inserts
replaceChildren() replaces all kids in one shot; avoids N layout thrashes.

Add TODO

Read input, push to state, re-render
function addTodo() {
  const input = document.getElementById("todo-input");
  const title = input.value.trim();
  if (!title) return;                    // guard empty
  todos.push({ id: Date.now(), title, done: false });
  input.value = "";                       // reset field
  render();
}
Stable unique id
id: crypto.randomUUID()   // or Date.now() for simple apps
Array index is NOT a stable id — it shifts after deletes.
Wire the button / Enter key
document.getElementById("add").addEventListener("click", addTodo);
input.addEventListener("keydown", e => {
  if (e.key === "Enter") addTodo();
});

Delete TODO

Remove from state by id, then re-render
function deleteTodo(id) {
  todos = todos.filter(t => t.id !== id);
  render();
}
filter returns a NEW array — reassign it. Delete from STATE, not the DOM directly.
Toggle done (same pattern via map)
function toggle(id) {
  todos = todos.map(t =>
    t.id === id ? { ...t, done: !t.done } : t
  );
  render();
}
Yank a single node (when not using full re-render)
el.remove();                        // modern
el.parentElement.removeChild(el);   // legacy equivalent

Event Handling & Delegation

Per-element handler (simple, from render)
del.onclick = () => deleteTodo(todo.id);
.onclick overwrites; addEventListener stacks multiple.
Event delegation — one listener on the parent
parent.addEventListener("click", (e) => {
  const btn = e.target.closest(".delete-btn");
  if (!btn) return;
  deleteTodo(Number(btn.dataset.id));
});
Survives re-renders: listener lives on the stable parent, not on rebuilt rows.
dataset is always a string
deleteTodo(Number(btn.dataset.id));
Coerce with Number()/parseInt before comparing to numeric ids.
Prevent form reload
form.addEventListener("submit", e => {
  e.preventDefault();
  addTodo();
});

Quick tips & gotchas

  • State is the source of truth: mutate the `todos` array, then call render(). Never patch the DOM directly in handlers — the two will drift out of sync.
  • Full re-render (innerHTML = "" then rebuild) is dead simple and correct for small lists; reach for DocumentFragment / replaceChildren only when it gets slow.
  • Use textContent, not innerHTML, for user-entered todo titles — innerHTML with untrusted input is an XSS hole.
  • dataset values are strings. Coerce (Number()/parseInt) before comparing dataset.id to numeric ids, or IDs will never match.
  • Give each todo a stable unique id (Date.now()/crypto.randomUUID()) — deleting by array index breaks after the first removal shifts everything.
  • filter() and map() return NEW arrays — you must reassign (todos = todos.filter(...)); they don't mutate in place.
  • Prefer event delegation (one listener on the parent) over per-row handlers: it keeps working after re-renders and avoids leaking listeners.
  • On a form, always e.preventDefault() in the submit handler or the page reloads and your state is wiped.
