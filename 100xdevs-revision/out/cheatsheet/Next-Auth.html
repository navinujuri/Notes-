Next Auth — Cheat Sheet

Cohort 2.0 · 9 lessons · quick reference

Concept: NextAuth & Why Not JWT + localStorage

What is NextAuth (Auth.js)
Full auth library for Next.js: handles OAuth (Google/GitHub/...), email, and credentials login, sessions, CSRF, and callback routing out of the box. Backend + frontend helpers in one package.
Why NOT store a JWT in localStorage
localStorage is readable by ANY JS on the page → XSS steals the token. No httpOnly. Can't auto-send on requests. Hard to invalidate/rotate server-side.
What NextAuth does instead
Session token in an httpOnly, Secure, SameSite cookie — JS can't read it, browser sends it automatically, built-in CSRF token. Session lookup happens server-side.
Session strategies
session: { strategy: "jwt" }   // stateless, encrypted JWT in cookie (default)
// or
session: { strategy: "database" } // session row in DB, needs an adapter
CredentialsProvider ONLY works with strategy: "jwt".

Install & Environment

Install (v4)
npm install next-auth
Required env vars (.env.local)
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<random-string>
Generate a secret
openssl rand -base64 32
Without NEXTAUTH_SECRET set, JWT sessions fail in production.
Provider secrets
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...

Catch-all API Route ([...nextauth])

Catch-all route concept
[...nextauth] captures ALL sub-paths: /api/auth/signin, /signout, /callback/:provider, /session, /csrf, /providers. One handler serves every auth endpoint.
Pages Router: pages/api/auth/[...nextauth].ts
import NextAuth from "next-auth";
import CredentialsProvider from "next-auth/providers/credentials";

export const authOptions = {
  providers: [ /* ... */ ],
  secret: process.env.NEXTAUTH_SECRET,
};

export default NextAuth(authOptions);
App Router: app/api/auth/[...nextauth]/route.ts
import NextAuth from "next-auth";
import { authOptions } from "@/lib/auth"; // define authOptions separately

const handler = NextAuth(authOptions);
export { handler as GET, handler as POST };
Folder is [...nextauth], file is route.ts. Export handler as BOTH GET and POST.
Give NextAuth access to the catch-all
The [...nextauth] path is literal — the folder/file must be exactly named that so NextAuth's internal router can resolve /api/auth/* to its handlers. Keep authOptions in its own module to reuse with getServerSession.

Providers

Credentials provider (email/password)
import CredentialsProvider from "next-auth/providers/credentials";

CredentialsProvider({
  name: "Credentials",
  credentials: {
    username: { label: "Email", type: "text" },
    password: { label: "Password", type: "password" },
  },
  async authorize(credentials) {
    const user = await db.user.findUnique({ where: { email: credentials.username }});
    if (user && await bcrypt.compare(credentials.password, user.hash)) {
      return { id: user.id, name: user.name, email: user.email };
    }
    return null; // null/throw => login rejected
  },
})
Return object = the user; return null to reject. Requires strategy: "jwt".
Google provider
import GoogleProvider from "next-auth/providers/google";

GoogleProvider({
  clientId: process.env.GOOGLE_CLIENT_ID!,
  clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
})
Authorized redirect URI in Google Console: http://localhost:3000/api/auth/callback/google
GitHub provider
import GithubProvider from "next-auth/providers/github";

GithubProvider({
  clientId: process.env.GITHUB_ID!,
  clientSecret: process.env.GITHUB_SECRET!,
})
GitHub OAuth App callback URL: http://localhost:3000/api/auth/callback/github

Callbacks & Server Session

Add fields to token & session
callbacks: {
  async jwt({ token, user }) {
    if (user) token.uid = user.id; // runs on sign-in
    return token;
  },
  async session({ session, token }) {
    if (session.user) session.user.id = token.uid;
    return session;
  },
}
session callback only exposes what you copy from the token. user is only present on the first jwt call.
Read session on the server (App Router)
import { getServerSession } from "next-auth";
import { authOptions } from "@/lib/auth";

const session = await getServerSession(authOptions);
if (!session) redirect("/signin");
Read session in API route (Pages Router)
import { getServerSession } from "next-auth/next";
const session = await getServerSession(req, res, authOptions);

Client Hooks & Actions

Wrap app in SessionProvider
// Pages: _app.tsx
<SessionProvider session={session}><Component {...pageProps} /></SessionProvider>

// App Router: make a "use client" Providers wrapper around SessionProvider
// then use it in app/layout.tsx
useSession() throws without a SessionProvider ancestor.
useSession
import { useSession } from "next-auth/react";
const { data: session, status } = useSession();
// status: "loading" | "authenticated" | "unauthenticated"
signIn
import { signIn } from "next-auth/react";
signIn("google");                 // OAuth redirect flow
signIn("github");
signIn("credentials", {
  username, password,
  redirect: false,                // handle result yourself
  callbackUrl: "/dashboard",
});
signOut
import { signOut } from "next-auth/react";
signOut({ callbackUrl: "/" });

Custom Pages & Route Protection

Point NextAuth to custom pages
// in authOptions
pages: {
  signIn: "/signin",
  signOut: "/signout",
  error: "/auth/error",   // ?error=CredentialsSignin etc.
}
NextAuth redirects to these instead of its built-in UI.
Custom signup / signin page
"use client";
import { signIn } from "next-auth/react";

export default function SignIn() {
  const [email, setEmail] = useState("");
  const [pw, setPw] = useState("");
  return (
    <form onSubmit={async (e) => {
      e.preventDefault();
      const res = await signIn("credentials", { email, password: pw, redirect: false });
      if (res?.error) alert("Invalid login");
      else router.push("/");
    }}>
      {/* inputs + <button>Sign in</button> */}
    </form>
  );
}
Signup (creating the user) is your own API route + db insert; then call signIn to log them in.
Protect routes via middleware
// middleware.ts (project root)
export { default } from "next-auth/middleware";
export const config = { matcher: ["/dashboard/:path*", "/admin/:path*"] };
Redirects unauthenticated users to the signIn page automatically.

Quick tips & gotchas

  • CredentialsProvider requires session: { strategy: "jwt" } — it does not work with the database strategy.
  • authorize() must return a user object on success and null on failure; a thrown error surfaces as ?error=... on your error page.
  • OAuth callback URLs must match EXACTLY in the provider console: /api/auth/callback/google and /api/auth/callback/github (update for prod domain).
  • Custom session fields must be copied through BOTH callbacks: set them on token in jwt(), then read from token in session().
  • Always set NEXTAUTH_SECRET in production — missing it breaks JWT encryption/verification.
  • useSession() needs a <SessionProvider> ancestor; in the App Router that means a "use client" wrapper component in layout.tsx.
  • In App Router, keep authOptions in its own file (e.g. lib/auth.ts) so both the [...nextauth] route and getServerSession can import it.
  • Use signIn(..., { redirect: false }) for credentials so you can show inline errors instead of a full-page redirect.
