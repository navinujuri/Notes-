Public Key Cryptography — Cheat Sheet

Cohort 2.0 · 10 lessons · quick reference

Banks vs Blockchains (mental model)

Bank = centralized ledger
One trusted party (bank/DB) holds balances; they can freeze, reverse, or edit rows. Auth = password + their say-so.
Blockchain = decentralized ledger
State replicated across nodes; no central admin. 'Ownership' = control of a private key. No password reset — lose the key, lose the funds.
Auth model shift
Banks: prove identity to a server. Blockchain: prove key ownership by signing — verified by anyone, no login.
Address = public identity
Derived from the public key (Solana: base58 of pubkey; Ethereum: last 20 bytes of keccak256(pubkey)). Safe to share, like an account number.

Bits, Bytes & Encodings

Units
1 byte = 8 bits\n1 bit  -> 0 or 1\n1 byte -> 0..255  (2^8 values)\nhex nibble = 4 bits; 1 byte = 2 hex chars
A 256-bit key = 32 bytes = 64 hex chars.
Node Buffer / Uint8Array is the raw byte container
const buf = Buffer.from("hello", "utf8"); // <Buffer 68 65 6c 6c 6f>\nbuf.length; // 5 bytes\nnew Uint8Array([104,101,108,108,111]);
Crypto libs speak bytes (Uint8Array), not strings. Convert explicitly.
utf8 <-> hex
Buffer.from("Hi").toString("hex");          // '4869'\nBuffer.from("4869", "hex").toString("utf8"); // 'Hi'
base64 (data transport, +33% size)
Buffer.from("Hi").toString("base64");        // 'SGk='\nBuffer.from("SGk=", "base64").toString();
base58 (Solana/Bitcoin addresses)
import bs58 from "bs58";\nconst s = bs58.encode(bytes);   // Uint8Array -> string\nconst b = bs58.decode(s);       // string -> Uint8Array
base58 drops ambiguous chars (0/O, I/l) — no line-break issues. Not the same alphabet as base64.

Hashing vs Encryption

Hashing = one-way fingerprint
Deterministic, fixed-length, irreversible. Same input -> same hash. Used for integrity, IDs, block/tx hashes. NOT for hiding recoverable data.
SHA-256 (32-byte digest)
import { createHash } from "crypto";\ncreateHash("sha256").update("data").digest("hex");
keccak-256 (Ethereum's hash)
import { keccak256, toUtf8Bytes } from "ethers";\nkeccak256(toUtf8Bytes("hello")); // '0x1c8a...'
Ethereum uses keccak256, NOT the NIST SHA3-256 — different padding.
Encryption = two-way, reversible with a key
Ciphertext -> plaintext given the key. Symmetric = same key both ways; Asymmetric = key pair.
Symmetric encrypt (AES-256-GCM)
import { createCipheriv, randomBytes } from "crypto";\nconst key = randomBytes(32), iv = randomBytes(12);\nconst c = createCipheriv("aes-256-gcm", key, iv);\nconst ct = Buffer.concat([c.update("msg"), c.final()]);\nconst tag = c.getAuthTag();
Never reuse an (key, iv) pair. GCM's authTag detects tampering.

Asymmetric Encryption & Keypairs

Core idea
Private key = secret (sign/decrypt). Public key = shareable (verify/encrypt). Deriving pub from priv is easy; reverse is infeasible.
Curves by chain
Solana -> ed25519. Ethereum & Bitcoin -> secp256k1. Match the curve to the chain or the address won't be valid.
Node native ed25519 keypair
import { generateKeyPairSync } from "crypto";\nconst { publicKey, privateKey } =\n  generateKeyPairSync("ed25519");
@noble/ed25519 (raw bytes)
import * as ed from "@noble/ed25519";\nconst priv = ed.utils.randomPrivateKey();        // 32 bytes\nconst pub  = await ed.getPublicKeyAsync(priv);   // 32 bytes
Use the *Async fns out of the box; sync fns need ed.hashes.sha512 wired up.
Solana Keypair
import { Keypair } from "@solana/web3.js";\nconst kp = Keypair.generate();\nkp.publicKey.toBase58();   // address\nkp.secretKey;              // Uint8Array(64) = priv(32)+pub(32)
secretKey is 64 bytes (seed 32 + pubkey 32). Store it, never expose it.
Restore from secret
Keypair.fromSecretKey(Uint8Array.from(saved64));\nKeypair.fromSeed(seed32); // seed-only restore

Sign / Verify & Blockchain Transactions

Sign & verify (proves key ownership)
const msg = new TextEncoder().encode("gm");\nconst sig = await ed.signAsync(msg, priv);\nconst ok  = await ed.verifyAsync(sig, msg, pub); // true
Signature proves the holder of priv approved msg — without revealing priv.
Ethereum sign (ethers v6)
import { Wallet } from "ethers";\nconst w = Wallet.createRandom();\nconst sig = await w.signMessage("gm");
Tx flow on chain
1) build tx (from, to, amount, recentBlockhash/nonce) 2) sign with priv 3) broadcast 4) validators verify sig against from-address 5) tx included in a block.
Solana transfer tx
import { Transaction, SystemProgram,\n  Connection, clusterApiUrl } from "@solana/web3.js";\nconst tx = new Transaction().add(\n  SystemProgram.transfer({\n    fromPubkey: kp.publicKey, toPubkey: dest,\n    lamports: 1e9 })); // 1 SOL = 1e9 lamports\ntx.sign(kp);
Amounts are integers in base units (lamports/wei) — never floats.

HD Wallets (BIP-32/39/44)

Why HD
One mnemonic -> deterministic tree of infinite keypairs. Back up 12/24 words once; regenerate every account from it.
Mnemonic (BIP-39)
import * as bip39 from "bip39";\nconst mnemonic = bip39.generateMnemonic();   // 12 words (128-bit)\nbip39.validateMnemonic(mnemonic);
Mnemonic -> seed
const seed = bip39.mnemonicToSeedSync(mnemonic); // 64-byte Buffer
Optional passphrase is a 25th word: mnemonicToSeedSync(m, pass).
Derivation path (BIP-44)
m / 44' / coin' / account' / change / index\nBTC: m/44'/0'/0'/0/0\nETH: m/44'/60'/0'/0/0\nSOL: m/44'/501'/0'/0'
' = hardened index. Solana wallets use all-hardened path (no change/index level).
Derive Solana keypair #i
import { derivePath } from "ed25519-hd-key";\nconst path = `m/44'/501'/${i}'/0'`;\nconst key = derivePath(path, seed.toString("hex")).key;\nconst kp = Keypair.fromSeed(key);
Derive Ethereum wallet (ethers)
import { HDNodeWallet } from "ethers";\nconst w = HDNodeWallet.fromPhrase(\n  mnemonic, undefined, `m/44'/60'/0'/0/${i}`);\nw.address; w.privateKey;

Create a Wallet (end to end)

Solana wallet from scratch
const mnemonic = bip39.generateMnemonic();\nconst seed = bip39.mnemonicToSeedSync(mnemonic);\nconst key = derivePath("m/44'/501'/0'/0'",\n  seed.toString("hex")).key;\nconst wallet = Keypair.fromSeed(key);\nconsole.log(wallet.publicKey.toBase58());
Ethereum wallet from scratch
import { Wallet } from "ethers";\nconst w = Wallet.createRandom();\nw.address; w.privateKey; w.mnemonic.phrase;
Multi-account loop
for (let i = 0; i < 5; i++) {\n  const p = `m/44'/501'/${i}'/0'`;\n  const k = derivePath(p, seed.toString('hex')).key;\n  console.log(i, Keypair.fromSeed(k).publicKey.toBase58());\n}
Bump the account index to get the next address from the same seed.
Install deps
npm i @solana/web3.js ed25519-hd-key bip39 bs58 ethers tweetnacl

Quick tips & gotchas

  • Private key = money. Anyone with it can drain funds instantly; there's no reset, no support desk. Never log, commit, or paste it.
  • Match curve to chain: Solana=ed25519, Ethereum/Bitcoin=secp256k1. A key from the wrong curve produces an invalid/unusable address.
  • Solana secretKey is 64 bytes (32-byte seed + 32-byte pubkey), not 32. Use Keypair.fromSecretKey for the 64-byte blob, fromSeed for the 32-byte seed.
  • Hashing is one-way — don't reach for it to 'hide' recoverable data; use encryption. Encryption is reversible with a key; hashing never is.
  • Amounts are integer base units: lamports (1 SOL = 1e9) and wei (1 ETH = 1e18). Floats cause rounding bugs and rejected txns.
  • The mnemonic IS the wallet — same 12/24 words regenerate every key. Order matters; a passphrase acts as a hidden 25th word that changes the whole tree.
  • ' in a derivation path means hardened. Solana uses an all-hardened path (m/44'/501'/i'/0'); Ethereum uses m/44'/60'/0'/0/i — copy them exactly.
  • Ethereum uses keccak256, not SHA3-256 (different padding) — using the wrong one yields wrong hashes/addresses. Address = last 20 bytes of keccak256(pubkey).
