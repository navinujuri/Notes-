Ganache, Trufle, Hardhat and Foundry — Cheat Sheet

Cohort 3.0 · 12 lessons · quick reference

Ganache & Truffle (legacy stack)

Install (both deprecated Mar 2023 — know for legacy repos)
npm i -g truffle ganache
Start local chain (10 funded accounts, chainId 1337)
ganache            # CLI, RPC on http://127.0.0.1:8545
ganache -d         # deterministic mnemonic (same keys every run)
Scaffold a Truffle project
truffle init       # contracts/, migrations/, test/, truffle-config.js
Compile / test / migrate
truffle compile
truffle test
truffle migrate --network development
truffle-config.js network block
networks: {
  development: { host: '127.0.0.1', port: 8545, network_id: '*' }
}
network_id '*' matches any chain; Ganache defaults to 1337/5777
Interactive console against a network
truffle console --network development

Hardhat (JS/TS ecosystem)

Init a project
mkdir myapp && cd myapp
npm init -y
npm i -D hardhat
npx hardhat init      # pick TypeScript + toolbox
Compile / test / clean
npx hardhat compile
npx hardhat test
npx hardhat clean
Run the built-in in-process node (replaces Ganache)
npx hardhat node     # 20 funded accounts, RPC :8545, chainId 31337
Run a script against a network
npx hardhat run scripts/deploy.ts --network localhost
hardhat.config.ts
import '@nomicfoundation/hardhat-toolbox';
export default {
  solidity: '0.8.24',
  networks: {
    sepolia: { url: process.env.RPC_URL, accounts: [process.env.PK!] }
  }
};
console.log inside Solidity (debug)
import 'hardhat/console.sol';
console.log('bal: %s', balance);
Only works on the Hardhat network, strip before mainnet deploy

Foundry — install & scaffold

Install via foundryup (Linux/macOS/WSL)
curl -L https://foundry.paradigm.xyz | bash
foundryup            # installs forge, cast, anvil, chisel
On Windows use WSL; foundryup keeps the toolchain updated
New project
forge init hello_foundry
# src/  test/  script/  lib/  foundry.toml
Build / test / format
forge build
forge test
forge fmt
Local node (Ganache/Hardhat-node equivalent)
anvil                # RPC :8545, chainId 31337, 10 funded accts
foundry.toml essentials
[profile.default]
src = 'src'
out = 'out'
libs = ['lib']
solc = '0.8.24'
remappings = ['@openzeppelin/=lib/openzeppelin-contracts/']

External packages & remappings

Install a dependency (git submodule under lib/)
forge install OpenZeppelin/openzeppelin-contracts
forge install foundry-rs/forge-std
Pin a version / tag
forge install OpenZeppelin/openzeppelin-contracts@v5.1.0
Update / remove
forge update
forge remove OpenZeppelin/openzeppelin-contracts
Auto-generate remappings
forge remappings > remappings.txt
Import then resolves e.g. `@openzeppelin/contracts/...` to lib/
Hardhat equivalent (npm, not submodules)
npm i @openzeppelin/contracts
Hardhat imports from node_modules; no remappings needed

First contract & ERC-20

Minimal contract (src/Counter.sol)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;
contract Counter {
    uint256 public number;
    function set(uint256 n) public { number = n; }
    function inc() public { number++; }
}
ERC-20 via OpenZeppelin (src/MyToken.sol)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;
import {ERC20} from '@openzeppelin/contracts/token/ERC20/ERC20.sol';
contract MyToken is ERC20 {
    constructor(uint256 supply) ERC20('MyToken', 'MTK') {
        _mint(msg.sender, supply);
    }
}
Standard ERC-20 surface (from base)
totalSupply() balanceOf(a) transfer(to,amt)
approve(spender,amt) allowance(o,s) transferFrom(from,to,amt)
decimals() defaults to 18; override it to change
Mint / burn helpers
_mint(to, amount);
_burn(from, amount);
Internal — wrap in access-controlled public fns (e.g. Ownable)

Testing in Foundry (forge-std)

Test file layout (test/MyToken.t.sol)
import {Test} from 'forge-std/Test.sol';
contract MyTokenTest is Test {
    MyToken t;
    function setUp() public { t = new MyToken(1000e18); }
    function test_Supply() public view {
        assertEq(t.totalSupply(), 1000e18);
    }
}
setUp() runs before EACH test; fns prefixed test_ are cases
Common cheatcodes (vm.*)
vm.prank(alice);          // next call from alice
vm.startPrank(alice); ... vm.stopPrank();
vm.deal(alice, 1 ether);  // set ETH balance
vm.expectRevert(bytes('ERR'));
address bob = makeAddr('bob');
Test an emitted event
event Transfer(address indexed from, address indexed to, uint256 v);
function test_Emit() public {
    vm.expectEmit(true, true, false, true);
    emit Transfer(address(this), bob, 10);   // expected event
    t.transfer(bob, 10);                       // triggering call
}
expectEmit flags = (topic1, topic2, topic3, data); emit the expected event, THEN make the call
Assertions & fuzzing
assertEq(a,b); assertTrue(x); assertGt(a,b);
function testFuzz_Transfer(uint96 amt) public { ... }
Fn args = fuzzed inputs; use vm.assume() to bound them
Verbosity & gas
forge test -vvv           # show traces on failure
forge test --match-test test_Emit
forge test --gas-report

Deploying contracts

Foundry deploy script (script/Deploy.s.sol)
import {Script} from 'forge-std/Script.sol';
contract Deploy is Script {
    function run() external {
        vm.startBroadcast();
        new MyToken(1_000_000e18);
        vm.stopBroadcast();
    }
}
Run the script (broadcast + verify)
forge script script/Deploy.s.sol \
  --rpc-url $RPC_URL --private-key $PK \
  --broadcast --verify
Drop --broadcast for a dry-run simulation; --verify needs ETHERSCAN_API_KEY
One-liner deploy (no script)
forge create src/MyToken.sol:MyToken \
  --rpc-url $RPC_URL --private-key $PK \
  --constructor-args 1000000000000000000000000
cast — chain interaction from CLI
cast call $ADDR 'totalSupply()(uint256)' --rpc-url $RPC_URL
cast send $ADDR 'transfer(address,uint256)' $TO 10 --private-key $PK
cast balance $ADDR
Hardhat deploy (ethers v6)
const T = await ethers.getContractFactory('MyToken');
const t = await T.deploy(ethers.parseEther('1000000'));
await t.waitForDeployment();
console.log(await t.getAddress());

Quick tips & gotchas

  • Ganache + Truffle are deprecated (ConsenSys sunset Mar 2023) — new projects use Foundry or Hardhat; only touch Truffle for legacy maintenance.
  • Default local chainIds differ: Ganache 1337, Anvil & Hardhat node 31337 — a wrong chainId in your wallet/config is the #1 'nonce/connection' bug.
  • Foundry deps are git submodules in lib/, not npm packages — after `git clone` run `forge install` (or `git submodule update --init`) or imports won't resolve.
  • vm.expectEmit(t1,t2,t3,data): emit the EXPECTED event first, then call the function that emits the real one — order matters and mismatched indexed topics fail silently-looking.
  • vm.prank sets msg.sender for only the NEXT call; use startPrank/stopPrank to span multiple calls.
  • Never hardcode private keys in scripts or git — load from env (`$PK`) or use `cast wallet` / keystore; Anvil's printed test keys are public and mainnet-unsafe.
  • forge test -vvvv gives full call traces incl. reverts; ratchet verbosity up only when a test fails to keep output readable.
  • Use `_mint`/`_burn` behind access control (Ownable/AccessControl) — leaving a public mint open is a classic ERC-20 exploit.
