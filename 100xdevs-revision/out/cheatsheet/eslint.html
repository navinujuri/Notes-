ESLint, Prettier and Pre-commit hooks — Cheat Sheet

Cohort 2.0 · 9 lessons · quick reference

Linting 101 & ESLint setup

Linting = static analysis that flags bugs, bad patterns & style before runtime (unused vars, no-undef, ==). Formatting = whitespace/quotes/semicolons (Prettier's job).
Lint = correctness; format = cosmetics. Keep them separate.
Install + scaffold config
npm i -D eslint
npm init @eslint/config@latest   # interactive; writes eslint.config.js
Run
npx eslint .            # lint cwd
npx eslint . --fix      # auto-fix fixable rules
npx eslint src --max-warnings=0   # CI: fail on any warning
package.json scripts
"scripts": {
  "lint": "eslint .",
  "lint:fix": "eslint . --fix"
}

Flat config (eslint.config.js — ESLint v9+ default)

Base JS config — array of config objects, first-match-wins merge
// eslint.config.js
import js from "@eslint/js";
import globals from "globals";

export default [
  js.configs.recommended,
  {
    files: ["**/*.{js,mjs,cjs}"],
    languageOptions: {
      ecmaVersion: "latest",
      sourceType: "module",
      globals: { ...globals.browser, ...globals.node },
    },
    rules: {
      "no-unused-vars": "warn",
      eqeqeq: ["error", "always"],
    },
  },
];
Global ignore = an object with ONLY an `ignores` key
{ ignores: ["dist/**", "coverage/**", "**/*.min.js"] }
.eslintignore is gone in flat config — ignores live here. node_modules & dotfiles ignored by default.
Legacy .eslintrc (v8 / opt-in)
// .eslintrc.json
{
  "root": true,
  "extends": ["eslint:recommended"],
  "parserOptions": { "ecmaVersion": "latest", "sourceType": "module" },
  "env": { "browser": true, "node": true }
}
Force legacy in v9: ESLINT_USE_FLAT_CONFIG=false. Removed entirely in v10.

Rules, severity & disabling

Severity: "off"/0, "warn"/1, "error"/2 (nonzero exit)
rules: {
  "no-console": "off",
  "no-debugger": "warn",
  "prefer-const": "error",
  quotes: ["error", "single"],   // rule + options tuple
}
Disable inline
// eslint-disable-next-line no-console
console.log(x);
alert(1); // eslint-disable-line no-alert

/* eslint-disable no-unused-vars */
// ...block...
/* eslint-enable no-unused-vars */
File-level directive
/* eslint-disable */          // whole file
/* eslint no-console: "error" */  // override one rule for file
Report unused disables (catch stale suppressions)
npx eslint . --report-unused-disable-directives

React & TypeScript

TypeScript via typescript-eslint helper
npm i -D typescript-eslint

// eslint.config.js
import tseslint from "typescript-eslint";
export default tseslint.config(
  js.configs.recommended,
  ...tseslint.configs.recommended,
);
Type-aware rules need languageOptions.parserOptions.projectService: true (slower).
React + hooks (flat)
import react from "eslint-plugin-react";
import hooks from "eslint-plugin-react-hooks";

{
  files: ["**/*.{jsx,tsx}"],
  plugins: { react, "react-hooks": hooks },
  settings: { react: { version: "detect" } },
  languageOptions: { parserOptions: { ecmaFeatures: { jsx: true } } },
  rules: {
    ...react.configs.flat.recommended.rules,
    ...hooks.configs.recommended.rules,
    "react/react-in-jsx-scope": "off",
    "react/prop-types": "off",
  },
}
react-in-jsx-scope off for React 17+ new JSX transform. Vite adds eslint-plugin-react-refresh.

Next.js

create-next-app installs eslint-config-next automatically
npx next lint        # deprecated in Next 16 — call eslint directly instead
Flat config with FlatCompat (Next 15 default)
// eslint.config.mjs
import { FlatCompat } from "@eslint/eslintrc";
const compat = new FlatCompat({ baseDirectory: import.meta.dirname });
export default [
  ...compat.extends("next/core-web-vitals", "next/typescript"),
];
core-web-vitals = stricter perf rules; next/typescript adds TS rules.

Prettier + ESLint integration

Install & run
npm i -D prettier
npx prettier --write .    # format in place
npx prettier --check .    # CI: fail if unformatted
.prettierrc.json + .prettierignore
{
  "semi": true,
  "singleQuote": true,
  "trailingComma": "all",
  "printWidth": 80,
  "tabWidth": 2
}
.prettierignore mirrors .gitignore syntax (dist, build, *.md).
eslint-config-prettier — turns OFF ESLint rules that fight Prettier
npm i -D eslint-config-prettier

// eslint.config.js — MUST be LAST
import prettier from "eslint-config-prettier";
export default [ js.configs.recommended, /* ... */, prettier ];
Recommended approach: ESLint lints, Prettier formats, config-prettier keeps them from overlapping.
eslint-plugin-prettier — runs Prettier AS a lint rule (optional)
// reports formatting as "prettier/prettier" errors
rules: { "prettier/prettier": "error" }
Convenient but slower & noisier; large repos usually run prettier standalone instead.

VSCode & popular shared configs

Extensions
dbaeumer.vscode-eslint
esbenp.prettier-vscode
.vscode/settings.json — format + fix on save
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  }
}
"explicit" replaced the old boolean true in recent VSCode ESLint versions.
Airbnb config
npx install-peerdeps --dev eslint-config-airbnb
// extends: "airbnb", "airbnb/hooks"
Very opinionated. Flat-config support lags — use eslint-config-airbnb-extended for v9.
Standard / others
npm i -D eslint-config-standard   // no-semi, 2-space
// or Biome — single fast Rust tool replacing ESLint+Prettier
npx @biomejs/biome check --write .

Pre-commit hooks (Husky + lint-staged)

Install & init Husky v9
npm i -D husky lint-staged
npx husky init   # adds "prepare":"husky" + creates .husky/pre-commit
prepare script auto-installs hooks on npm install for teammates.
.husky/pre-commit (v9 — no shebang/husky.sh needed)
npx lint-staged
lint-staged config (package.json) — runs only on STAGED files
"lint-staged": {
  "*.{js,jsx,ts,tsx}": ["eslint --fix", "prettier --write"],
  "*.{json,md,css,html}": ["prettier --write"]
}
lint-staged appends staged filenames to each command — don't pass your own globs (would re-lint the whole repo).
Alternative: Lefthook (fast, parallel, language-agnostic)
# lefthook.yml
pre-commit:
  parallel: true
  commands:
    lint:
      glob: "*.{js,ts,jsx,tsx}"
      run: npx eslint {staged_files} --fix
      stage_fixed: true
Bypass a hook (emergencies only)
git commit --no-verify -m "wip"

Quick tips & gotchas

  • ESLint v9 defaults to flat config (eslint.config.js). .eslintrc + .eslintignore are deprecated and dropped in v10 — migrate or pin v8.
  • Order matters: put eslint-config-prettier (and any *-config-prettier) LAST in the flat array so it can disable earlier formatting rules.
  • A flat-config object with ONLY `ignores` is a global ignore; ignores alongside other keys only scopes THAT block.
  • Lint vs format: let Prettier own whitespace/quotes/commas and ESLint own logic. Avoid eslint-plugin-prettier on big repos — run `prettier --write` separately for speed.
  • React 17+ new JSX transform → turn off react/react-in-jsx-scope and react/jsx-uses-react (false positives otherwise).
  • Type-aware TS rules require parserOptions.projectService/project; they're much slower, so scope them to source files only.
  • In CI use `eslint --max-warnings=0` and `prettier --check .` so warnings and unformatted code fail the build.
  • Husky v9 hook files are plain scripts — the old `#!/bin/sh` + `. "$(dirname)/_/husky.sh"` header is gone; `next lint` is deprecated in Next 16, call eslint directly.
