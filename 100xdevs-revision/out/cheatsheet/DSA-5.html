C++ Part 1 — Cheat Sheet

Cohort 2.0 · 8 lessons · quick reference

Setup & Compilation

Install compiler (Linux/macOS)
sudo apt install g++      # Debian/Ubuntu
xcode-select --install     # macOS (clang++)
g++ = GNU, clang++ = LLVM; both accept the same flags below.
Install on Windows
# MSYS2 / MinGW-w64
pacman -S mingw-w64-ucrt-x86_64-gcc
Add the ...\ucrt64\bin dir to PATH so g++ resolves.
Compile a single file
g++ main.cpp -o main
./main            # .\main.exe on Windows
No -o -> a.out (a.exe). Compiles AND links in one step.
Recommended dev flags
g++ -std=c++17 -Wall -Wextra -g main.cpp -o main
-std pins the standard; -Wall -Wextra surface bugs; -g adds debug symbols.
Optimize for release
g++ -std=c++17 -O2 main.cpp -o main
-O2 typical; -O3 aggressive. Don't mix -g and -O2 when debugging.
Check version
g++ --version

Program Structure & I/O

Minimal program
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl;
    return 0;   // 0 = success
}
main MUST return int. Falling off main implicitly returns 0.
Output with cout
cout << "x = " << x << ", y = " << y << "\n";
<< chains left-to-right. Prefer \n over endl in loops (endl flushes = slow).
Read input with cin
int age; string name;
cin >> age >> name;
>> skips leading whitespace and stops at the next whitespace.
Read a whole line
string line;
getline(cin, line);
After cin >> x, a newline stays in the buffer — use cin.ignore() before getline.
Avoid namespace pollution
std::cout << std::endl;
`using namespace std;` is fine in small programs; avoid in headers.
Comments
// single line
/* multi
   line */

Data Types & Variables

Integer types
int a = 42;          // usually 4 bytes
long long big = 9e18; // 8 bytes, ~9.2e18 max
unsigned int u = 3u; // no negatives
Use long long for values > ~2.1e9 to avoid overflow.
Floating point
float f = 3.14f;   // ~7 digits
double d = 3.14159265; // ~15 digits
Prefer double; float literals need the f suffix.
Character & bool
char c = 'A';   // single quotes, 1 byte
bool ok = true; // true/false -> 1/0
"A" (double quotes) is a C-string, not a char.
String (C++ std)
#include <string>
string s = "hello";
s += " world";
cout << s.size() << s[0];
std::string is dynamic; std::string is NOT included by <iostream> reliably — include <string>.
const & auto
const double PI = 3.14159;  // immutable
auto n = 10;                // deduced -> int
auto needs an initializer; const must be initialized at declaration.
Sizes & casting
cout << sizeof(int);           // bytes
double r = (double)a / b;      // C-style
int x = static_cast<int>(3.9); // -> 3 (truncates)
Int/int division truncates: 3/2 == 1. Cast one operand to get 1.5.

Conditionals

if / else if / else
if (score >= 90) grade = 'A';
else if (score >= 80) grade = 'B';
else grade = 'F';
Condition must be in (). Braces optional for single statements but recommended.
Logical & comparison ops
if (a > 0 && b != 0) ...
if (x < 0 || y < 0) ...
if (!done) ...
&& / || short-circuit: RHS skipped if result already known.
Ternary operator
int max = (a > b) ? a : b;
Great for simple assignments; don't nest deeply.
switch
switch (day) {
    case 1: cout << "Mon"; break;
    case 2: cout << "Tue"; break;
    default: cout << "?";
}
Missing break = fall-through. Only works on int/char/enum, not strings/doubles.
Common == vs = bug
if (x == 5) ...  // compare
x = 5;           // assign
if (x = 5) assigns then tests truthiness — always true here. -Wall warns.

Loops

for loop
for (int i = 0; i < n; i++) {
    cout << i << " ";
}
init; condition; update. i declared in loop is scoped to it.
while loop
int i = 0;
while (i < n) { cout << i; i++; }
Condition checked before body — may run 0 times.
do-while loop
int i = 0;
do { cout << i; i++; } while (i < n);
Body runs at least once (condition checked after). Note trailing semicolon.
Range-based for (C++11)
int arr[] = {1,2,3};
for (int x : arr) cout << x;
for (auto& x : arr) x *= 2;  // modify in place
Use auto& to mutate or avoid copies of large elements.
break & continue
for (int i = 0; i < n; i++) {
    if (i == target) break;    // exit loop
    if (i % 2) continue;       // skip to next iter
}
Nested loops
for (int i = 0; i < r; i++)
  for (int j = 0; j < c; j++)
    cout << grid[i][j];
break/continue affect only the innermost loop.

Functions

Definition & call
int add(int a, int b) {
    return a + b;
}
int s = add(2, 3);   // 5
Return type before name; void = returns nothing.
Prototype (declare before use)
int add(int, int);   // declaration
int main() { add(2,3); }
int add(int a,int b){ return a+b; }
Needed if the definition appears after the call site.
Pass by value vs reference
void inc(int x)  { x++; }   // copy, no effect
void inc(int& x) { x++; }   // modifies caller's var
Use & to modify the argument or avoid copying large objects.
const reference (read-only, cheap)
void print(const string& s) { cout << s; }
Idiomatic for passing big objects you won't modify.
Default arguments
int pow2(int base, int exp = 2) { ... }
pow2(5);      // exp=2
pow2(5, 3);   // exp=3
Defaults go in the declaration only; must be trailing params.
Overloading
int  area(int s)            { return s*s; }
double area(double w,double h){ return w*h; }
Same name, different parameter types/count. Return type alone can't distinguish.

Arrays

Declare & initialize
int a[5];               // uninitialized
int b[5] = {1,2,3};     // rest = 0
int c[] = {1,2,3,4};    // size deduced = 4
Fixed size must be a compile-time constant.
Access & bounds
a[0] = 10;
cout << a[4];  // last valid index for size 5
Indices 0..size-1. Out-of-bounds is undefined behavior (no bounds check!).
Length of a stack array
int n = sizeof(a) / sizeof(a[0]);
Works only where the array's full type is visible — NOT after passing to a function.
Iterate
for (int i = 0; i < n; i++) cout << a[i];
for (int x : a) cout << x;   // range-based
Pass to function
void fill(int arr[], int n) { ... }
fill(a, 5);
Arrays decay to pointers — pass the size separately; sizeof inside gives pointer size.
2D array
int m[2][3] = {{1,2,3},{4,5,6}};
cout << m[1][2];  // 6
Row-major. All dimensions except the first are required in params: void f(int m[][3]).
Prefer std::vector for dynamic size
#include <vector>
vector<int> v = {1,2,3};
v.push_back(4);
cout << v.size();
Resizable, knows its own size, safer than raw arrays.

Quick tips & gotchas

  • Integer division truncates: 7/2 == 3. Cast an operand to double for real division: (double)7/2.
  • Use long long for sums/products that can exceed ~2.1e9; int silently overflows (undefined behavior).
  • Uninitialized local variables and arrays hold garbage — always initialize before reading.
  • Prefer \n over std::endl inside loops; endl forces a buffer flush and is much slower.
  • In switch, a missing break causes fall-through into the next case — a classic silent bug.
  • Arrays passed to functions decay to pointers, so sizeof(arr) no longer gives the array size — always pass the length too.
  • Pass large objects (string, vector) by const& to avoid expensive copies; use & when you need to modify the caller's value.
  • Compile with -Wall -Wextra: it catches == vs =, unused vars, and signed/unsigned mismatches before they bite.
