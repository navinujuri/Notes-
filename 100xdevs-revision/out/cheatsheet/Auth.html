Authentication — Cheat Sheet

Cohort 2.0 · 8 lessons · quick reference

Concepts & Setup

Authentication vs Authorization
AuthN = who are you (login). AuthZ = what can you do (roles/permissions). Different concerns.
Install the usual stack
npm i express jsonwebtoken bcrypt cookie-parser cors
Never store plaintext passwords — hash with bcrypt
const bcrypt = require("bcrypt");
const hash = await bcrypt.hash(password, 10); // salt rounds
const ok   = await bcrypt.compare(password, hash); // -> bool
bcrypt auto-salts. Never use md5/sha256 for passwords.
Keep the secret out of code
// .env
JWT_SECRET=super-long-random-string
// usage: process.env.JWT_SECRET
Leaking the secret = anyone can forge valid tokens.

JWT Basics

Sign a token (login success)
const jwt = require("jsonwebtoken");
const token = jwt.sign(
  { id: user._id },              // payload
  process.env.JWT_SECRET,
  { expiresIn: "1h" }
);
Verify a token (protected route)
const decoded = jwt.verify(token, process.env.JWT_SECRET);
// decoded.id  -> throws if invalid/expired
Structure: header.payload.signature
3 base64url parts joined by '.'. Payload is only ENCODED, not encrypted — anyone can read it. Never put secrets/passwords in it.
Decode without verifying (debug only)
jwt.decode(token); // no signature check — never trust for auth

Approach 1 — JWT + localStorage

Store token after login (frontend)
localStorage.setItem("token", token);
Send on every request via Authorization header
const token = localStorage.getItem("token");
axios.get("/me", {
  headers: { Authorization: "Bearer " + token }
});
Convention is the 'Bearer <token>' prefix.
Set a default header once
axios.defaults.headers.common["Authorization"] = "Bearer " + token;
Logout = remove it
localStorage.removeItem("token");
Tradeoff
Simple, works cross-domain/mobile. But JS-readable → any XSS can steal it. Not auto-sent by browser.

Approach 2 — Cookies

Set an httpOnly cookie on login
res.cookie("token", token, {
  httpOnly: true,   // JS can't read via document.cookie
  secure: true,     // HTTPS only
  sameSite: "lax",  // CSRF mitigation
  maxAge: 3600000   // ms (1h)
});
Read cookies with cookie-parser
const cookieParser = require("cookie-parser");
app.use(cookieParser());
// later:
const token = req.cookies.token;
Clear on logout
res.clearCookie("token");
Tradeoff
Auto-sent by browser every request; httpOnly blocks XSS token theft. But vulnerable to CSRF → use sameSite + CSRF tokens.

Cookie Properties (cheat)

httpOnly
Blocks document.cookie access from JS. Defeats XSS token exfiltration.
secure
Cookie only sent over HTTPS. Required when sameSite: 'none'.
sameSite
sameSite: "strict" | "lax" | "none"
strict = never cross-site; lax = sent on top-level GET nav (default-ish); none = always but needs secure.
maxAge / expires
maxAge: 3600000            // relative ms
expires: new Date(Date.now()+3600000)
No maxAge/expires = session cookie (dies on browser close).
domain / path
domain: ".example.com", path: "/"
Controls scope — which hosts/routes the cookie is sent to.
signed cookies
app.use(cookieParser("secret"));
res.cookie("t", v, { signed: true });
req.signedCookies.t // tamper-detected
Signs value so client tampering is detectable.

Express Backend — full example

Signup: hash + store
app.post("/signup", async (req, res) => {
  const { email, password } = req.body;
  const hash = await bcrypt.hash(password, 10);
  await User.create({ email, password: hash });
  res.json({ msg: "created" });
});
Signin: compare + issue token
app.post("/signin", async (req, res) => {
  const user = await User.findOne({ email: req.body.email });
  if (!user || !(await bcrypt.compare(req.body.password, user.password)))
    return res.status(401).json({ msg: "bad creds" });
  const token = jwt.sign({ id: user._id }, process.env.JWT_SECRET);
  // header approach:
  res.json({ token });
  // OR cookie approach:
  // res.cookie("token", token, { httpOnly: true }).json({ msg: "ok" });
});
Auth middleware (header)
function auth(req, res, next) {
  const token = (req.headers.authorization || "").split(" ")[1];
  if (!token) return res.status(401).json({ msg: "no token" });
  try {
    req.userId = jwt.verify(token, process.env.JWT_SECRET).id;
    next();
  } catch {
    res.status(403).json({ msg: "invalid token" });
  }
}
Auth middleware (cookie variant)
const token = req.cookies.token; // rest identical
Protect a route
app.get("/me", auth, (req, res) => res.json({ id: req.userId }));

Frontend (React) + CORS

CORS must allow credentials for cookies
app.use(cors({
  origin: "http://localhost:5173", // exact origin, not *
  credentials: true
}));
credentials:true + wildcard origin is rejected by browsers.
Send cookies from React
axios.get("/me", { withCredentials: true });
// or global: axios.defaults.withCredentials = true;
fetch equivalent: credentials: 'include'.
Login flow (header/localStorage)
const { data } = await axios.post("/signin", { email, password });
localStorage.setItem("token", data.token);
Protect a page
useEffect(() => {
  axios.get("/me", { headers: { Authorization: "Bearer " + localStorage.getItem("token") } })
    .catch(() => navigate("/login"));
}, []);

Quick tips & gotchas

  • JWT payload is base64-encoded, NOT encrypted — decode it on jwt.io. Never store passwords/secrets in the payload.
  • localStorage token = exposed to XSS (any injected JS reads it). httpOnly cookie = exposed to CSRF instead. Pick your threat and mitigate it (sameSite/CSRF tokens for cookies, strict CSP for localStorage).
  • Always set expiresIn on tokens — a stolen token with no expiry is valid forever.
  • For cookie auth cross-origin: backend needs cors({ credentials:true, origin:<exact> }) AND frontend needs withCredentials:true. Both or it silently fails.
  • sameSite:'none' REQUIRES secure:true — browsers drop it otherwise. Localhost over http can bite you here.
  • Use bcrypt.compare (constant-time), never === on hashes; never hash passwords with md5/sha.
  • Return 401 for 'not authenticated' (no/bad token) and 403 for 'authenticated but not allowed'.
  • Keep JWT_SECRET in env, long and random; rotating it invalidates all existing tokens.
