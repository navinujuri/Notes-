Custom Hooks — Cheat Sheet

Cohort 2.0 · 5 lessons · quick reference

Hooks fundamentals & rules

Built-in hooks you'll compose
const [v, setV] = useState(0);       // local state
useEffect(() => {...}, [deps]);       // side effects
const ref = useRef(null);             // mutable box / DOM
const memo = useMemo(() => calc, [d]);// cache value
const cb = useCallback(fn, [d]);      // cache fn identity
const ctx = useContext(MyContext);    // read provider
Hooks are functions starting with `use` that let function components hook into React state/lifecycle.
Rules of Hooks (enforced by eslint-plugin-react-hooks)
// Call at the TOP LEVEL only:
if (cond) useState();   // BAD - conditional
for (...) useEffect();   // BAD - loop
function handler() { useState(); } // BAD - nested fn
Only call hooks at the top level of a component OR another hook. Same order every render — React tracks them by call index.
Effect cleanup runs before re-run & on unmount
useEffect(() => {
  const id = setInterval(tick, 1000);
  return () => clearInterval(id); // cleanup
}, []);
Return a function to tear down subscriptions/timers/listeners. StrictMode double-invokes effects in dev to surface missing cleanup.

Anatomy of a custom hook

A custom hook = a function named useX that calls other hooks
function useToggle(initial = false) {
  const [on, setOn] = useState(initial);
  const toggle = useCallback(() => setOn(o => !o), []);
  return [on, toggle];
}

// usage
const [isOpen, toggleOpen] = useToggle();
It's just extraction/reuse of stateful logic. Each call gets its OWN isolated state — hooks share logic, not state.
Return shape: array for 2 values, object for many
return [value, setValue];              // destructure w/ any name
return { data, loading, error, refetch }; // self-documenting
Arrays let callers rename; objects avoid positional mistakes when returning 3+ things.
Accept + forward deps so effects stay reactive
function useTitle(title) {
  useEffect(() => { document.title = title; }, [title]);
}
Whatever the hook uses from arguments/state must be in the dep array — treat the hook like a mini-component.

Data fetching hooks

useFetch — data/loading/error triad with abort
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  useEffect(() => {
    const ctrl = new AbortController();
    setLoading(true);
    fetch(url, { signal: ctrl.signal })
      .then(r => { if (!r.ok) throw new Error(r.status); return r.json(); })
      .then(setData)
      .catch(e => { if (e.name !== 'AbortError') setError(e); })
      .finally(() => setLoading(false));
    return () => ctrl.abort();
  }, [url]);
  return { data, loading, error };
}
AbortController cancels the in-flight request when `url` changes or the component unmounts — prevents setting state on stale/unmounted responses.
Add refetch via a manual trigger
const [tick, setTick] = useState(0);
const refetch = () => setTick(t => t + 1);
useEffect(() => { /* fetch */ }, [url, tick]);
return { data, loading, error, refetch };
useDebounce — debounce fast-changing search input
function useDebounce(value, delay = 500) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(id); // reset timer on each keystroke
  }, [value, delay]);
  return debounced;
}
// const results = useFetch(`/api?q=${useDebounce(query)}`)
Cleanup clears the previous timeout so the effect only fires after the value stops changing for `delay` ms.

Browser functionality hooks

useLocalStorage — state synced to localStorage
function useLocalStorage(key, initial) {
  const [val, setVal] = useState(() => {
    const s = localStorage.getItem(key);
    return s ? JSON.parse(s) : initial;
  });
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(val));
  }, [key, val]);
  return [val, setVal];
}
Lazy initializer `() => ...` reads storage only once. Wrap getItem in try/catch for SSR / private-mode / bad JSON.
useWindowSize — track viewport on resize
function useWindowSize() {
  const [size, setSize] = useState({ w: innerWidth, h: innerHeight });
  useEffect(() => {
    const onResize = () => setSize({ w: innerWidth, h: innerHeight });
    window.addEventListener('resize', onResize);
    return () => window.removeEventListener('resize', onResize);
  }, []);
  return size;
}
Always remove the listener in cleanup or you leak on every mount. Consider throttling onResize (see timer section).
useMediaQuery — reactive breakpoints
function useMediaQuery(query) {
  const [match, setMatch] = useState(() => matchMedia(query).matches);
  useEffect(() => {
    const mql = matchMedia(query);
    const h = e => setMatch(e.matches);
    mql.addEventListener('change', h);
    return () => mql.removeEventListener('change', h);
  }, [query]);
  return match;
}
// const isDark = useMediaQuery('(prefers-color-scheme: dark)')
useOnlineStatus / useClickOutside
// online
const [online, setOnline] = useState(navigator.onLine);
useEffect(() => {
  const on = () => setOnline(true), off = () => setOnline(false);
  addEventListener('online', on); addEventListener('offline', off);
  return () => { removeEventListener('online', on); removeEventListener('offline', off); };
}, []);

// click outside
function useClickOutside(ref, cb) {
  useEffect(() => {
    const h = e => { if (ref.current && !ref.current.contains(e.target)) cb(); };
    document.addEventListener('mousedown', h);
    return () => document.removeEventListener('mousedown', h);
  }, [ref, cb]);
}

Performance / Timer hooks

useInterval — declarative setInterval (Dan Abramov pattern)
function useInterval(callback, delay) {
  const saved = useRef(callback);
  useEffect(() => { saved.current = callback; }, [callback]);
  useEffect(() => {
    if (delay == null) return;            // pause when null
    const id = setInterval(() => saved.current(), delay);
    return () => clearInterval(id);
  }, [delay]);
}
The ref keeps the latest callback without resetting the interval each render. Pass `delay = null` to pause.
useTimeout — same ref trick, one-shot
function useTimeout(callback, delay) {
  const saved = useRef(callback);
  useEffect(() => { saved.current = callback; }, [callback]);
  useEffect(() => {
    if (delay == null) return;
    const id = setTimeout(() => saved.current(), delay);
    return () => clearTimeout(id);
  }, [delay]);
}
useThrottle — cap update frequency (scroll/resize)
function useThrottle(value, limit = 200) {
  const [throttled, setThrottled] = useState(value);
  const last = useRef(Date.now());
  useEffect(() => {
    const now = Date.now();
    const remaining = limit - (now - last.current);
    if (remaining <= 0) { last.current = now; setThrottled(value); return; }
    const id = setTimeout(() => { last.current = Date.now(); setThrottled(value); }, remaining);
    return () => clearTimeout(id);
  }, [value, limit]);
  return throttled;
}
Throttle = at most once per interval (steady stream). Debounce = only after activity stops. Pick throttle for scroll, debounce for search.
usePrevious — remember last render's value
function usePrevious(value) {
  const ref = useRef();
  useEffect(() => { ref.current = value; });
  return ref.current; // returns value from previous render
}

Quick tips & gotchas

  • Name every custom hook `useSomething` — the `use` prefix is what lets the linter apply the Rules of Hooks and lets you call other hooks inside.
  • Custom hooks share LOGIC, not STATE. Two components calling the same hook each get fully independent state — there's no magic global.
  • Every value from props/state used inside an effect must be in its dependency array. Silencing eslint-plugin-react-hooks warnings hides stale-closure bugs.
  • Always return a cleanup function from effects that add listeners, timers, or subscriptions — otherwise you leak and get 'setState on unmounted component' warnings.
  • For data fetching, use AbortController to cancel stale requests; otherwise a slow earlier response can overwrite a newer one (race condition).
  • Timer/interval hooks: stash the callback in a useRef and update it in a separate effect, so the interval isn't torn down and recreated every render.
  • Debounce for 'wait until user stops' (search boxes); throttle for 'run at most every N ms' (scroll, resize, mousemove). They are not interchangeable.
  • Prefer battle-tested libs (usehooks-ts, react-use, @uidotdev/usehooks, or TanStack Query for fetching) over hand-rolling — but know the internals for interviews.
