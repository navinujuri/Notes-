Planning, Architecture & Stack

Build-any-project flow
Feature list -> UI/UX (end-user + merchant) -> DB schema -> architecture/stack -> bootstrap -> iterate. Decide the schema BEFORE coding routes.
Core features (wallet clone)
End user: on-ramp (add money via bank), P2P transfer, balance view, txn history. Merchant: receive money, cash-out. Two separate apps, one shared DB.
Stack
Next.js (App Router) + TypeScript + TailwindCSS
Prisma + PostgreSQL
NextAuth (Credentials/phone) + bcrypt
Recoil (client state)
Turborepo monorepo
Monorepo layout
apps/
  user-app/       # end-user Next.js app
  merchant-app/   # merchant Next.js app
packages/
  db/     # prisma client + schema (@repo/db)
  store/  # recoil atoms/selectors (@repo/store)
  ui/     # shared React components (@repo/ui)
Two frontends share one DB package -> single source of truth for the wallet schema.

Bootstrap the Monorepo

Scaffold Turborepo
npx create-turbo@latest
# pick npm; gives apps/web,docs + packages/ui,eslint,ts-config
Rename/duplicate apps/web into user-app and merchant-app; delete docs if unused.
Run everything from root
npm install            # installs all workspaces
npm run dev            # turbo runs every app's dev in parallel
Give each app a distinct port: dev script -> "next dev --port 3001".
Local Postgres (Docker)
docker run -e POSTGRES_PASSWORD=mysecretpassword -d -p 5432:5432 postgres
Env vars (per app + db)
DATABASE_URL="postgresql://postgres:mysecretpassword@localhost:5432/postgres"
NEXTAUTH_URL=http://localhost:3001
JWT_SECRET=verysecret
DATABASE_URL lives in packages/db/.env; NEXTAUTH_URL/JWT_SECRET in each app's .env.

Prisma (packages/db)

Init inside the db package
cd packages/db
npm install prisma @prisma/client
npx prisma init
Schema models
model User {
  id      Int     @id @default(autoincrement())
  email   String? @unique
  name    String?
  number  String  @unique
  password String
  Balance Balance[]
  OnRampTransaction OnRampTransaction[]
  sentTransfers p2pTransfer[] @relation("FromUser")
  receivedTransfers p2pTransfer[] @relation("ToUser")
}
model Balance {
  id Int @id @default(autoincrement())
  userId Int @unique
  amount Int
  locked Int
  user User @relation(fields:[userId], references:[id])
}
model OnRampTransaction {
  id Int @id @default(autoincrement())
  status OnRampStatus
  token String @unique
  provider String
  amount Int
  startTime DateTime
  userId Int
  user User @relation(fields:[userId], references:[id])
}
enum OnRampStatus { Success Failure Processing }
Store money in the smallest unit (paise) as Int -> amount*100. Never use float for currency.
Migrate + generate
npx prisma migrate dev --name init
npx prisma generate
npx prisma studio   # GUI to inspect data
Export a singleton client
// packages/db/index.ts
import { PrismaClient } from "@prisma/client";
const prismaClientSingleton = () => new PrismaClient();
declare global { var prisma: undefined | ReturnType<typeof prismaClientSingleton>; }
const prisma = globalThis.prisma ?? prismaClientSingleton();
export default prisma;
if (process.env.NODE_ENV !== "production") globalThis.prisma = prisma;
Singleton avoids exhausting connections on Next.js hot-reload.
Expose it as @repo/db/client
// packages/db/package.json
{
  "name": "@repo/db",
  "exports": { "./client": "./index.ts" }
}
// usage: import prisma from "@repo/db/client";

  key: "totalBalance",
  get: ({ get }) => { const b = get(balanceAtom); return b.amount + b.locked; }

NextAuth + Auth

Install
npm install next-auth bcrypt
npm install -D @types/bcrypt   # in the app
authOptions (Credentials/phone)
// apps/user-app/app/lib/auth.ts
import CredentialsProvider from "next-auth/providers/credentials";
import bcrypt from "bcrypt";
import db from "@repo/db/client";
export const authOptions = {
  providers: [CredentialsProvider({
    name: "Credentials",
    credentials: { phone: {label:"Phone",type:"text"}, password:{label:"Password",type:"password"} },
    async authorize(c) {
      const hashed = await bcrypt.hash(c!.password, 10);
      const existing = await db.user.findFirst({ where: { number: c!.phone } });
      if (existing) {
        const ok = await bcrypt.compare(c!.password, existing.password);
        return ok ? { id: String(existing.id), name: existing.name, email: existing.number } : null;
      }
      const user = await db.user.create({ data: { number: c!.phone, password: hashed } });
      return { id: String(user.id), name: user.name, email: user.number };
    }
  })],
  secret: process.env.JWT_SECRET || "secret",
  callbacks: { async session({ token, session }: any){ session.user.id = token.sub; return session; } }
};
Route handler (App Router)
// apps/user-app/app/api/auth/[...nextauth]/route.ts
import NextAuth from "next-auth";
import { authOptions } from "../../../lib/auth";
const handler = NextAuth(authOptions);
export { handler as GET, handler as POST };
Read session on server
import { getServerSession } from "next-auth";
import { authOptions } from "./lib/auth";
const session = await getServerSession(authOptions);
if (!session?.user) redirect("/api/auth/signin");

Transfer / dashboard page (server component)
// apps/user-app/app/(dashboard)/transfer/page.tsx
import prisma from "@repo/db/client";
import { getServerSession } from "next-auth";
import { authOptions } from "../../lib/auth";
async function getBalance() {
  const s = await getServerSession(authOptions);
  const b = await prisma.balance.findFirst({ where: { userId: Number(s?.user?.id) } });
  return { amount: b?.amount || 0, locked: b?.locked || 0 };
}
export default async function Page() {
  const balance = await getBalance();
  return <div>{/* AddMoney card + BalanceCard(balance) */}</div>;
}
Fetch on the server, pass to client cards. Divide by 100 for display (paise -> rupees).

On-Ramp, Seed & Landing Redirect

On-ramp via server action
// apps/user-app/app/lib/actions/createOnrampTransaction.ts
"use server";
import { getServerSession } from "next-auth";
import { authOptions } from "../auth";
import prisma from "@repo/db/client";
export async function createOnRampTransaction(provider: string, amount: number) {
  const session = await getServerSession(authOptions);
  const userId = session?.user?.id;
  if (!userId) return { message: "User not logged in" };
  await prisma.onRampTransaction.create({ data: {
    provider, status: "Processing", startTime: new Date(),
    token: Math.random().toString(), userId: Number(userId), amount: amount * 100
  }});
  return { message: "On ramp transaction added" };
}
"use server" action callable directly from a client button; no manual API route needed.

Quick tips & gotchas

  • Money = Int in paise (amount * 100). Floats round wrong; convert to rupees (/100) only at display.
  • Keep packages/ui components dumb (props only). Do auth/state in a per-app *Client wrapper so both user-app and merchant-app reuse the UI.
  • RecoilRoot and SessionProvider must be in a 'use client' Providers component; add transpilePackages:['@repo/store','@repo/ui'] to next.config.js.
  • Prisma client must be a global singleton, else Next.js hot-reload spawns dozens of connections ('too many clients').
  • Set NEXTAUTH_URL and JWT_SECRET per app, and DATABASE_URL in packages/db/.env; run migrate/generate from inside packages/db.
  • Prefer server actions ('use server') over manual /api routes for mutations like on-ramp and transfer — less boilerplate, typesafe.
  • P2P transfer must be a prisma.$transaction: lock sender balance, decrement sender, increment receiver — all-or-nothing to avoid partial transfers.
  • getServerSession(authOptions) in server components/actions; useSession() only in client components. Redirect unauthenticated users to /api/auth/signin.
