Recursion — Cheat Sheet

Cohort 2.0 · 6 lessons · quick reference

Anatomy & Template

Every recursion = base case + recursive case
function rec(n) {
  if (n <= 0) return; // base case FIRST
  console.log(n);     // work
  rec(n - 1);         // recurse on smaller input
}
Write the base case first. Missing/wrong base case => RangeError: Maximum call stack size exceeded.
Head vs tail recursion (work before vs after the call)
function tail(n){ if(n===0) return; console.log(n); tail(n-1); } // prints n..1
function head(n){ if(n===0) return; head(n-1); console.log(n); } // prints 1..n
Where you place the work relative to the call flips the print/build order — the stack unwinds LIFO.
Factorial — classic combine(f(n-1))
const fact = n => n <= 1 ? 1 : n * fact(n - 1);
Return-value vs accumulator/void style
// return value
const sum = n => n === 0 ? 0 : n + sum(n - 1);
// accumulator (tail-position)
const sumAcc = (n, acc = 0) => n === 0 ? acc : sumAcc(n - 1, acc + n);

Nearby Squares — warm-up patterns

Map an iterative loop to f(n) = combine(f(n-1))
// iterative
let s = 0; for (let i = 1; i <= n; i++) s += i;
// recursive
const s2 = n => n === 0 ? 0 : n + s2(n - 1);
Mechanical fluency: identify the last unit of work, delegate the rest to the smaller call.
Power a^b via recursion
const pow = (a, b) => b === 0 ? 1 : a * pow(a, b - 1);
Fast power — O(log b) (halve the exponent)
function fastPow(a, b){
  if (b === 0) return 1;
  const half = fastPow(a, Math.floor(b / 2));
  return b % 2 ? a * half * half : half * half;
}
Compute half ONCE then square it — calling fastPow twice makes it O(b) again.
Reverse a number / string recursively
const rev = s => s === '' ? '' : rev(s.slice(1)) + s[0];

Divide & Conquer

Pattern: split → solve subproblems → combine
Subproblems are independent (unlike backtracking). Cost = split + subcalls + combine.
Merge sort — T(n)=2T(n/2)+O(n)=O(n log n)
function mergeSort(a){
  if (a.length <= 1) return a;
  const m = a.length >> 1;
  const L = mergeSort(a.slice(0, m)), R = mergeSort(a.slice(m));
  const out = []; let i = 0, j = 0;
  while (i < L.length && j < R.length) out.push(L[i] <= R[j] ? L[i++] : R[j++]);
  return out.concat(L.slice(i), R.slice(j));
}
The O(n) combine (merge) step dominates here.
Binary search — T(n)=T(n/2)+O(1)=O(log n)
function bsearch(a, x, lo = 0, hi = a.length - 1){
  if (lo > hi) return -1;
  const mid = (lo + hi) >> 1;
  if (a[mid] === x) return mid;
  return a[mid] < x ? bsearch(a, x, mid + 1, hi) : bsearch(a, x, lo, mid - 1);
}
Only one subproblem recursed into => log depth, not a full tree.
Master Theorem — T(n)=aT(n/b)+O(n^d)
// compare d vs log_b(a):
// d >  log_b a  -> O(n^d)
// d == log_b a  -> O(n^d log n)   (merge sort: a=2,b=2,d=1)
// d <  log_b a  -> O(n^(log_b a))
Quick way to read off complexity from the recurrence.

How to Think Recursively

3-step recipe
// 1. Base case: smallest input, answer directly
// 2. Assume rec(smaller) is ALREADY correct (leap of faith)
// 3. Combine that result to solve the current input
Don't trace the whole call tree — trust the subcall and just write the combine step.
Correctness = induction (3 checks)
(a) base case correct; (b) each call strictly shrinks toward base (termination); (c) combine correct assuming subcall is.
Complexity from the recursion tree
// #calls x work-per-call
// linear chain rec(n-1): O(n)
// two branches rec(n-1)+rec(n-2): O(2^n)
// halving one branch: O(log n)
Branching factor + depth => size of tree => time.
Space is O(max depth), always
Every un-returned frame is live at once. Balanced tree=O(log n); skewed/linked-list-shaped=O(n).

Backtracking (choose · explore · un-choose)

Core template
function backtrack(state, path, res){
  if (isSolution(state, path)) { res.push([...path]); return; }
  for (const choice of options(state)) {
    if (!valid(choice)) continue;   // prune
    path.push(choice);              // choose
    backtrack(next(state, choice), path, res); // explore
    path.pop();                     // un-choose (restore!)
  }
}
The pop()/undo is the signature of backtracking — forget it and later branches see dirty state.
Subsets (power set) — 2^n
function subsets(nums){
  const res = [];
  const go = (i, cur) => {
    if (i === nums.length) { res.push([...cur]); return; }
    go(i + 1, cur);                 // skip nums[i]
    cur.push(nums[i]); go(i + 1, cur); cur.pop(); // take nums[i]
  };
  go(0, []); return res;
}
Permutations — n!
function permute(nums){
  const res = [], used = Array(nums.length).fill(false), cur = [];
  const go = () => {
    if (cur.length === nums.length) { res.push([...cur]); return; }
    for (let i = 0; i < nums.length; i++){
      if (used[i]) continue;
      used[i] = true; cur.push(nums[i]);
      go();
      cur.pop(); used[i] = false; // undo
    }
  };
  go(); return res;
}
Prune early = the whole point
Abandon a branch the instant it can't yield a valid solution (N-Queens, Sudoku). Pruning is what beats brute force.

Staircase → Memoization → DP

Climb n stairs (1 or 2 steps) = Fibonacci
// naive: O(2^n) — overlapping subproblems recomputed
const climb = n => n <= 2 ? n : climb(n - 1) + climb(n - 2);
ways(n) = ways(n-1) + ways(n-2). Slow because ways(n-3)... recomputed exponentially.
Top-down memoization — O(n) time / O(n) space
function climbMemo(n, memo = {}){
  if (n <= 2) return n;
  if (memo[n] !== undefined) return memo[n];
  return memo[n] = climbMemo(n - 1, memo) + climbMemo(n - 2, memo);
}
Cache the first result for each distinct subproblem => every state solved once.
Bottom-up DP + O(1) space (roll 2 vars)
function climbDP(n){
  let a = 1, b = 2;                 // ways(1), ways(2)
  for (let i = 3; i <= n; i++) [a, b] = [b, a + b];
  return n <= 2 ? n : b;
}
Recognizing when to memoize
Overlapping subproblems + optimal substructure => DP. This is the recursion→DP bridge.

Stack, Limits & Iterative Conversion

Check / raise the effective limit
// Node: default depth ~10k-15k frames
// node --stack-size=8000 script.js   (KB, risky)
// Better: convert deep recursion to iteration
JS engines (V8) do NOT do tail-call optimization in practice — deep tail recursion still overflows.
Recursion → explicit stack (iterative DFS)
function dfs(root){
  const stack = [root];
  while (stack.length){
    const node = stack.pop();
    visit(node);
    for (const c of node.children) stack.push(c);
  }
}
Same algorithm, but frames live on the heap (large) instead of the call stack (small/limited).
Trampoline (simulate TCO)
const trampoline = fn => (...a) => { let r = fn(...a); while (typeof r === 'function') r = r(); return r; };
const sumT = trampoline(function s(n, acc = 0){ return n === 0 ? acc : () => s(n - 1, acc + n); });
Return a thunk instead of calling; loop drives it — flat stack for deep tail recursion.

Quick tips & gotchas

  • Base case FIRST, and prove the input strictly shrinks toward it — that guarantees termination.
  • "Maximum call stack size exceeded" almost always means a missing/unreachable base case or input not shrinking.
  • Space cost of recursion is O(max depth), even when the logic looks O(1) — every pending frame is live.
  • Two recursive calls => suspect exponential time; if subproblems overlap, memoize to collapse it to O(n).
  • Backtracking: always undo (pop/restore) your choice after the recursive call, or sibling branches inherit dirty state.
  • V8/Node does NOT optimize tail calls — rewrite deep recursion as an explicit-stack loop or use a trampoline.
  • When copying results out of a mutable path in backtracking, push a COPY ([...path]) — not the shared reference.
  • Compute-once then reuse: fastPow/divide-and-conquer must store the half result, not call the function twice.
