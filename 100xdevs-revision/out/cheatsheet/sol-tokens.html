Programs, Accounts and the Token Program — Cheat Sheet

CLI Setup & Config

Install Solana CLI (Anza)
sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
solana --version
Ships solana, solana-keygen, spl-token. Windows: use WSL. Old release.solana.com URL still redirects.
Point CLI at a cluster
solana config set --url devnet   # -ul local | -ud devnet | -ut testnet | -um mainnet
solana config get
Config lives at ~/.config/solana/cli/config.yml. Devnet endpoint: https://api.devnet.solana.com
Generate a wallet / keypair
solana-keygen new              # writes ~/.config/solana/id.json
solana address
Keypair file is the ed25519 secret key JSON array. Back up the seed phrase.
Fund & check balance (devnet)
solana airdrop 2
solana balance
Devnet airdrop is rate-limited (~2 SOL); retry or use faucet.solana.com if throttled.

Accounts & the Solana Data Model

Everything is an account
// Account layout
{ lamports: u64,     // balance
  owner: Pubkey,     // program that controls this account
  data: [u8],        // opaque bytes (state)
  executable: bool,  // true => it's a program
  rent_epoch: u64 }
Only the owner program can mutate an account's data or debit lamports.
Web2 vs Solana data model
Web2: server code + DB rows live together, mutable in place. Solana: programs are STATELESS code; all state lives in separate accounts the program owns. Same program serves many state accounts.
Programs are executable accounts
A deployed program is an account with executable=true, owned by the BPF Loader. It reads/writes other accounts passed into each instruction.
Rent exemption
solana rent 165   # min lamports for a 165-byte account
Accounts must hold a min balance (~2 years rent) to persist, else they get purged. SDKs compute this via getMinimumBalanceForRentExemption.
Why Solana
Parallel execution (Sealevel) because txs declare which accounts they touch; ~400ms slots, sub-cent fees, PoH ordering. Contrast EVM's single-threaded state.

SPL Token — CLI

Create a mint
spl-token create-token --decimals 6
# => Creating token <MINT_ADDRESS>
Mint account stores: supply, decimals, mint authority, freeze authority. It is NOT where balances live.
Create a token account (ATA) to hold it
spl-token create-account <MINT>
One token (ATA) account per (owner, mint) pair holds the balance.
Mint supply into your ATA
spl-token mint <MINT> 1000
Amount is in whole tokens; CLI scales by decimals. Requires you to hold mint authority.
Transfer
spl-token transfer <MINT> 100 <RECIPIENT_WALLET> --fund-recipient
--fund-recipient creates & pays rent for the recipient ATA if missing.
Inspect
spl-token accounts
spl-token supply <MINT>
spl-token account-info <MINT>

Create a Token — JS (@solana/spl-token)

High-level helpers
import { Connection, Keypair, clusterApiUrl } from "@solana/web3.js";
import { createMint, getOrCreateAssociatedTokenAccount, mintTo }
  from "@solana/spl-token";

const conn  = new Connection(clusterApiUrl("devnet"));
const payer = Keypair.generate();  // fund via airdrop first

const mint = await createMint(conn, payer, payer.publicKey, null, 6);
//                                      mintAuth      freezeAuth  decimals
const ata = await getOrCreateAssociatedTokenAccount(
  conn, payer, mint, payer.publicKey);
await mintTo(conn, payer, mint, ata.address, payer, 100 * 10 ** 6);
createMint sends createAccount + initializeMint under the hood. Amounts are base units: multiply by 10**decimals.
Low-level: build the mint manually
import { SystemProgram, Transaction, sendAndConfirmTransaction } from "@solana/web3.js";
import { MINT_SIZE, TOKEN_PROGRAM_ID, getMinimumBalanceForRentExemptMint,
         createInitializeMint2Instruction } from "@solana/spl-token";

const mint = Keypair.generate();
const lamports = await getMinimumBalanceForRentExemptMint(conn);
const tx = new Transaction().add(
  SystemProgram.createAccount({
    fromPubkey: payer.publicKey, newAccountPubkey: mint.publicKey,
    space: MINT_SIZE, lamports, programId: TOKEN_PROGRAM_ID }),
  createInitializeMint2Instruction(mint.publicKey, 6, payer.publicKey, null));
await sendAndConfirmTransaction(conn, tx, [payer, mint]);
The mint keypair must sign createAccount. This is exactly what the CLI/createMint do.

PDAs & Associated Token Accounts

What a PDA is
Program Derived Address: a deterministic address OFF the ed25519 curve, so it has no private key. Only the deriving program can 'sign' for it. Used for program-owned state.
Derive a PDA (JS)
import { PublicKey } from "@solana/web3.js";
const [pda, bump] = PublicKey.findProgramAddressSync(
  [Buffer.from("vault"), user.toBuffer()], programId);
Returns [address, bump]. Bump is the first byte (255..0) that pushes the hash off-curve; store/pass it to re-derive cheaply.
Derive a PDA (Rust)
let (pda, bump) = Pubkey::find_program_address(
    &[b"vault", user.as_ref()], &program_id);
ATA is a PDA
import { getAssociatedTokenAddressSync, TOKEN_PROGRAM_ID,
         ASSOCIATED_TOKEN_PROGRAM_ID } from "@solana/spl-token";
const ata = getAssociatedTokenAddressSync(mint, owner);
// seeds = [owner, TOKEN_PROGRAM_ID, mint] under ASSOCIATED_TOKEN_PROGRAM_ID
For Token-2022 pass programId=TOKEN_2022_PROGRAM_ID (seed changes the derived address).

Token-2022 (Token Extensions)

Program IDs
Token:      TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA
Token-2022: TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb
ATA prog:   ATokenGPvbdGVxr1b2hvZbsiqW5xWH25efTNsLJA8knL
Token-2022 is a separate, backward-compatible program adding opt-in 'extensions'. A mint belongs to exactly one program.
Create a Token-2022 mint (CLI)
spl-token create-token --program-2022 --decimals 9
# or explicit: -p TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb
All later spl-token cmds infer the program from the mint's owner.
Common extensions
MetadataPointer, TokenMetadata (embedded), TransferFee, NonTransferable ('soulbound'), InterestBearing, TransferHook, ConfidentialTransfer, DefaultAccountState, PermanentDelegate.
JS: pick program + size for extensions
import { TOKEN_2022_PROGRAM_ID, ExtensionType, getMintLen }
  from "@solana/spl-token";
const space = getMintLen([ExtensionType.MetadataPointer]);
Extensions grow the mint account; allocate the right space or initializeMint fails.

Token-2022 with Metadata

CLI: metadata embedded in the mint
spl-token create-token --program-2022 --enable-metadata --decimals 9
spl-token initialize-metadata <MINT> "My Token" "MTK" https://.../meta.json
No separate Metaplex metadata account needed — metadata lives inside the mint via the TokenMetadata extension.
JS: single tx (pointer + mint + metadata)
import { TOKEN_2022_PROGRAM_ID, ExtensionType, getMintLen, LENGTH_SIZE,
  TYPE_SIZE, createInitializeMintInstruction,
  createInitializeMetadataPointerInstruction } from "@solana/spl-token";
import { createInitializeInstruction, pack } from "@solana/spl-token-metadata";

const md = { mint: mint.publicKey, name:"My Token", symbol:"MTK",
             uri:"https://.../meta.json", additionalMetadata: [] };
const mintLen = getMintLen([ExtensionType.MetadataPointer]);
const mdLen   = TYPE_SIZE + LENGTH_SIZE + pack(md).length;
const lamports = await conn.getMinimumBalanceForRentExemption(mintLen + mdLen);

const tx = new Transaction().add(
  SystemProgram.createAccount({ fromPubkey: payer.publicKey,
    newAccountPubkey: mint.publicKey, space: mintLen, lamports,
    programId: TOKEN_2022_PROGRAM_ID }),
  createInitializeMetadataPointerInstruction(mint.publicKey,
    payer.publicKey, mint.publicKey, TOKEN_2022_PROGRAM_ID), // pointer -> self
  createInitializeMintInstruction(mint.publicKey, 9,
    payer.publicKey, null, TOKEN_2022_PROGRAM_ID),
  createInitializeInstruction({ programId: TOKEN_2022_PROGRAM_ID,
    mint: mint.publicKey, metadata: mint.publicKey,
    name: md.name, symbol: md.symbol, uri: md.uri,
    mintAuthority: payer.publicKey, updateAuthority: payer.publicKey }));
await sendAndConfirmTransaction(conn, tx, [payer, mint]);
Order matters: createAccount -> initMetadataPointer -> initMint -> initMetadata. Rent must cover mint + packed metadata; the metadata init writes past mintLen so fund for the sum.

Quick tips & gotchas

  • Rent: every account must be rent-exempt or it gets reaped. Always fund via getMinimumBalanceForRentExemption before createAccount.
  • Token amounts are ALWAYS base units. 1 token at 6 decimals = 1_000_000. The CLI scales for you; the SDK does not.
  • Mint account holds supply/authority; balances live in separate token (ATA) accounts — one per (owner, mint). Don't confuse the two.
  • Set mint authority to null (freeze mint) to make supply permanently fixed; set freeze authority to null if you never want to freeze holders.
  • A mint is owned by ONE program. Token vs Token-2022 accounts are not interchangeable — pass the correct programId to every helper (getOrCreateAssociatedTokenAccount, getAssociatedTokenAddressSync, mintTo).
  • PDAs have no private key (off-curve). A program signs for its PDA with invoke_signed + the stored bump; clients can't sign for a PDA.
  • Token-2022 extensions must be chosen at mint creation and change the account size — use getMintLen([...extensions]) or initializeMint reverts.
  • Devnet airdrops are rate-limited; if `solana airdrop` fails, wait and retry or use the web faucet. Use --fund-recipient on transfers to auto-create the destination ATA.
