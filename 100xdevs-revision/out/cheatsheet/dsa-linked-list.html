Linked List — Cheat Sheet

Cohort 2.0 · 6 lessons · quick reference

Node & List Setup (Singly)

Node structure
class Node {
  constructor(val, next = null) {
    this.val = val;
    this.next = next;
  }
}
A list is just a chain of nodes; you hold a reference to head. Empty list = head is null.
List wrapper (track head/tail/size)
class LinkedList {
  constructor() {
    this.head = null;
    this.tail = null;
    this.size = 0;
  }
}
Keeping a tail pointer makes append O(1) instead of O(n).
Build from array
function fromArray(arr) {
  const dummy = new Node(0);
  let cur = dummy;
  for (const x of arr) { cur.next = new Node(x); cur = cur.next; }
  return dummy.next; // head
}
Traverse / print
let cur = head, out = [];
while (cur) { out.push(cur.val); cur = cur.next; }
console.log(out.join(' -> '));
Loop condition is `while (cur)`, not `cur.next` — the latter skips the last node.
Dummy (sentinel) head trick
const dummy = new Node(0, head);
// ...operate on dummy.next...
return dummy.next;
Eliminates edge cases when the head itself may be inserted/deleted.

Core Operations & Complexity

Prepend (push front) — O(1)
head = new Node(val, head);
Append (push back) — O(1) with tail, else O(n)
const node = new Node(val);
if (!this.head) this.head = this.tail = node;
else { this.tail.next = node; this.tail = node; }
this.size++;
Insert at index i
if (i === 0) { head = new Node(val, head); return; }
let prev = head;
for (let k = 0; k < i - 1; k++) prev = prev.next;
prev.next = new Node(val, prev.next);
Link the new node's next BEFORE reassigning prev.next, or you lose the tail.
Delete by value
const dummy = new Node(0, head);
let cur = dummy;
while (cur.next && cur.next.val !== val) cur = cur.next;
if (cur.next) cur.next = cur.next.next;
return dummy.next;
Search / get at index
let cur = head, i = 0;
while (cur && i < idx) { cur = cur.next; i++; }
return cur ? cur.val : null;
Access is O(n) — no random indexing like arrays.
Complexity cheat
Prepend O(1) | Append O(1)*/O(n) | Insert/Delete at pos O(n) | Search O(n) | Access-by-index O(n). Extra advantage over arrays: O(1) insert/delete once you HOLD the node/prev pointer.

Reversing a Linked List

Iterative — 3 pointers (prev/cur/next)
function reverse(head) {
  let prev = null, cur = head;
  while (cur) {
    const next = cur.next; // save
    cur.next = prev;       // flip
    prev = cur;            // advance
    cur = next;
  }
  return prev; // new head
}
O(n) time, O(1) space. Return prev — cur is null at the end.
Recursive
function reverse(head) {
  if (!head || !head.next) return head;
  const newHead = reverse(head.next);
  head.next.next = head;
  head.next = null;
  return newHead;
}
O(n) stack space. `head.next.next = head` flips the pointer coming back up.
Reverse sublist [L, R]
const dummy = new Node(0, head);
let prev = dummy;
for (let i = 1; i < L; i++) prev = prev.next;
let cur = prev.next;
for (let i = 0; i < R - L; i++) {
  const t = cur.next;
  cur.next = t.next;
  t.next = prev.next;
  prev.next = t;
}
return dummy.next;
Head-insertion (LeetCode 92). Positions are 1-indexed.

Cycle Detection (Floyd's Tortoise & Hare)

Has cycle? (bool)
function hasCycle(head) {
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) return true;
  }
  return false;
}
Guard both fast && fast.next before double-hop or you deref null.
Find cycle START node
let slow = head, fast = head;
while (fast && fast.next) {
  slow = slow.next; fast = fast.next.next;
  if (slow === fast) {
    let p = head;
    while (p !== slow) { p = p.next; slow = slow.next; }
    return p; // entry of the loop
  }
}
return null;
After meeting, reset one pointer to head; they meet again at the cycle entry (math: 2(F+a)=F+a+nC).
Cycle length
// after slow === fast:
let len = 1, p = slow.next;
while (p !== slow) { p = p.next; len++; }
return len;
HashSet alternative — O(n) space
const seen = new Set();
let cur = head;
while (cur) { if (seen.has(cur)) return cur; seen.add(cur); cur = cur.next; }
return null;
Simpler but uses extra memory; Floyd's is O(1) space and preferred in interviews.

Add Two Numbers (LeetCode 2)

Digits stored in REVERSE order
function addTwoNumbers(l1, l2) {
  const dummy = new Node(0);
  let cur = dummy, carry = 0;
  while (l1 || l2 || carry) {
    const sum = (l1?.val ?? 0) + (l2?.val ?? 0) + carry;
    carry = Math.floor(sum / 10);
    cur.next = new Node(sum % 10);
    cur = cur.next;
    l1 = l1?.next;
    l2 = l2?.next;
  }
  return dummy.next;
}
Loop while EITHER list has nodes OR carry != 0 — a final carry (e.g. 5+5) needs an extra node.
Forward order variant (LeetCode 445)
Reverse both lists first (or push vals onto stacks), add, then reverse the result — or build result by prepending.

Doubly Linked List (DLL)

Node with prev + next
class DNode {
  constructor(val) {
    this.val = val;
    this.prev = null;
    this.next = null;
  }
}
Push front
const node = new DNode(val);
if (!this.head) { this.head = this.tail = node; }
else { node.next = this.head; this.head.prev = node; this.head = node; }
Push back
const node = new DNode(val);
if (!this.tail) { this.head = this.tail = node; }
else { node.prev = this.tail; this.tail.next = node; this.tail = node; }
Delete a node in O(1) (given the node)
if (node.prev) node.prev.next = node.next;
else this.head = node.next;      // node was head
if (node.next) node.next.prev = node.prev;
else this.tail = node.prev;      // node was tail
DLL's superpower: delete without a prev-search. Basis for LRU cache.
Traverse backward
let cur = this.tail;
while (cur) { console.log(cur.val); cur = cur.prev; }
Extra prev pointer = 2x pointer bookkeeping; every insert/delete must fix BOTH directions.

Classic Two-Pointer Patterns

Find middle (slow/fast)
let slow = head, fast = head;
while (fast && fast.next) { slow = slow.next; fast = fast.next.next; }
return slow;
For even length this returns the SECOND middle. Start fast at head.next for the first middle.
Nth node from end (remove)
const dummy = new Node(0, head);
let fast = dummy, slow = dummy;
for (let i = 0; i < n; i++) fast = fast.next;
while (fast.next) { fast = fast.next; slow = slow.next; }
slow.next = slow.next.next;
return dummy.next;
Advance fast by n first, then move both — gap of n locates the node before target.
Merge two sorted lists
function merge(a, b) {
  const dummy = new Node(0);
  let tail = dummy;
  while (a && b) {
    if (a.val <= b.val) { tail.next = a; a = a.next; }
    else { tail.next = b; b = b.next; }
    tail = tail.next;
  }
  tail.next = a || b; // attach remainder
  return dummy.next;
}
Remove duplicates from SORTED list
let cur = head;
while (cur && cur.next) {
  if (cur.val === cur.next.val) cur.next = cur.next.next;
  else cur = cur.next;
}
return head;
Only advance cur when NOT deleting, else you skip runs of 3+ dupes.
Palindrome check
Find middle -> reverse second half -> compare both halves node by node. O(n) time, O(1) space.

Quick tips & gotchas

  • Always save `cur.next` in a temp before reassigning it during reversal/insertion — otherwise you orphan the rest of the list.
  • Use a dummy/sentinel node whenever the head might be deleted or inserted (delete-by-value, remove-nth, merge, add-two-numbers) to kill head edge cases.
  • For fast pointer double-hops, guard `while (fast && fast.next)` — checking only `fast` throws on `fast.next.next`.
  • Loop traversal uses `while (cur)`, not `while (cur.next)` — the latter drops the last node.
  • Reversal returns `prev` (the last non-null node), not `cur` (which ends up null).
  • In Add Two Numbers, keep looping while `carry` is nonzero even after both lists end (e.g. 5+5 -> extra '1' node).
  • DLL insert/delete must update BOTH prev and next on both neighbors, plus head/tail when touching the ends.
  • Floyd's cycle detection is O(1) space; reserve the HashSet version for when you also need to return the exact node quickly or the list is short.
