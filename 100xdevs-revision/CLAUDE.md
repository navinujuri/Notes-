## What this project is
A self-contained **revision resource for the [100xDevs](https://projects.100xdevs.com) course**.
It scrapes the (login-gated) course, then generates a static, offline website:

- **`out/revision.html`** — level-wise index of **88 tracks / 888 lessons**, searchable, with
  per-lesson progress check-boxes (localStorage). Each lesson links to its notes page + PDF.
- **`out/interview/<track>.html`** — per-track SDE-2 interview prep (per-step bullets + real-world
  example + Q&A). Reached via the 🎯 button on each track.
- **`out/cheatsheet/<track>.html`** — per-track code-forward cheat sheet (grouped snippets + tips).
  Reached via the 📄 button on each track.

Pure static output — no server, no runtime. Open `out/revision.html` directly.

## Commands
**Just view the output:** double-click **`run.cmd`** (Windows) or run **`./run.sh`** (macOS/Linux) —
opens `out/revision.html`, no Node needed.

```bash
npm start          # build + open the index in the browser  ← the usual entry point
npm run build      # regenerate out/revision.* + interview/ + cheatsheet/ from the JSON data
npm run open       # just open out/revision.html
npm run scrape     # (re)scrape the course — opens Chrome, log in once
npm run parse      # decode scrape payload -> out/tracks.json
```

## Architecture / data flow
```
scrape.mjs  ──▶ out/debug-0.html      (homepage; embeds the whole catalog as Next.js RSC)
parse.mjs   ──▶ out/flight.txt        (decoded RSC payload)
parse2.mjs  ──▶ out/tracks.json       (88 tracks × lessons, with URLs)   ← committed source data
build.mjs   ──▶ out/revision.* + interview/ + cheatsheet/                ← render step
```
Interview & cheat content are generated by **multi-agent workflows** (one agent per track), then
merged into JSON that `build.mjs` renders:
```
gen-wf.mjs        ──▶ wf-interview.mjs   ─(Workflow tool)─▶ journal.jsonl
gen-wf-cheat.mjs  ──▶ wf-cheat.mjs       ─(Workflow tool)─▶ journal.jsonl
assemble.mjs <journal> interview-content.json  interview   ← merge → interview-content.json
assemble.mjs <journal> cheatsheet-content.json cheat        ← merge → cheatsheet-content.json
```

## Source-of-truth files (committed, edit these)
| File | Purpose |
|------|---------|
| `out/tracks.json` | Track/lesson tree + URLs. Regenerate via scrape→parse. |
| `interview-content.json` | SDE-2 interview content per track. |
| `cheatsheet-content.json` | Cheat-sheet content per track. |
| `build.mjs` | Rendering + the curated **LEVELS** and **SUMMARIES** — edit here to re-categorize or reword. |

To only re-render (content already present), just `npm run build` — no scrape needed.

## Important notes
- **Never commit secrets.** `.pw-profile/` (the saved login session) and `node_modules/` are
  gitignored. `.pw-profile/` re-creates itself on the next `npm run scrape`.
- **The site is login-gated (NextAuth/Google).** Scraping needs a human login once; the session
  persists in `.pw-profile/`. There is no public track-listing page — the full catalog lives in the
  homepage's RSC payload, which is why `parse.mjs` decodes `debug-0.html`.
- **Content workflows can hit the account session limit** partway. That's fine: re-run with
  `Workflow({scriptPath, resumeFromRunId})` — completed agents replay from cache, only failures
  re-run. Then re-run the matching `assemble.mjs` + `npm run build`.
- **`assemble.mjs` is generic:** `node assemble.mjs <journal.jsonl> [outFile] [kind=interview|cheat]`.
  It merges into the existing JSON with *existing wins*, so hand-written flagship entries
  (e.g. `nextjs-1`) are preserved over regenerated ones.
- Both cohorts (2.0 = `cohort:0`, 3.0 = `cohort:3`) barely overlap — the merged index de-dupes them;
  see `overlap.mjs`.

## Conventions
- Plain Node ESM scripts (`.mjs`), no build step, no framework.
- Generated HTML is theme-aware (light/dark via `prefers-color-scheme`) and self-contained.
